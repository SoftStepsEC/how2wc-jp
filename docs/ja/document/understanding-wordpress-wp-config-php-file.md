`wp-config.php` ファイルは WordPress のコア設定ファイルです。このファイルには、ウェブサイトがデータベースに接続し、適切に機能するために必要な設定が含まれています。

このファイルは WordPress を初めてセットアップしたときに作成され、次のような情報が保存されます。

- データベースの認証情報（データベース名、ユーザー名、パスワード、サーバー）
- サイトの保護を強化するセキュリティキー
- 問題のトラブルシューティングに役立つデバッグ設定

`wp-config.php` ファイルは、WordPress インストールのルートディレクトリにあります。マネージド WordPress ホストをご利用の場合は、このファイルを頻繁に操作する必要はないかもしれません。しかし、ファイルの場所と機能を知っておくと、サイトのカスタマイズやトラブルシューティングを行う際に役立つことがあります。

**注:**このドキュメントは、高度なトラブルシューティングに役立つガイドとして提供されています。ただし、記載されている手順は[サポートポリシー](https://woocommerce.com/support-policy/)の対象外であり、実装に関する直接的なサポートは提供できません。資格のあるWordPress/WooCommerce開発者のサポートをご希望の場合は、[Codeable](https://codeable.io/?ref=z4Hnp)または[Woo Agency Partner](https://woocommerce.com/development-services/)を強くお勧めします。

`wp-config.php` ファイルは、WordPress インストールのルートディレクトリにあります。これは WordPress がインストールされているメインフォルダで、多くの場合、`wp-content`、`wp-includes`、`wp-admin` などのフォルダも含まれています。

`wp-config.php` にアクセスするには:

1. **ホスティングアカウントにログイン**: ウェブホストが提供するコントロールパネルまたはファイルマネージャーを使用します。ほとんどのマネージドWordPressホストは、ダッシュボードにファイルマネージャーを提供しています。
2. **ルートディレクトリに移動**: WordPressサイトのファイルが保存されている「public_html」「www」などの名前のフォルダを探します。
3. **wp-config.php を見つける**: ルートディレクトリに移動したら、「wp-config.php」ファイルを探します。このファイルに変更を加えるとサイトの機能に影響する可能性があるため、開く際は注意してください。
4. **ダウンロードまたは開いて編集**: 安全に編集するためにコピーをダウンロードするか、ホストが許可している場合はファイルマネージャーで直接開くことができます。変更を加える前に必ずバックアップを作成してください。

サイトのファイル マネージャーまたはルート ディレクトリにアクセスする方法がわからない場合は、ホスティング プロバイダーにお問い合わせください。

`wp-config.php` ファイルには、次のような WordPress サイトの主要な設定が含まれています。

- **データベース接続の詳細**: これらの設定には、データベース名、ユーザー名、パスワード、ホストが含まれます。これらの設定により、WordPress はサイトのすべてのコンテンツ、設定、ユーザー情報が保存されているデータベースに接続できるようになります。これらの詳細が変更されたり、正しくなかったりすると、WordPress はデータベースにアクセスできなくなり、サイトが読み込まれなくなります。
- **セキュリティキーとソルト**: WordPress は、セキュリティキーとソルトを使用してユーザーのログインセッションと Cookie を保護します。これらはランダムな文字列であり、ハッカーによるパスワードの解読やサイトの不正アクセスを大幅に困難にします。WordPress はこれらを自動的に生成しますが、セキュリティを強化するために手動で更新することもできます。
- **データベーステーブルプレフィックス**: WordPress はデータベース内のテーブルにデータを保存します。テーブルプレフィックスは、各テーブル名の先頭に表示される `wp_` のような短いコードです。このプレフィックスを変更すると、攻撃者がテーブル名を推測しにくくなり、セキュリティを強化できます。これは、デフォルト設定のサイトで特に役立ちます。
- **デバッグモード**: WordPress でデバッグモードを有効にすると、サイト上のエラーと警告が表示され、トラブルシューティングや問題の特定に役立ちます。開発中やエラーの修正時に便利です。ただし、訪問者に技術情報が表示されないように、本番サイトでは無効にする必要があります。
- **ファイル権限**: `wp-config.php` ファイルには、WordPress インストール内の特定のファイルの読み取り、書き込み、実行を許可するユーザーを制御するファイル権限設定を含めることができます。適切な権限設定は、機密ファイルを保護し、サイトの重要な部分への不正アクセスを防ぐのに役立ちます。
- **メモリ制限**: WordPress は、プラグインの読み込み、大きな画像の処理、複数の訪問者の同時処理など、サイト上のタスクを処理するためにメモリを使用します。`wp-config.php` ファイルでメモリ制限を設定することで、WordPress に多くのリソースを割り当てることができるため、特にリソースを大量に消費するプラグインやテーマを実行している場合は、サイトのパフォーマンスが向上します。

これらの設定により、WordPress が正しく機能し、サイトが保護され、訪問者に安定したエクスペリエンスを提供できるようになります。

`wp-config.php` ファイルを編集する場合は、必ず事前にサイトのバックアップを作成してください。このファイルは WordPress が正常に機能するために**必須** であり、誤りがあるとサイトにアクセスできなくなる可能性があります。万が一何か問題が発生した場合でも、サイトを簡単に復元できるよう、ファイルとデータベースの両方のバックアップを作成してください。

多くのマネージドWordPressホスティングサービスでは、クイックバックアップオプションを提供しています。また、プラグインを使用してフルバックアップを作成することもできます。この手順を実行することで、安全策が講じられ、サイトのダウンタイムを防ぐことができます。

ファイル自体の編集方法については、[WordPress.org の wp-config 編集ガイド](https://developer.wordpress.org/advanced-administration/wordpress/wp-config/) に詳しい情報が記載されています。

WordPressのデバッグモードは、サイト上のエラー、警告、通知を表示する機能です。テーマ、プラグイン、カスタムコードに関する問題の具体的な詳細が明らかになるため、問題のトラブルシューティングに役立ちます。

デバッグ モードを有効にすると、修正が必要な箇所を特定できるため、エラーの解決が容易になり、サイトのパフォーマンスが向上します。

wp-config.php ファイルを使用して WordPress サイトでデバッグ モードを有効にするには:

1. **wp-config.php ファイルにアクセス**: ホストのファイルマネージャーまたは FTP クライアントを使用して、WordPress インストールのルートディレクトリにある wp-config.php ファイルを見つけます。
2. **ファイルを開いて編集**: ファイルのコピーをコンピューターにダウンロードするか、ホストで許可されている場合はファイルマネージャーで直接開きます。
3. **デバッグ行を検索**: ファイル内で次の行を探します。

```
define('WP_DEBUG', false);
```

この行が表示されない場合は、「/* 編集はこれで終わりです。ブログを楽しんでください。*/」という行のすぐ上に追加できます。

4. **デバッグ モードを有効にする**: 次のように false を true に変更します。

```
define('WP_DEBUG', true);
```

5. **変更を保存**: ファイルを保存して閉じ、サイトを更新します。WordPressにエラーと警告が表示されるので、問題のトラブルシューティングに役立ちます。

トラブルシューティングの後、訪問者にエラー メッセージが表示されないように、必ず `WP_DEBUG` を false に戻してください。

また、デバッグ モードを有効にして、エラー メッセージがフロント エンドに読み込まれないようにし、代わりにログ ファイルに保存することもできます。それには、次の手順に従ってください。

1. **wp-config.php ファイルにアクセス**: ホスティングサービスのファイルマネージャーまたは FTP クライアントを使用して、WordPress インストールのルートディレクトリにある wp-config.php ファイルを見つけます。
2. **ファイルを開いて編集**: ファイルのコピーをコンピューターにダウンロードするか、ホスティングサービスでファイルマネージャーから直接開きます（ファイルマネージャーで直接開くことも可能です）。
3. デバッグ設定を追加します:
- ファイル内で、`/* That's all, stop editing! Happy blogging. */` という行のすぐ上に、次の行を探すか追加します。
  

```
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
@ini_set('display_errors', 0);
```

4. **変更を保存**: ファイルを保存して閉じ、サイトを更新します。これでエラーはデバッグログファイルに記録されますが、公開されることはありません。

トラブルシューティングが完了したら、`WP_DEBUG` と `WP_DEBUG_LOG` を false に戻すことを忘れないでください。詳細については、公式 WordPress 開発者サイトの [WordPress デバッグ ガイド](https://developer.wordpress.org/advanced-administration/debug/debug-wordpress/) をご覧ください。
