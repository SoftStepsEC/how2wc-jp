このガイドでは、WooCommerce マーケットプレイスで商品ページやドキュメントを作成するためのフォーマットとライティングの標準を示します。これらのガイドラインに従うことで、マーケットプレイス全体で一貫性が保たれ、拡張機能を閲覧・購入する販売者にプロフェッショナルなエクスペリエンスを提供できます。

商品ページの構造、必要な要素、アセットの仕様については、[商品ページのコンテンツとアセット](https://woocommerce.com/document/product-page-content-and-assets/)を参照してください。

製品のコピー全体では、「WooCommerce 製品名」ではなく、**「WooCommerce の製品名」** という形式を使用します。

この形式により、商品名が [商標規則](https://woocommerce.com/trademark-guidelines/) に準拠し、固有の識別子が先頭に表示されるため、発見可能性と明確性が向上します。

包括的なガイダンスについては、[WooCommerce の文法、句読点、大文字化ガイド](https://developer.woocommerce.com/docs/best-practices/coding-standards/grammar-punctuation-capitalization) をご覧ください。主なポイントは以下のとおりです。

3 つ以上の項目をリストする場合は、**オックスフォード コンマを使用します**。

- ✓「注文の追跡、在庫管理、返金処理を行います。」
- ✗「注文の追跡、在庫管理、返金処理を行います。」

可能な限り受動態ではなく能動態を使用してください。

- ✓「注文が発送されると、拡張機能からメール通知が送信されます。」
- ✗「注文が発送されると、拡張機能からメール通知が送信されます。」

**大文字の使用:**

- 見出しは文頭大文字を使用します（最初の単語と固有名詞のみ大文字にします）
- 強調のためにすべて大文字を使用しないでください
- 製品名、機能、WooCommerce 用語は正しく大文字にします

**スペルと用語:**

- アメリカ英語の綴りを使用してください
- 「ecommerce」（小文字、ハイフンなし）を使用してください
- 「WooCommerce」（「Woocommerce」や「wooCommerce」ではなく）を使用してください

これらの標準に従って、クリーンで一貫性のあるフォーマットを維持してください。カスタムスタイルはマーケットプレイスのデザインシステムに干渉し、一貫性のないエクスペリエンスを生み出します。

**しないでください：**

- 見出しまたは本文を中央揃えにする
- デフォルトのテキストカラーを変更する
- カスタムフォントサイズまたはフォントファミリーを追加する
- 太字または斜体の書式設定を過度に使用する

**する：**

- テキスト全体を通して左揃えにする
- 見出しと段落はデフォルトのスタイルを継承する
- 重要な用語のみに太字を控えめに使う

**しないでください：**

- 1ページあたり複数のH1タグを含める（商品タイトルがH1タグとして機能します）
- 見出しを中央揃えにする
- 見出しにカスタムカラーまたはスタイルを適用する

**する：**

- メインセクションにはH2を使用します。
- H2セクション内のサブセクションにはH3を使用します。
- 論理的な見出し階層を維持します（階層を飛ばさないでください）。

**しないでください：**

- カスタムスタイルのセクションまたはコンテナを作成する
- セクションの周囲に境界線を追加する
- コンテンツ領域に背景色を適用する
- セクション間にカスタムの間隔を追加する
- 列にスタイルを追加する

**する：**

- 標準の段落ブロックと見出しブロックを使用する
- セクションの間隔はマーケットプレイステンプレートに任せる
- スタイルを変更せずにデフォルトの列ブロックを使用する

**しないでください：**

- カスタムCSSで箇条書きや番号のデザインをカスタマイズ
- リスト項目間の間隔をカスタマイズ
- 2階層以上のネストされたリストを作成

**する：**

- 標準の順序なしリストまたは順序付きリストブロックを使用する
- スタイル付きチェックマークには、承認済みのリストクラスを適用する（以下の書式設定ツールを参照）

製品ページやドキュメントでは絵文字を使用しないでください。絵文字はアクセシビリティの問題を引き起こしたり、デバイス間で表示が一致しなかったり、プロフェッショナルではない印象を与えたりする可能性があります。

- ✗「🚀 売上を伸ばそう！」
- ✓「売上を伸ばそう」

外部ウェブサイトへのリンクは含めないでください。サポートおよびドキュメントへのリンクはすべて、MarketplaceのURLにリンクしてください。

- WooCommerce.com でホストされている製品ドキュメントへのリンク
- マーケットプレイスのサポートリソースへのリンク
- 外部のサポートポータル、ドキュメントサイト、ランディングページへのリンクは禁止

WooCommerce マーケットプレイスには、一貫性を保ちながら視覚的に魅力的なコンテンツを作成するための書式設定ツールがいくつか用意されています。

プロフェッショナルなチェックマーク リストを作成するには、次の CSS クラスを使用します。

**プライマリチェックマークリスト** (`wccom-tick-list-primary`)

緑色の円で囲まれた太字のチェックマークを作成します。商品説明の上部にある主な特典リストに使用します。

```
<ul class="wccom-tick-list-primary">  <li>Accept payments globally</li>  <li>Automatic tax calculation</li>  <li>Real-time shipping rates</li></ul>
```

**セカンダリチェックマークリスト** (`wccom-tick-list-secondary`)

軽量の緑色のチェックマークを作成します。ページ全体の機能リストや二次コンテンツに使用します。

```
<ul class="wccom-tick-list-secondary">  <li>Supports 100+ currencies</li>  <li>PCI compliant</li>  <li>Automatic updates</li></ul>
```

重要な情報を強調表示するには、ボックスのショートコードを使用します。

製品のドキュメントを作成するときは、次の追加ガイドラインに従ってください。

- 拡張機能の機能を明確に説明する導入から始めてください。
- 「ドキュメント - インストール」というタイトルの同期パターンを使用してください。
- これは、WooCommerce.com で購入した拡張機能を、両社のお客様がインストールする方法について説明しています。
  
- 製品の機能と設定をすべて文書化する
- よくある問題に対するトラブルシューティングのガイダンスを提供する

詳細なドキュメント フレームワークについては、[サポートとドキュメントのベスト プラクティス](https://developer.woocommerce.com/docs/extensions/best-practices-extensions/support-and-documentation/) を参照してください。

ビジュアルコンテンツを追加する場合:

- 最新のWordPress管理画面のカラースキームを使用する
- 別のテーマをドキュメント化する場合を除き、最新のWordPressコアテーマを使用する
- スクリーンショットは明瞭で読みやすいものにする
- ファイルサイズを最適化する
- スクリーンショットに機密情報や個人情報を含めない
- アクセシビリティのために、すべての画像に代替テキストを追加する
