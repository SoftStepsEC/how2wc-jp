WooCommerce には、投稿、ページ、サイト エディターのパターンやテンプレートに挿入してカタログをカスタマイズしたり、製品を宣伝したり、買い物客に優れたチェックアウト エクスペリエンスを提供したりできるブロックが多数含まれています。

このガイドでは、ブロックの基本的な仕組み、ブロックの設定場所、問題が発生した場合にクラシック テンプレートに戻す方法について説明します。

WooCommerceに組み込まれているサイト編集機能を最大限に活用するには、[ブロックテーマ](https://wordpress.org/documentation/article/block-themes/)を使用する必要があります。ブロックテーマがなくてもブロックを利用することは可能ですが、投稿と固定ページのみで、ショップのテンプレートをカスタマイズするためのサイトエディターにはアクセスできません。

いつものように、ブロックを含む最新の WooCommerce 機能を最大限に活用するには、WordPress と WooCommerce を最新の安定バージョンに更新することを強くお勧めします。

[ブロックテーマ](https://wordpress.org/support/article/block-themes/) は、サイトエディターでカスタマイズ可能なテンプレートで構成されるテーマです。つまり、投稿やページのコンテンツだけでなく、ヘッダー、フッター、サイドバー、その他の要素を含むサイトのすべての領域がブロックで構成されています。WooCommerce のストア編集機能をすべて活用するには、サイトでブロックテーマを使用する必要があります。

利用可能なブロック テーマを調べるには:

1. WordPress管理ダッシュボードから**「外観」>「テーマ」**に移動します。
2. 「新しいテーマを追加」をクリックします。
3. 「ブロックテーマ」オプションをクリックします。

表示されるテーマはすべてサイトエディターで編集できるため、WooCommerce のストア編集機能を活用できます。

ブロックベースのテーマを**カスタマイズ**するには、まず有効化する必要があります。有効化に不安がある場合は、まずテストサイトを作成してテーマの動作を試してみることをお勧めします。

ブロックは、サイトやストアを構成するコンテンツ要素です。投稿やページに追加する各アイテムがブロックとなります。段落、画像、動画、ギャラリー、音声、リストなど、様々な項目にブロックを追加できます。

[エディター](https://woocommerce.com/document/woocommerce-store-editing/the-editor/)を使用して、ページやテンプレートにブロックを追加できます。

各ブロックには、[ブロックツールバー](https://woocommerce.com/document/woocommerce-store-editing/blocks/#block-toolbar) と [ブロック設定サイドバー](https://woocommerce.com/document/woocommerce-store-editing/blocks/#block-settings) にオプションがあります。ブロックのコンテンツはエディターの中央に表示されます。ブロックの目的に応じて、ブロックツールバーとブロック設定サイドバーのオプションは異なります。多くの WooCommerce ブロックは、現在のコンテキストに合わせて動的コンテンツを使用します（例：各顧客のカートに現在何が入っているかを表示するなど）。エディターでは、これらの動的ブロックにサンプルコンテンツが表示されます。

ページ全体にブロックを挿入する方法は複数あります。

- ページの左上には、四角い「+」アイコンが付いたツールバーがあります。このアイコンをクリックすると、ブロック挿入ツールが開きます。ブロックを検索して挿入できます。また、ブロックごとの追加情報と小さなプレビューも表示されます。
- エディターフィールドには、ブロック挿入ツールのオプションが表示されます。エディター内に既に存在するブロックの上、間、または下をクリックすると、「+」アイコンが表示されます。

**プロのヒント:** 新しいブロックを追加する簡単な方法は、「/」に続けて挿入したいブロック名を入力することです。これは[スラッシュコマンド](https://woocommerce.com/document/woocommerce-store-editing/blocks/#searching-for-blocks)と呼ばれます。

次の方法でブロックを検索できます。

- ブロック挿入ツール – ブロック名またはキーワード（例：「画像」や「見出し」）の先頭部分をブロック挿入ツールの検索フィールドに入力します。
- スラッシュコマンド – ブロック名またはキーワードの先頭部分を入力し始めると、自動提案された検索結果が表示されます。

**検索してもブロックが見つからない？** ブロックは特定のコンテキスト（テンプレート、エリア、内部ブロック）でのみ利用可能または挿入されるように制限できます。また、開発者は特定のエリアで利用可能なブロックを制限できます。以下に例を示します。

- 一部のブロックは特定のコンテキストにのみ追加できます（例：「カートクロスセル」ブロックは、カートブロックの「[カートアイテム](https://woocommerce.com/document/woocommerce-store-editing/customizing-cart-and-checkout/cart-block/#cart-items)」セクション内にのみ追加できます）。
- 一部のコンテキストでは、追加できるブロックが制限されます（例：カートブロックの「カート合計」セクションには、*段落*、*画像*、*区切り*、および*利用可能な支払い方法*ブロックのみを追加できます）。
- 一部のブロックは、テンプレートまたはエリアごとに1つしか追加できません（例：「*利用可能な支払い方法*」ブロックは、カートブロックの*カート合計*セクションに1つしか追加できません）。

これらの制限により、特定の実装用にブロックを構築する開発者は、ブロックとストアの機能を壊すコンテキストにブロックが挿入されないようにすることができます。

ページまたはテンプレートからブロックを削除するには、いくつかの方法があります。ブロックを削除するには：

1. ブロックを選択します。
2. ブロックのツールバーまたはリストビューにある3つの点のアイコンをクリックします。
3. 「削除」オプションをクリックします。

ブロックを削除する別の方法は、ブロックを選択して[キーボードショートカット](https://wordpress.org/documentation/article/keyboard-shortcuts-block-editor/)を押すことです。（*Macの場合はControl + Option + Z*、Windowsの場合は*Shift + Alt + Z*）一部のブロックは、選択後にキーボードの「Delete」または「Backspace」キーを押して削除できます。

一部のテンプレートには、変更を公開した後にストアの実際のコンテンツが表示される場所を示すプレースホルダーブロックが含まれています。意図しないエラーやストアの情報の欠落を避けるため、効果を十分に理解した上でテストを実施してから、テンプレートからプレースホルダーブロックを削除することをお勧めします。

ブロックをカスタマイズするには、まずカスタマイズしたいブロックを選択します。コンテンツエリアでブロックをクリックするか、**リスト表示**を使用して特定のブロックを探し、リストから選択することでブロックを選択できます。

ブロックのカスタマイズオプションには、[ブロックツールバー](https://woocommerce.com/document/woocommerce-store-editing/blocks/#block-toolbar)と[ブロック設定](https://woocommerce.com/document/woocommerce-store-editing/blocks/#block-settings)の2つの場所からアクセスできます。

ブロックをクリックすると、そのブロックの上または下にツールバーが表示されます。ツールバーには、そのブロックを操作するための重要なオプションが含まれています。

ブロック ツールバーのアイコンにマウス カーソルを合わせると、各アイコンの機能を説明するツールヒントが表示されます。

#### グローバルブロックツールバーオプション

いくつかのオプションはほとんどのブロックのツールバーに表示されます。以下で説明します。

- **変形**: ブロックを類似のブロックタイプに変更します。例えば、カートブロックをクラシックバージョンのカートに変更したい場合は、このアイコンをクリックするとオプションが表示されます。
- **ドラッグ**: 6つの点のアイコンをクリックしたまま、ブロックをページ内の別のセクションにドラッグします。
- **上/下に移動**: 上矢印または下矢印をクリックすると、ブロックをページ上で1つ上または下に移動します。
- **配置**: ブロックの位置を変更します。通常は左、右、中央に加え、テーマでサポートされている場合はワイドと全幅が表示されます。
- その他のオプション: ツールバーの端にある縦に並んだ3つの点をクリックすると、追加の操作を含むメニューが開きます。
- **コピー**: ブロックをクリップボードにコピーし、他の場所に貼り付けることができます。
- **複製**: 元のブロックのすぐ下にブロックのコピーを作成します。
- **前に追加**: 現在のブロックの上にスペースを作成し、新しいブロックを追加します。
- **後に追加**: 現在のブロックの下に新しいブロックを追加するためのスペースを作成します。
- **スタイルのコピー**: ブロックで使用されているスタイルをコピーします。
- **スタイルの貼り付け**: ブロックのスタイルをコピーした後、選択したブロックにそれらのスタイルを貼り付けます。
- **グループ化/グループ解除**: このオプションを使用すると、複数のブロックをグループ化できます。複数のブロックに色やパディングを適用するのに便利です。
- **ロック**: ブロックの移動や削除を無効にします。[ロックされたブロック](https://woocommerce.com/document/woocommerce-store-editing/blocks/#locked-blocks) の詳細については、こちらをご覧ください。
- **パターンの作成**: カスタマイズしたブロックを保存して、サイト内の他の場所に追加できるようにします。 [ブロックパターン](https://woocommerce.com/document/woocommerce-store-editing/patterns-template-parts/) の詳細については、こちらをご覧ください。
- **名前変更:** リストビューに表示されるブロックに新しい名前を付けます。
- **移動先:** このオプションをクリックし、ページ内の別のセクションをクリックすると、ブロックがそのセクションに移動します。
- **HTMLとして編集**: ブロックのコンテンツをHTMLで編集します。
- **削除**: 現在のブロックをページから削除します。
  

これらの設定は、すべてのブロックで利用できるわけではありません。例えば、[ブロックがロックされている](https://woocommerce.com/document/woocommerce-store-editing/blocks/#locked-blocks)場合、削除オプションは表示されず、移動オプションも表示されない場合があります。

#### ブロック固有のツールバーオプション

各ブロックには、そのブロック固有のツールバーオプションも用意されています。例えば、画像ブロックのツールバーには、リンクの追加、画像の切り抜きなどの設定が含まれています。

#### 親ブロックの選択

別のブロック内にネストされたブロックを選択すると、ブロックツールバーの左側に別のボタンが表示されます。このボタンをクリックすると、選択範囲が現在のブロックの直近の親ブロックに移動します。

ブロック設定サイドバーには、作業中のブロックのカスタマイズオプションがさらに表示されます。各ブロックには独自の設定があり、設定サイドバーで確認できます。

サイドバーが表示されない場合は、エディターの右上隅にある「**保存**」ボタンの横にある設定アイコンをクリックしてください。このアイコンは、2つの不均等な列がある正方形のように見えます。

展開されたサイドバーの上部には、2つのタブがあります。現在選択されているブロックの設定を確認するには、「ブロック」タブが表示されていることを確認してください。利用可能なオプションはブロックごとに異なり、サイドバーで変更できる設定がないブロックもあります。

ロックされたブロックは、開発者やサイト管理者が特定のブロックに対する特定の操作を制限できるブロックエディターの機能です。ロックされたブロックは、フルサイト編集（FSE）やカスタムブロックパターンやテンプレートを作成する際に特に便利です。

独自のレイアウトを作成する際に、エディターでブロックをロックして移動や削除を防ぐことができます。ブロックをロックすると、ブロックツールバーの「その他のオプション」メニュー（3つの点）からロックを解除することもできます。

開発者は、テンプレートのレイアウト、コンテンツ、機能を保護するために、ブロックをロックすることもできます。多くのWooCommerceテンプレートでは、コンテンツが不要なコンテキストから外れて壊れるのを防ぐため、また特定の要素が意図したとおりに順序付けまたは配置されたままであることを保証するために、ブロックをロックしています。このようにブロックがデフォルトでロックされている場合、ロックを解除できないことがよくあります。

例えば、チェックアウトブロック内のほとんどのチェックアウトフィールドは、リサーチとテストの成果である高いコンバージョン率のレイアウトを維持するためにロックされています。これらのロックされたブロックは、リストビューではロックアイコンで表示され、選択されたブロックツールバーにもロックアイコンが表示されます。

リストビューを見ると、いくつかのブロックが他のブロックの中にネストされていることに気付くかもしれません。これらのネストされたブロックを**内部ブロック**と呼びます。内部ブロックを持つブロックを移動または削除すると、その内部ブロックもすべて移動または削除されます。

WooCommerces では、カタログを展示したり、さまざまな方法で製品を宣伝したり、買い物客に優れたチェックアウト エクスペリエンスを提供したりするために使用できるブロックが多数導入されています。

## クラシックテンプレートの使用: プレースホルダーブロックとショートコード

WooCommerceブロックは完全に機能し、ほとんどの拡張機能は現時点でブロックベースのエクスペリエンスを完全にサポートしています。ただし、WooCommerceブロックを使用している間、ストアで実行されている拡張機能が期待どおりに動作しない可能性があります。下位互換性を確保するため、従来のWooCommerceテンプレートとカート+チェックアウトショートコードは、カスタマイズされたチェックアウトフローを必要とする既存のストア、およびブロックテンプレートではまだ対応できない特定のニーズを持つ新規ストアで引き続きご利用いただけます。

ブロックテーマを使用しているものの、ショップでクラシックテンプレートをカスタマイズしている場合、またはストアの編集や新しいブロックテンプレートとまだ互換性のない拡張機能を使用している場合は、ブロックエディターでモダンブロックをクラシックブロックに交換することで、クラシックテンプレートに戻すことができます。

これらのクラシック ブロックは、従来のテンプレートを表示するためのプレースホルダーとして機能するため、ブロック エディターでカスタマイズすることはできませんが、ブロック テーマにクラシック テンプレートを表示できるようになります。

これらのテンプレートのサイトエディターにはクイックスワップオプションが用意されているので、必要に応じてすぐに元に戻すことができます。

### 従来のカートとチェックアウトに戻す

カートとチェックアウトの両方のブロックを、ブロックツールバーの変換ボタンを使ってクラシックバージョンに戻すことができます。手順は以下のとおりです。

1. **ブロックテーマを使用している場合**：**「外観」>「エディター」>「ページ」>**「カート」または「チェックアウト」を選択し、*編集アイコン*****をクリックします。*****非ブロックテーマを使用している場合**：**「ページ」>「すべてのページ」**に移動し、カート/チェックアウトページを見つけて編集します。
2. リストビューを開き、「カート」または「チェックアウト」ブロックを選択します。互換性のない拡張機能がインストールされている場合は通知が表示され、1クリックで元に戻すことができます。
3. ブロックツールバーの左端にある**「変換」**ボタンをクリックします。
4. 「クラシックショートコード」を選択します。
5. ブロックがクラシックショートコードのプレースホルダーブロックに変換されます。
6. 上部バーの「更新」ボタンをクリックして変更を保存します。

新しいブロック テンプレートをクラシック バージョンと交換するには、次の 2 つのオプションがあります。

- 推奨：元に戻したいテンプレートを表示しているときに、設定サイドバーの「テンプレート」タブで「クラシックテンプレートに戻す」ボタンをクリックします。このオプションは、以下のテンプレートで利用できます。
- 商品カタログテンプレート
- 単一商品テンプレート
- 注文確認テンプレート
- 商品検索結果テンプレート
  

- または、テンプレートから新しいブロックを手動で削除し、下の表にリストされている従来のブロックをテンプレートに追加することもできます。
