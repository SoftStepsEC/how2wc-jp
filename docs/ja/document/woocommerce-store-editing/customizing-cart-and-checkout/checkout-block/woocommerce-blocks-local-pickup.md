Blocksのローカルピックアップは、お客様に1つ以上のピックアップ場所を簡単に提供できる、強化された新しい配送方法です。新しく使いやすいローカルピックアップ設定ページを提供することで、ローカルピックアップオプションの設定がより簡単になりました。

この新しい方法は、次の点で合理化された顧客体験を提供します。

- 購入者が受け取り場所を選択する際に配送先住所を入力する必要がない
- 1つ以上の場所への道順や受け取り手順を入力できる
- 店舗住所ではなく受け取り場所に基づいて税金を計算できる
- 購入者が配送方法と配達方法を簡単に切り替えられる、シンプルでモダンなUIを提供する

1. ローカルピックアップを有効化して使用するには、チェックアウトで[チェックアウトブロック](https://woocommerce.com/document/woocommerce-store-editing/customizing-cart-and-checkout/checkout-block/)を使用している必要があります。[こちらのガイド](https://woocommerce.com/document/woocommerce-store-editing/customizing-cart-and-checkout/#using-the-cart-and-checkout-blocks)に従って、古いチェックアウトを置き換えることができます。

2. 完了したら、**WooCommerce -> 設定 -> 配送** に移動すると、新しいローカルピックアップタブが表示されます。

3. 「ローカルピックアップを有効にする」チェックボックスをクリックして、ローカルピックアップを有効にします。

4. (オプション) タイトル入力で、メソッド名を「ローカルピックアップ」から他の任意の名前に変更できます。

5. (オプション)「ローカルピックアップを選択した顧客に価格を追加する」チェックボックスをオンにして、ローカルピックアップに請求する金額を選択します。これにより、コストと税金のステータスの入力が表示されます。

**注**: 店舗受取は配送エリア内で設定されていないため、顧客の所在地に関わらずチェックアウト時に利用できます。店舗受取を一部の配送エリアのみに制限するには、[従来の店舗受取配送方法](https://woocommerce.com/document/local-pickup/)をご利用ください。

チェックアウト時にローカルピックアップが表示されるようにするには、少なくとも 1 つの場所を追加する必要があります。

1. ローカルピックアップページで、以下の場所のセクションが表示されます。

2.「集荷場所を追加」をクリックしてモーダルを開きます。

3. 受取場所には、名前、住所、そして専用の詳細情報セクションが設定できます。これらのフィールドはすべて顧客に表示されるため、顧客はそれらを使って受取場所を見つけることができます。また、顧客がその場所を選択した場合、住所は注文にかかる税金の計算にも使用されます。

4. 場所の追加が完了したら、「変更を保存」をクリックします。

ローカルピックアップが有効になると、チェックアウトに配送方法ブロックとピックアップオプションブロックの 2 つの新しいブロックが表示されます。

配送ブロックはクリック可能で、ピックアップオプションと配送オプションを切り替えることができます。これは、顧客に表示されるものと同じ動作です。

すべてのブロックのタイトルと説明はカスタマイズ可能で、ブロック設定サイドバーにはブロックの外観を制御するための追加設定もいくつかあります。

- **アイコンを表示** – このオプションをオンにすると、「発送」ボタンに配送バンのアイコンが表示され、「受け取り」ボタンに店舗のアイコンが表示されます。
- **送料を表示** – このオプションを有効にすると、顧客が選択できる最も安い配送方法に応じて料金が変化するダイナミック字幕が表示されます。その配送方法のすべてのオプションが無料の場合は、ボタンに「無料」と表示されます。

ピックアップ方法ブロックでは、説明の追加やタイトルの編集だけでなく、テキスト、セパレーター、画像を内部ブロックとして追加することもサポートされています。

**(オプション)** 配送オプションのラベル「発送」と「集荷」をクリックして入力することで、好みの文言に編集することもできます。

ローカルピックアップは、配送ゾーンで有効にする従来のローカルピックアップオプションとは異なります。両方を同時に使用すると、お客様に混乱が生じる可能性があります。両方有効にしている場合は、ローカルピックアップページに通知が表示されます。

配送ゾーンの方法とは異なり、新しいローカルピックアップは、顧客が住所を提供しなかった場合でも常に顧客に表示され、配送先住所を収集しようとしません。

新しいローカルピックアップは、チェックアウトブロック内の専用のステップに表示され、配送方法の選択で選択すると表示されます。これにより、より多くのカスタマイズオプションが提供され、顧客が簡単に区別できるようになります。

従来のローカルピックアップでは、選択された場合、店舗住所（顧客の住所ではない）に基づいて税金が算出されていましたが、新しいローカルピックアップでは、選択された場所の住所に基づいて税金が算出されるため、より適切な税金報告が可能になります。

現時点では、新しいローカルピックアップ機能にはいくつかの制限があります。

- 配送オプション「住所が入力されるまで配送料を非表示にする」が有効になっている場合、チェックアウト画面に受け取り場所が表示されません。
- チェックアウト画面に複数の荷物がある場合、それらの荷物は「受け取り」オプションで1つの荷物として表示されます。これは、すべての荷物に同じ選択が行われ、その注文の税金に同じ住所が使用されるようにするためです。
- 複数の荷物があり、そのうちの1つが集荷不可（ローカルピックアップに対応していない）の場合、「ローカルピックアップ」は無効になります。
- ローカルピックアップは、配送エリアや住所に関係なく、すべてのお客様に提供されます。特定の配送エリアでのみローカルピックアップを制限する必要がある場合は、[従来のローカルピックアップ方法](https://woocommerce.com/document/local-pickup/)を使用することをお勧めします。
