このページは、Woo モバイル アプリを通じて顧客向けの領収書を生成しようとする際に発生する可能性のある一般的な問題のトラブルシューティングに役立つ便利なガイドです。

レシートが生成されない場合は、デバイスがオフラインになっていないかご確認ください。オフラインになっているとレシートにアクセスできない可能性があります。注文詳細画面の「レシートを見る」リンクをタップして、印刷を再試行するか、レシートをお客様に再送信してください。**ご注意ください** アプリのキャッシュをクリアするか、アプリを再インストールすると、デバイスからレシートが削除されます。

Stripe のダッシュボードから領収書を再印刷することもできます。

モバイルアプリデバイスがプリンターと同じWi-Fiネットワークに接続され、プリンターの電源がオンになっていることを確認してください。Bluetoothプリンターの場合は、プリンターがモバイルアプリデバイスとペアリングされ、アクティブに接続されていることを確認してください。

WiFi Direct対応のポータブルプリンター（例：[Brother RJ-4250WB-L](https://brothermobilesolutions.com/products/mobile-printers/ruggedjet-series/ruggedjet-4-series/brother-ruggedjet-rj4250wbl/)）をご利用の場合は、モバイルデバイスがセルラーデータにアクセスできることを確認し、デバイスのWiFi設定からプリンターのWiFiに接続してください。詳細は[こちら](https://woocommerce.com/document/getting-started-with-in-person-payments-woopayments/#portable-wifi-printer)をご覧ください。

他のプリンターでモバイルデバイスからの印刷を有効にするには、追加のソフトウェアが必要になる場合があります。Brother RJ-4250WB-Lをご使用でない場合は、プリンターの販売店にお問い合わせください。

レシートプレビュー画面でプリンターを選択できるにもかかわらず、印刷を開始しても何も起こらない場合は、プリンターを再起動してみてください。エラーメッセージが表示された場合は、その内容をメモし、プリンターのベンダーにサポートを依頼してください。

Brother RJ-4250WB-L の場合は、レシートプレビュー画面で正しい用紙サイズ (4 インチ x 39 インチ) が選択されていることを確認してください。

### レシートの生成または印刷に失敗した場合、アプリ ログにエラーが表示されますか?

**Android:** 印刷エラーはAndroidオペレーティングシステムによって管理されます。ほとんどの場合、システムログに関連情報が含まれますが、「WooCommerce」や「StripeTerminal」などのプレフィックスは含まれません。アプリが記録する可能性のあるイベントは、RECEIPT_PRINT_FAILEDトラッキングイベントのみです。

**iOS:** レシートを生成するためのデータがない場合、アプリはエラーをログに記録します。ただし、レシートの印刷に失敗した場合は、印刷プロセスはアプリの制御外であるため、エラーはログに記録されず、表示もされません。
