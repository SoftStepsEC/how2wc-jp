FTP（ファイル転送プロトコル）は、インターネットまたはローカルネットワークを介してクライアントとサーバー間でファイルを転送する方法です。WooCommerceなどのWordPressサイトを管理する場合、FTPを使用すると、Webサーバー上で直接ファイルをアップロード、ダウンロード、または変更することができます。

これは、テーマやプラグインのインストール、サイトのバックアップの作成、ファイルの直接編集による問題のトラブルシューティングに特に役立ちます。

FTP を使用するには、ファイル転送プロセスを容易にするソフトウェア アプリケーションである FTP クライアントと、サーバーの FTP 資格情報 (通常はサーバー アドレス、ユーザー名、パスワードが含まれます) が必要です。

**注:**このドキュメントは、高度なトラブルシューティングに役立つガイドとして提供されています。ただし、記載されている手順は[サポートポリシー](https://woocommerce.com/support-policy/)の対象外であり、実装に関する直接的なサポートは提供できません。資格のあるWordPress/WooCommerce開発者のサポートをご希望の場合は、[Codeable](https://codeable.io/?ref=z4Hnp)または[Woo Agency Partner](https://woocommerce.com/development-services/)を強くお勧めします。

FTP 経由でサイトのファイルにアクセスするには、FTP クライアントを使用する必要があります。

ウェブホスティングプロバイダーはそれぞれ、この作業を容易にするための様々なツールやリソースを提供しています。一般的に、ホストはサーバーアドレス、ユーザー名、パスワードを含むFTP認証情報を提供します。

広く使われているFTPクライアントの一つに[FileZilla](https://filezilla-project.org/)があります。これは無料で、ほとんどのオペレーティングシステムと互換性があります。FileZillaなどのクライアントにFTP認証情報を入力すると、サーバーに接続し、サイトとの間で直接ファイルを転送できるようになります。

これらの手順についてサポートが必要な場合は、ホストに問い合わせてください。

FileZilla などの FTP クライアントを使用して WordPress サイトに接続する場合、サイトのファイルを見つける方法は次のとおりです。

1. **サーバーに接続**: FTPクライアントを開き、FTP認証情報を入力してサーバーに接続します。
2. **ルートディレクトリに移動**: ルートディレクトリはホスティングサービスによって異なる場合があります。このディレクトリの一般的な名前は、`public_html`、`www`、`htdocs`、またはドメイン名です。
3. WordPressファイルの場所を確認: ルートディレクトリ内で、WordPressインストールフォルダを探します。WordPressがルートディレクトリにインストールされている場合は、wp-admin、wp-content、wp-includesなどのフォルダがルートディレクトリ内にあります。
- **wp-admin**: WordPress管理ファイルが含まれています。
- **wp-content**: テーマ、プラグイン、アップロードファイルが格納されています。
- **wp-includes**: 必須のWordPressファイルが含まれています。
  

テーマやプラグインのインストール、変更、バックアップなど、ほとんどのタスクでは、主に`wp-content`フォルダを操作することになります。このディレクトリには、アクティブなテーマのファイル、プラグイン、そしてWordPressダッシュボードからアップロードしたメディアファイルが含まれています。

WordPressサイト上の各ファイルとフォルダには、所有者、グループ、その他すべてのユーザーがファイルに対して実行できる操作を指定するパーミッション（権限）があります。例えば、「755」や「644」といった数字で表記されたパーミッションは、ファイルやディレクトリへのアクセスと変更を制御します。適切なパーミッションを設定することで、サイトの整合性とセキュリティを維持できます。

WordPressサイトのファイルとフォルダの権限を確認するには、通常、FileZillaなどのFTPクライアントを使用します。権限の確認と解釈方法は以下の通りです。

1. **FTPサーバーに接続**: FTPクライアントを開き、認証情報でログインします。
2. **WordPressファイルへ移動**: ルートディレクトリまたはWordPressインストールフォルダへ移動します。
3. **権限を確認**: FileZillaのファイルリストには、各ファイルの権限が表示されます。列を確認すると、「Permissions」などのラベルが付いているものがあります。「drwxr-xr-x」や「-rw-r--r--」といったエントリがあるはずです。

権限は、一連の文字 (`drwxr-xr-x`) または **3 桁の数字** で表示されます。各文字または数字には特定の意味があります。

- **最初の文字**: ディレクトリ (d) かファイル (-) かを示します。
- 次の3文字 (所有者権限): ファイルの所有者が実行できる操作を示します。
- r: 読み取り権限 (ファイルの内容を表示できます)
- w: 書き込み権限 (ファイルを変更できます)
- x: 実行権限 (ファイルをプログラムとして実行できます)
  
- **次の3文字（グループ権限）**：同じr、w、x表記を使用して、ユーザーグループが実行できる操作を示します。
- **最後の3文字（その他のユーザーの権限）**：他のすべてのユーザーが実行できる操作を示します。

権限は通常 3 桁の数字です。

- **7** (`rwx`): 読み取り、書き込み、実行
- **6** (`rw-`): 読み取りと書き込み
- **5** (`r-x`): 読み取りと実行
- **4** (`r--`): 読み取りのみ

- **ディレクトリ**: 755 (`drwxr-xr-x`) — 所有者は読み取り、書き込み、実行が可能です。その他のユーザーは読み取りと実行のみ可能です。
- **ファイル**: 644 (`-rw-r--r--`) — 所有者は読み取りと書き込みが可能です。その他のユーザーは読み取りのみ可能です。

これらの権限は、スクリプトやその他のサイト機能による必要な操作を妨げることなく、機密ファイルを不正な変更から保護するのに役立ちます。これらの設定を不適切に調整すると、サイトがセキュリティリスクにさらされる可能性があるため、慎重に管理することが重要です。
