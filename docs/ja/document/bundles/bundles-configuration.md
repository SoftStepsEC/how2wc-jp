WooCommerce.com の製品の使用を開始するには、注文確認ページまたはアカウントの [マイサブスクリプション](https://woocommerce.com/my-account/my-subscriptions/) セクションの「ストアに追加」機能を使用できます。

1. [My subscriptions](https://woocommerce.com/my-account/my-subscriptions/) に移動します。
2. インストール予定の商品の横にある「**ストアに追加**」ボタンを見つけます。
3. 画面の指示に従うと、商品が自動的にストアに追加されます。

代替オプションと詳細については、[WooCommerce.com サブスクリプションの管理](https://woocommerce.com/document/managing-woocommerce-com-subscriptions/) を参照してください。

製品バンドルを作成するには:

1. **「製品」>「新規追加」** に移動します。
2. **「製品データ」** パネルで **「製品バンドル」** タイプを選択します。
3. **「バンドル製品」** タブに移動します。
4. **「製品を追加」** をクリックし、検索フィールドを使用して製品を検索し、バンドルに追加します。必要な回数だけ繰り返し、検索結果をクリックするたびに製品を追加します。
5. **「公開」** して保存します。

**バンドル商品** タブに追加したすべての商品には、最小数量や最大数量などのさまざまなパラメータを制御できる専用のオプションセットがあります。これらのオプションについては、[バンドル商品の設定](https://woocommerce.com/document/bundles/bundles-configuration/#bundled-product-options) で説明されています。

**同じ基本商品または可変商品**を複数バンドルすることも可能です。これにより、様々な可能性が生まれます。例えば、[同じ可変商品の複数のバリエーション](https://woocommerce.com/document/bundles/bundles-configuration/#add-variation)をバンドルできます。

顧客に購入を許可する設定と商品に基づいて事前に設定された静的バンドルを作成できます。または、オプション商品、変動商品、または数量を設定できる商品を含む、設定可能なバンドルを作成することもできます。

このプラグインは、**シンプル**商品、**可変**商品、および**シンプル/可変サブスクリプション**をサポートしています。グループ化された商品と外部商品はサポートされておらず、バンドルすることもできません。

製品バンドルはバンドルできません。また、グループ化された製品にバンドルを追加した場合、バンドルは正しく機能しません。

バンドルをカートに追加すると、次のものが作成されます。

1. 親ラインアイテム。
2. その下にグループ化されたいくつかの子ラインアイテム（バンドルされたアイテムごとに 1 つ）。

このアプローチにより、バンドル全体の基本価格を定義したり、必要に応じてバンドルされた個々のアイテムの**価格と税率を保持**したりすることが可能になります。

**注**: カート/注文テンプレートで親アイテムを非表示にする方法をお探しですか？[アイテムのグループ化 > なし](https://woocommerce.com/document/bundles/bundles-configuration/#grouping-options) オプションを選択してください。

製品バンドル タイプでは次のことが可能になります。

- 製品バンドル全体の基本価格を設定する。および/または
- バンドルされた個々の製品の価格を維持する。

バンドルの**基本価格**を設定するには:

1. **「商品」>「すべての商品」** に移動し、編集したいバンドルを選択します。
2. 左側のメニューで **「一般」** タブを選択します。
3. **「通常価格」** を入力し、必要に応じて **「セール価格」** を入力します。
4. **「更新」** を選択します。

バンドルの基本価格のみを設定した場合、その設定は価格に影響しません。

バンドル製品の価格と税率を維持するには:

1. **商品データ > バンドル商品** でバンドル商品を見つけます。
2. **基本設定** セクションの **個別価格** ボックスにチェックを入れます。

**個別価格設定** を有効にすると、**%割引** を定義することもできます。割引は、商品をまとめて購入したり、アクセサリなどのオプションの「アドオン」を購入したりする際のインセンティブとなります。

バンドルされたアイテムの数量が設定可能な場合は通常、**個別価格設定** をチェックする必要があります。これは、[アイテムを一括販売する](https://woocommerce.com/document/bundles/bundles-use-case-sell-in-bulk/) 場合によく見られます。

#### 価格が空白のバンドル製品

**個別価格設定** が無効になっている場合でも、バンドルに含まれるすべての商品には価格を設定する必要があります。WooCommerce では、価格が空白の商品は購入できません。

#### バンドル商品の価格の表示を制御する

**個別価格** を有効にすると、バンドル製品の価格は、i) バンドルの製品詳細ページ、ii) カート、iii) 注文詳細にデフォルトで表示されます。この動作を変更するには、「**詳細設定**」タブの「**価格表示**」オプションを使用してください。

#### 個別の価格とカート/注文の合計

[商品のグループ化](https://woocommerce.com/document/bundles/bundles-configuration/#grouping-options)で**グループ化**が選択されている場合、親カート/注文商品の横に表示される小計には、個別価格の子商品の小計が含まれます。この動作は、[このスニペット](https://woocommerce.com/document/bundles/bundles-snippets/#prevent_subtotals_modifications)で変更できます。

#### サブスクリプション製品のバンドル

サブスクリプション製品をバンドルに追加する場合、サブスクリプションを作成するには、サブスクリプション製品を**個別価格設定**する必要があります。そうでない場合、サブスクリプション製品は1回限りの購入として扱われます。

#### セール中

すべてのバンドル商品が**個別価格**かつ**割引**されている場合、バンドル商品は**セール中**とみなされます。その後、ブロックやショートコードを含む、セール商品が出品されているすべての場所に表示されます。

一方、バンドル製品のいずれかが**個別価格設定**されていないか、**個別価格設定されていても割引されていない**場合、バンドル製品の設定だけではバンドルは**セール中**とみなされません。

シンプルな製品や製品のバリエーションと同様に、バンドルは次のように分類できます。

- **物理** — 物理的に発送または受け取りが必要なバンドル。
- **仮想** — 発送を必要としないバンドル。

このプラグインを使用すると、**仮想**バンドルに物理製品を追加できます。ただし、仮想バンドルを購入すると、そこに含まれるすべての物理製品が仮想製品として扱われます。

配送の観点から、**物理的な**バンドルは**組み立て済み**または**未組み立て**として分類されます。

- **組み立て済み** バンドルは、共通のコンテナに物理的に組み立てられた複数の製品で構成されます。
- **未組み立て** バンドルは、組み立てを必要としないいくつかの製品をグループ化します。

**例: 組み立てられたバンドル**

**Woo Tech** は WooCommerce を使用してコンピュータをオンラインで販売しています。同社のエントリーレベルのデスクトップ PC「**Woo Basic**」は、在庫管理されたコンポーネントリストから組み立てられています。**Woo Basic** PC が組み立てられると、組み立てられたコンポーネントの元の梱包は廃棄され、PC はすべての製品マニュアルと付属資料とともに **1 つの箱** に梱包されて出荷されます。

物理的なバンドルを配送用に構成するには、次の手順に従います。

1. 作成するバンドルが **組み立て済み**、**未組み立て**、または **部分的に組み立てられた** バンドルのいずれであるかを判断します。

- 先ほどご紹介した受注生産の **Woo Basic** PC のような **組み立て済み** バンドルは、1 つの箱で出荷されます。
- **未組み立て** バンドルは、組み立てを必要としない製品のコレクションです。出荷時の特性（重量、寸法）は維持されます。
- **一部組み立て済み** バンドルには、共通の物理的なコンテナが使用されます。バンドルされたアイテムの中には、そのコンテナ内で組み立てられたものもあれば、元の箱で出荷されるものもあります。

1. 左側のメニューの**配送**タブに移動し、**バンドルタイプ**を選択します。**組み立て済み**または**未組み立て**の2つのオプションから選択できます。バンドルが**一部組み立て済み**の場合は、この時点で**組み立て済み**オプションを選択してください。

1. 前の手順で **組み立て済み** オプションを選択した場合は、共通コンテナの **重量**、**寸法**、**配送クラス** を構成します。

アセンブリの重量が静的な場合は、「**組立重量 > 無視**」オプションを選択してください。動的に重量を計算する場合は、「**組立重量 > 保持**」オプションを選択してください。「**保持**」を選択した場合、「重量**」フィールドはコンテナの重量として扱われ、アセンブリの重量は以下の要素を加算して計算されます。

- 指定されたコンテナの**重量**、および
- すべての**組み立て済み**バンドルアイテムの重量。

1. バンドルが**部分的に組み立てられている**場合は、[**バンドル製品**] タブに移動し、組み立てを必要としないバンドル製品ごとに [**個別発送**] を有効にします。

**個別配送** は、「複数の配送に分割」することを意味するものではありません。例えば、デスクトップPCには組み立て済みの部品が含まれている場合がありますが、外付けモニターが専用の箱に入れて同じ配送で同梱されている場合があります。この場合、モニターは **個別配送** として設定する必要があります。

#### 外部注文履行、配送、​​在庫管理サービスに関する注意事項

バンドル タイプの配送設定オプションを使用すると、**WooCommerce Shipping API** に依存した配送方法やその他のビジネス ロジックを使用して配送コストを正確に計算できます。

WooCommerce は、開発者がカート内の商品の配送プロパティを柔軟にカスタマイズできる機能を提供しています。例えば、バンドル商品をカートに追加すると、WooCommerce はそれを合計価格と重量を持つ単一の物理商品として認識します。配送料はこの物理表現に基づいて計算されます。

ただし、注文データを扱う場合にはこの柔軟性は維持されません。WooCommerce から注文データを読み取る外部のフルフィルメント、配送、または在庫管理サービスをご利用の場合、バンドルされた商品は単一の商品としてインポートされない可能性があることにご注意ください。

製品バンドルは、開発者が外部システムと [注文データを同期](https://woocommerce.com/document/bundles/bundles-functions-reference/#get_order_items)するときにバンドルの [物理的な表現を変更](https://woocommerce.com/document/bundles/bundles-functions-reference/#get_product_from_item)できるようにする API を提供します。

[サポートポリシー](https://woocommerce.com/support-policy/)に基づき、カスタマイズ/統合に関するサポートは提供できません。WordPress/WooCommerceのカスタマイズおよびカスタム統合については、[認定WooExpert](https://woocommerce.com/experts/)のご利用を強くお勧めします。

**バンドル製品** タブに追加した製品ごとに、専用のオプションセットが表示されます。これらは**基本設定** と **詳細設定** の 2 つのタブに分かれています。

**最小数量** と **最大数量** — バンドル商品の許容数量範囲を定義するために使用されます。お客様は指定された最小/最大数量の範囲内で任意の数量を選択できます。**最大数量** フィールドを空白のままにすると、最大値は無制限になります。

個別販売オプションが有効になっている場合、バンドルの**最小数量**と**最大数量**は 1 より大きくできません。

製品バンドルでは小数点以下の数量はサポートされません。

**オプション** — 有効にすると、お客様はバンドル商品をバンドルに含めるかどうかを選択できます。バンドル商品の**最小数量**が**0**の場合、このチェックボックスは非表示になります。**最小数量**が**0**のバンドル商品は、自動的にオプションとみなされます。

**バリエーションをフィルター** — **特定のバリエーション** を販売可能にすることができます。これはバリエーション商品にのみ適用されます。

**デフォルトの選択を上書き** — バンドルされた変数製品の事前選択された属性値を上書きするために使用されます。

**個々のバリエーションを追加する**

顧客に属性を選択させることなくバンドルに単一のバリエーションを追加するには、**フィルターバリエーション** を **デフォルトの選択を上書き** と組み合わせて使用​​する必要があります。

- **バリエーションをフィルター** を有効にしてバリエーションを検索し、追加します。
- **デフォルトの選択を上書き** を有効にして、すべてのバリエーション属性に**特定の値** を選択します。この手順は、アクティブなバリエーションが既に特定の属性値にマッピングされている場合でも、このバンドル商品の属性選択ドロップダウンを非表示にする場合に必要です。

バンドルされた可変商品にそれぞれ30種類以上のバリエーションがある場合、計算に影響が出る可能性があります。[詳細と軽減方法については、こちらをクリックしてください](https://woocommerce.com/document/change-limit-on-number-of-variations-for-dynamic-variable-product-dropdowns/)。

**個別価格** — バンドル商品の価格をバンドルの基本価格に加算します。このオプションを有効にすると、**割引** を設定できます。どちらのオプションも、[価格設定](https://woocommerce.com/document/bundles/bundles-configuration/#pricing) セクションに記載されています。

**個別配送** — **商品データ > 配送** で **バンドルタイプ > 組み立て済み** が選択されている場合、バンドルされた商品がバンドル全体（組み立て済みまたは未組み立て）に対してどのように配送されるかを制御します。[配送](https://woocommerce.com/document/bundles/bundles-configuration/#shipping) セクションに記載されています。

**表示** — バンドル製品の表示を制御します: i) バンドルの単一製品ページ、ii) カート/チェックアウト ページ、iii) 注文の詳細を表示するとき (電子メール通知に表示される注文の詳細を含む)。

**決済ゲートウェイ取引の可視性**

バンドル製品を注文/電子メール テンプレートで非表示としてマークしても、支払いゲートウェイがトランザクション詳細アーカイブと顧客通知電子メールにそのバンドル製品をリストすることを妨げることはありません。

**価格の表示** — バンドル製品の価格の表示を制御します。i) バンドルの製品詳細ページ、ii) カート内、iii) 注文詳細表示時。**個別価格** にチェックが入っている場合に適用されます。

**タイトルを上書き** と **短い説明を上書き** — バンドル商品の元のタイトルや短い説明を変更または非表示にするために使用します。上書きされた商品タイトルは、カート/チェックアウト、注文詳細を含むすべてのテンプレートの場所で元の商品タイトルに置き換えられますのでご注意ください。

**短い説明を上書き** フィールドは、ショートコードと HTML マークアップをサポートしています。

**サムネイルを非表示** — バンドルの単一製品の詳細ページを表示するときにデフォルトで表示されるバンドル製品のサムネイルの表示を制御します。

**レイアウト** オプションは、単一商品ページにおけるバンドル商品の詳細の表示形式を制御します。**標準**、**表形式**、**グリッド** の 3 つのオプションから選択できます。

**数量を設定できる**複数のバンドル製品を含むバンドルには、アイテムの数量の表示/比較が容易になるため、**表形式**オプションをお勧めします。

WooCommerce対応のWordPressテーマの多くは、デフォルトで**2列**の商品ページレイアウトを採用しています。1列目にはメインの商品画像とギャラリーを配置し、2列目には商品の詳細とオプションを配置します。このスペース節約のアプローチは、通常フォーム要素が少ない、シンプル商品や可変商品に適しています。

商品バンドルは、入力に複数のステップやインタラクションが必要となる、複雑なフォームになりがちです。テーマの2列レイアウトでは、商品バンドルがスペースに制約され、操作しにくいと感じる場合があります。テーマによっては、個々の商品ページのレイアウトを変更して、**商品の詳細とオプションを1列に表示**できる場合があります。例えば、[Storefront](https://woocommerce.com/storefront/) を使用している場合、[Storefront Powerpack](https://woocommerce.com/products/storefront-powerpack/) プラグインには、以下のオプションが含まれています。

- 商品フォームのコンテンツに幅を広く割り当てる、または
- 商品フォームのコンテンツをメインの商品画像とギャラリーの下に重ねて配置する。

製品バンドルでレイアウトの問題が発生しており、テーマで製品ごとのレイアウト オプションが提供されていない場合は、次の手順を実行します。

1. **商品データ > バンドル商品** に移動します。
2. **フォームの場所** オプションを見つけます。
3. **タブの前** を選択して、フォームのコンテンツを商品画像と概要の後に重ねます。

ほとんどの場合、**Before Tabs** を選択すると、ページ幅全体がバンドルのフォーム コンテンツに割り当てられます。

**Before Tabs** オプションは、すべてのテーマで確実に動作するとは限りません。

ブロック テーマを使用する場合、**フォームの場所** オプションは使用できません。

**アイテムのグループ化** オプションは、カート/注文テンプレートの親/子の行項目の **表示** と **インデント** を制御します。

**グループ化** — カート/注文テンプレート内のバンドル商品は、商品バンドル名にちなんで名付けられた親商品の下にグループ化されます。バンドル商品はインデント表示され、バンドル商品の数量は親商品の数量と同期されます。[組み立て済み](https://woocommerce.com/document/bundles/bundles-configuration/#shipping)バンドル、または[基本価格](https://woocommerce.com/document/bundles/bundles-configuration/#static-pricing)バンドルを作成する場合は、このオプションを選択する必要があります。

**グループ化** オプションを有効にすると、カート/注文テンプレート内の親/子の明細項目の小計が合計され、親項目の実際の小計が集計された小計に置き換えられます。また、個別価格の商品の小計はインデントされ、**小計:** というプレフィックスが付きます。この動作は、[このスニペット](https://woocommerce.com/document/bundles/bundles-snippets/#prevent_subtotals_modifications) で変更できます。

**フラット** — **グループ化** に似ていますが、インデントされる代わりに、バンドルされたカート/注文アイテムには、属するバンドルへの参照が含まれます。

**なし** — すべてのカート/注文テンプレートで親アイテムを非表示にします。バンドルされたカート/注文アイテムには、それらが属するバンドルへの参照が含まれます。バンドルは、すべての顧客向けテンプレートで[グループ化された商品](https://woocommerce.com/document/managing-products/#section-11)に似たものになります。

**「アイテムのグループ化 > なし」** は、[基本価格](https://woocommerce.com/document/bundles/bundles-configuration/#static-pricing)が設定されていない[未組み立て](https://woocommerce.com/document/bundles/bundles-configuration/#shipping)バンドルでのみ機能します。このオプションを選択するには、以下の手順を実行してください。

1. **商品データ > 配送** に移動し、**未組み立て** バンドルタイプを選択します。
2. **商品データ > 一般** に移動し、**通常価格/セール価格** フィールドが空であることを確認します。
3. 最後に、**商品データ > バンドル商品** に移動し、少なくとも 1 つのバンドル商品が **個別価格** になっていることを確認します。

**アイテムのグループ化 > なし** がアクティブな場合、**少なくとも 1 つのバンドル アイテム** が含まれているか選択されている場合にのみバンドルを購入できます。

商品バンドルを使用すると、ワインのケース、石鹸 6 個パック、キャンディー ボックスなど、固定数または可変数の「スロット」を持つパーソナライズされたバンドルを作成して販売できます。

ピックアンドミックス バンドルを作成するには、**最小バンドル サイズ** および **最大バンドル サイズ** オプションを使用して、顧客がパーソナライズされたバンドルを購入するために選択する必要がある製品の最小数量と最大数量を定義します。

[Bulk Discount Deck Pack](https://woocommerce.com/document/bundles/bundles-use-case-sell-in-bulk/) バンドルなどの設定可能なバンドルは、**カートで編集** オプションを有効にすることでカート ページから編集できます。

[Bulk Discount Deck Pack](https://woocommerce.com/document/bundles/bundles-use-case-sell-in-bulk/) バンドルで**カート内で編集**を有効にすると、カート内のバンドルタイトルに「編集」リンクが追加されます。このリンクをクリックすると、現在の設定を基準にしてバンドルを**再設定**できます。完了したら、「**カートを更新**」をクリックして、カート内の設定を新しい設定に**置き換え**ます。

おそらく、**バンドル販売** を見たことがあるでしょう。[クロスセル](https://woocommerce.com/document/related-products-up-sells-and-cross-sells/) と同様に、**バンドル販売** は、**現在表示されている製品と一緒に便利なアドオンを購入する** というオファーです。

クロスセルと比較して、バンドル販売は見つけやすく、すぐにコンバージョンに繋がりやすいという利点があります。そのため、以下のようなオプション商品を推奨するのに最適です。

- 閲覧中の商品と**頻繁に購入**され、密接に関連している。
- **コストが低い**。
- 通常は**利益率が高い**。

製品バンドルを使用すると、既存の製品にバンドル販売を追加できます。

- **商品** 管理ページに移動します。
- バンドル販売の候補として適切と思われる商品を **編集** します。
- **商品データ** > [リンクされた商品]** に移動し、**バンドル販売** フィールドを見つけます。
- 推奨するオプション商品を **検索** して **追加** します。
- 必要に応じて、**繰り返し** ます。
- 下記に **バンドル販売タイトル** を入力します（例：「よく一緒に購入される商品…」）。
- 必要に応じて、追加のインセンティブとして **バンドル販売割引** を入力します。
- 変更を保存するには、商品を **更新** してください。

バンドル販売として提供できるのは、**Simple** および **Simple Subscription** 製品のみです。

**Variable** 製品を **Simple** 製品のバンドル販売として提供する必要がありますか? 回避策があります:

1. シンプル商品を商品バンドルに変換します。
2. **商品データ > バンドル商品** に移動します。
3. 可変商品を追加します。
4. **オプション**、**個別価格**、**個別発送** のボックスにチェックを入れます。

WooCommerce を使って在庫管理をしている場合、商品バンドルを販売するたびに、その内容物の在庫数も減っていくことに気付くでしょう。このプロセスは WooCommerce によってシームレスに処理されます。商品バンドルは、バンドル内の商品ごとに独立したカート/注文明細項目を作成します。

在庫管理された製品を含む製品バンドルを表示すると、プラグインがコンテンツの在庫状況と在庫数量に基づいて在庫状況を調整することにも気付くでしょう。

- 在庫が不足している製品を含むバンドルは、カタログに **在庫不足** ステータスで表示され、その内容が補充されるまで購入できません。

- WooCommerce で商品の[在庫残数](https://woocommerce.com/document/configuring-woocommerce-settings/products/#inventory)を顧客に表示するように設定している場合、バンドルレベルで在庫管理を行っていない場合でも、この情報がバンドルに引き継がれます。例えば、フルーツポップのバンドルで特定のフレーバーに「ブラックベリー 4個以上」などの最小購入数を設定したにもかかわらず、ブラックベリーの在庫が2個しかない場合、バンドルは**在庫切れ**になります。

**重要:** この拡張機能では、製品バンドルで使用するために製品の在庫の一部を予約することはできません。

**注:** デフォルトでは、「在庫あり」は、商品の**在庫タブ**で**在庫数量を追跡**オプションが有効になっている場合にのみ表示されます。[ドキュメントのこのセクションのスニペット](https://woocommerce.com/document/configuring-woocommerce-settings/products/#inventory)を使用すると、在庫数量が追跡されていない場合でも、在庫のある商品に「在庫あり」と強制的に表示することができます。

同時に、このプラグインでは各バンドルに専用のSKUを割り当て、バンドルの内容とは独立して在庫を管理できます。これは、バンドルに含まれる商品以外のリソースが限られているため、一定期間内に無制限にバンドルを組み立てたり、フルフィルメントしたりできない場合に便利です。以下に例を示します。

**Alpha Estate** は家族経営のワイナリーで、WooCommerce を使ってワインをオンラインで販売しています。ボトル単位での販売に加え、商品バンドル機能を使って手作りの木製ケースにワインを詰めたボトルを販売しています。現在、ワイナリーには少なくとも数千本の赤ワインが在庫としてありますが、フルフィルメントチームは1日に20個以上の木製ケースを組み立てて出荷することができません。

このような課題に対処するには、**バンドル レベルで** 在庫管理を有効にすることができます。

1. 在庫数を制限したい商品バンドルを**編集**します。
2. **商品データ > 在庫**タブに移動します。
3. **在庫を管理しますか？**ボックスにチェックを入れます。
4. **在庫数量**を希望の値に設定します。

バンドルの**在庫状況**と**在庫数量**は、バンドルに含まれる商品の在庫状況/数量とは独立して設定できます。これは直感に反するように思えるかもしれませんが、これにより、前述の例のように、複雑な在庫管理が求められるシナリオでも商品バンドルを活用できるようになります。

このアプローチの1つの意味は、在庫状況と在庫数量のプロパティだけでは**在庫不足**のバンドルを特定できないということです。プラグインは、この確認を支援する専用のレポートを提供しています。レポートを表示するには、以下の手順に従ってください。

1. **WooCommerce > Analytics > Bundles** に移動します。
2. **表示 > Stock** をクリックします。

在庫管理対象商品の販売は、その商品を含むすべてのバンドルの在庫状況と数量に影響します。バンドルの在庫を十分に確保できるよう、拡張機能は以下の機能を提供します。

- ストア全体のバンドルの在庫残数を計算します。
- バンドルレベルで在庫管理を行っていない場合でも、影響を受けるすべてのバンドルについて、在庫なし/在庫不足の通知を送信します。(https://woocommerce.com/document/bundles/bundles-configuration/#managing-stock)

商品バンドルから在庫通知を受け取るには、**WooCommerce > 設定 > 製品 > 在庫** で在庫通知が有効になっていることを確認してください。

外部の在庫管理サービスは、バンドル/キットSKUのサポートに独自の手法を採用しています。バンドル/キットSKUを複数のコンポーネントSKUに自動的に分割する外部の在庫管理サービスをご利用の場合は、バンドルデータをサービスプロバイダーがサポートする形式でエクスポートするために、追加の統合作業が必要になる場合があることにご注意ください。

商品バンドルは、開発者が注文データを外部システムにエクスポートするときに[動的なキット SKU を生成](https://woocommerce.com/document/bundles/bundles-filters-reference/#woocommerce_bundle_container_order_item_sku)できるようにする API を提供します。

デフォルトでは、顧客がバンドルの製品ページにアクセスすると、各製品バンドルの在庫状況はバックグラウンド プロセスによって自動的に同期されます。

すべてのバンドルの在庫状況をプログラムで同期するには、次の **WP CLI** コマンドを使用できます。

```
wp wc pb sync-stock
```

バックグラウンド プロセスによるバンドルの在庫ステータスの同期を防止し、代わりに **WP CLI** 経由でオンデマンドで在庫ステータスの同期タスクのみを実行する場合は、次のスニペットを使用できます。

```
add_filter( 'woocommerce_pb_sync_stock_via_cli', '__return_true' );
```

**ヒント**: これはトラフィックの多いサイトに特に役立ちます。

あるいは、在庫状況の同期をバックグラウンド プロセスではなく **アクション スケジューラ** で処理したい場合は、次のスニペットを使用できます。

```
add_filter('woocommerce_pb_sync_parent_stock_use_action_scheduler', '__return_true');
```

**ヒント**: これは、トラフィックの多いサイトにも便利な代替手段です。

商品バンドルは、**Woocommerce > Analytics** の下に専用の**バンドル**セクションを作成します。ここから以下の操作が可能です。

- ストアで購入されたバンドルの詳細な収益分析を表示します。
- **在庫不足** のバンドルを特定し、その内容を補充します。

**重要**: これらのレポート機能には**WooCommerce 4.8以上**が必要です。[WooCommerce Analytics](https://woocommerce.com/document/woocommerce-analytics/)メニュー項目は、一部のマネージドホスティングプラットフォームでは利用できない場合がありますのでご注意ください。

**分析 > バンドル** の **収益** レポートでは、バンドルの販売実績に関する詳細な分析情報が提供されます。

レポートでは、選択した期間について次のメトリックが生成されます。

- **販売されたバンドル**。
- **販売されたバンドルアイテム**。
- **純売上高**。
- **バンドルを含む注文**。

計算された**純売上高**の数値には、次の収益が含まれていることに注意してください。

- バンドルの**基本価格**、および
- 購入時に**個別価格**だったバンドルアイテムの価格。

対照的に、**Woocommerce Analytics > Products** に移動して特定のバンドルを検索すると、報告される **Net Sales** 数値には、その **base price** から発生する収益のみが含まれます。

**アナリティクス > バンドル** ページには **在庫** レポートも含まれており、これを使用して [在庫不足](https://woocommerce.com/document/bundles/bundles-configuration/#inventory) のバンドルを特定し、補充することができます。アクセスするには、**表示 > 在庫** を選択します。

**在庫** レポートは、どのバンドルが在庫切れであるかを判断するのに役立つだけでなく、リストされているすべてのバンドルでどの製品を補充する必要があるかを提案します。
