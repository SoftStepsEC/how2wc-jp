どんなビジネスでも、顧客に返金する必要が生じることがあります。WooCommerce を使えば、自動または手動の2つの方法で返金を迅速に処理できます。

- 自動返金を推奨します。自動返金を利用する場合は、注文時に使用された支払い方法で顧客に返金されます。これはすべて、WooCommerce 管理画面の「注文編集」ページから行えます。
- 自動返金を処理するには、顧客が対応する決済ゲートウェイ経由で支払いを行っている必要があります。
  
- 手動返金の場合、WooCommerceストアでは注文が返金済みとして記録されますが、顧客に返金する手続きは手動で行う必要があります。
- ほとんどの場合、販売者アカウントにアクセスし、そのインターフェースから返金する必要があります。これは、使用する決済ゲートウェイによって異なります。
  

**注:**注文のステータスを「キャンセル」または「返金」に直接変更しても、顧客への返金は自動的に行われません。下記の返金手続きを必ず完了してください。

返金を自動的に処理すると、WooCommerce の注文のステータスが「返金済み」に変更され、注文の支払いに使用したのと同じ支払い方法で顧客に返金されます。すべて 1 つのプロセスで実行されます。

WooCommerceで返金処理を自動で完了したら、それ以上の手順は必要ありません。WooCommerceで自動返金を処理する一般的な手順は以下のとおりです。

1. **WooCommerce > 注文** に移動します。
2. 返金する注文を選択します。
3. 注文概要に移動し、**返金** ボタンを選択します。
4. 各品目に表示されるテキストボックスで、返金する商品の横にある**数量フィールド** を使用します。返金額は入力された数量に基づいて自動的に計算されます。在庫管理されていない場合は、商品数量を調整せずに、各品目の返金額を入力することもできます。
5. 必要に応じて、返金に関するメモを追加します。
6. **[決済ゲートウェイ] 経由で $[返金額] を返金** を選択します。
7. 注文ステータスを確認します。注文金額の全額が返金されない場合、注文ステータスは「返金済み」に変更されず、購入者に送信されるメールには「一部返金」と記載されます。

**注記：**

アカウント残高は払い戻し処理に影響します。処理方法は決済ゲートウェイごとに異なりますので、ご利用の決済ゲートウェイによる払い戻し処理方法（特にアカウント残高がマイナスの場合）に関する正確な情報については、決済処理業者のドキュメントをご参照ください。

#### 支払いが払い戻されたかどうかはどうすればわかりますか?

使用された支払い方法を通じて払い戻しが処理されると、支払いゲートウェイは通常、資金が支払い方法を通じて返金されたことを示す注文メモを追加します。

**注文メモ** で「返金が完了しました」というメッセージを確認し、返金が問題なく処理されたことを確認してください。必要であれば、ご注文に使用した決済ゲートウェイでアカウントにログインして、そちらでもご確認いただけます。

通常、手動返金オプションを使用する必要があるのは、[代金引換](https://woocommerce.com/document/cash-on-delivery/)や[小切手決済](https://woocommerce.com/document/cheque/)など、自動返金に対応していない決済ゲートウェイで支払いを受けた場合のみです。注文を手動で返金する場合、WooCommerce外で顧客の資金が返金されることを確認するのは、販売者様の責任となります。

- 確認：決済ゲートウェイのドキュメントまたはサポートで、手動での払い戻しが可能かどうかをご確認ください。以下のいずれかの方法で払い戻し手続きを行ってください。
- お支払い方法のアカウントにログインし、払い戻し手続きを行う。
- 銀行口座から手動で送金する。
  
- **対象範囲**：返金は通常、商品、税金、送料に及びます。ただし、決済ゲートウェイで請求される取引手数料は通常、返金されずに失われます。
- WooCommerce：返金は注文ページから処理できます。注文を手動で返金するには、以下の手順に従います。
1. **WooCommerce > 注文** に移動します。
2. 返金する注文を選択します。
3. 注文概要に移動し、「返金」を選択して処理を開始します。
4. 各項目に表示されるテキストボックスに、返金する**商品の数量**を指定します。返金額は、返金する商品に基づいて自動的に調整されます。在庫管理が行われていない場合は、商品数量を調整せずに返金額を入力することもできます。
5. 必要に応じて、**返金メモを追加**します。
6. **「$[返金額]を手動で返金」**を選択します。
  

**注:**注文の全額が返金されない場合、注文のステータスは「返金済み」に変更されず、購入者に送信されるメールには「一部返金」と記載されます。

返金対象として選択された商品は、「返金済み」と表示されます。注文に返金が適用されると、以下のように表示されます。

「注文の返金」メールに使用されるメール テンプレートは 1 つだけですが、注文の全額を返金したかどうかによってメールの表現が異なります。

- 注文全額が返金されない場合は、購入者に送信されるメールに「一部返金」と記載されます。

- 注文全額が返金される場合は、メールに注文が返金された旨が記載されます。

[WooCommerce が送信するメール](https://woocommerce.com/document/configuring-woocommerce-settings/emails) の詳細については、こちらをご覧ください。

返金する商品の数量を、全体に対する割合として入力できます。これは、一部返金を行う場合に、関連する税金を自動計算するのに便利です。

払い戻しが完了すると、数量フィールドは常に最も近い整数に切り上げられます。ただし、税金フィールドの計算には影響はなく、払い戻しの送信前に入力された端数を反映し続けます。

以下の例では、商品価格の4ドルと、それに相当する税率の返金が必要です。合計金額（4ドル）を「合計金額」欄に直接入力すると、税額は自動計算されず、手動で計算する必要があります。

ただし、返金額が商品の合計金額の何パーセントであるかを計算し、それを**数量** フィールドに入力すると、他の税金フィールドが自動的に計算されます。

計算式は次のとおりです: **払戻金額/合計 = 数量** またはこの例では **4/15 = 0.26666667** – 分数をコピーして **数量** フィールドに貼り付け、**Tab** キーを押して計算を開始します。

払い戻し手続きの際に、「払い戻し商品の再入荷」ボックスにチェックを入れることで、商品を再入荷することができます。

- 決済ゲートウェイのドキュメントをご確認ください。
- WooCommerce.com の有料拡張機能の場合は、ご購入いただきお試しください。[30日間返金保証](https://woocommerce.com/refund-policy/)をご用意しており、手間なく返金いたします。
- 決済ゲートウェイプロバイダーにお問い合わせください。
- [お問い合わせください](https://woocommerce.com/my-account/contact-support/)。

最適な出発点は、ポリシーと顧客が払い戻しをリクエストする方法を説明するページを作成することです。

返金ポリシーページのテンプレートサンプルと参考資料として、[返金ポリシーページガイドライン](https://woocommerce.com/document/refund-policy-page-guidelines/)をご用意しています。また、[WooCommerce.comの返金ポリシー](https://woocommerce.com/refund-policy/)も参考にご活用ください。

次のステップは、返金ポリシーを顧客に共有することです。顧客に返金ポリシーを公開するには、いくつかの方法があります。

- ストアのメニューにポリシーリンクを追加する（多くのストアではフッターメニューにこの方法を採用しています）
- 注文確認メールにポリシーリンクを追加する（[ショップメールのカスタマイズ方法](https://woocommerce.com/posts/how-to-customize-emails-in-woocommerce/) に関する詳細はこちら）

[支払いゲートウェイが払い戻しをサポートしているかどうかを確認する方法](https://woocommerce.com/document/check-if-payment-gateway-supports-refunds-subscriptions-preorders/)にアクセスし、クラスコンストラクターのクラスプロパティを調べてください。
