title: サイトのクロスブラウザテスト

ユーザーが好むデバイス、ブラウザ、場所に関係なく、シームレスなショッピング体験を提供するためには、複数のブラウザで WooCommerce ウェブサイトをテストすることが不可欠です。

クロスブラウザテストを実施することで、サイトの機能、外観、または使いやすさに影響する可能性のある互換性の問題を特定して解決できます。

**注:**このドキュメントは、高度なトラブルシューティングに役立つガイドとして提供されています。ただし、記載されている手順は[サポートポリシー](https://woocommerce.com/support-policy/)の対象外であり、実装に関する直接的なサポートは提供できません。資格のあるWordPress/WooCommerce開発者のサポートをご希望の場合は、[Codeable](https://codeable.io/?ref=z4Hnp)または[Woo Agency Partner](https://woocommerce.com/development-services/)を強くお勧めします。

ブラウザによってコードの解釈方法が異なり、サイトの外観や機能に不一致が生じることがあります。クロスブラウザテストは、**ユーザーが使用するブラウザやデバイスに関係なく、ウェブサイトがすべてのユーザーにとって適切に機能することを保証します**。

複数のブラウザでテストすることで、**問題を早期に発見・修正**し、すべてのユーザーにスムーズで信頼性の高いエクスペリエンスを提供できます。これにより、ユーザーとの信頼関係が構築され、テクニカルサポートの必要性が軽減され、ストア全体のパフォーマンスと成功につながります。

ブラウザ間でサイトをテストするのに役立つツールが利用可能です。

- [BrowserStack](https://www.browserstack.com/)
- [LambdaTest](https://www.lambdatest.com/)
- [Sauce](https://saucelabs.com/products/cross-browser-testing) [Labs](https://saucelabs.com/products/cross-browser-testing)

これらのツールを使用すると、モバイルデバイスを含む様々なブラウザやデバイスでサイトをテストできます。また、異なる場所からテストして、他の国の訪問者がどのような体験をしているかを確認することもできます。

**これらはサードパーティのリソースであるため、当社のハピネス エンジニアはこれらのツールの使用に関する問題をサポートできませんのでご了承ください。** ご質問がある場合は、それぞれのサードパーティ プロバイダーに直接お問い合わせください。

ホスティング プロバイダーによっては、WooCommerce サイトでクロスブラウザー テストを実行するためのツールを提供または推奨している場合もあります。

個々のページ要素を検査すると、サイトのテストに役立ち、Web サイトのさまざまな部分がどのように機能し、相互作用するかについての理解を深めることができます。

最も一般的なウェブブラウザでこれを行う方法は次のとおりです。

1. **右クリックして検査**：検査するページに移動します。検査する要素を右クリックし、コンテキストメニューから**検査**をクリックします。[DevTools](https://developer.chrome.com/docs/devtools/open) パネルが開き、HTML 構造内で選択した要素がハイライト表示されます。
2. **[要素] タブを使用する**：[要素] タブには、ページの HTML 構造が表示されます。DevTools で HTML コードのさまざまな部分にマウスポインターを合わせると、ウェブページ上でそのセクションがハイライト表示されるため、正確な要素を特定しやすくなります。
3. **スタイルを変更する**：[要素] タブの右または下にある [スタイル] パネルでは、選択した要素に適用されている CSS を確認および編集できます。変更はウェブページにリアルタイムで反映されます。これらの変更は CSS サイトのスタイルシートに直接保存されないことに注意してください。ブラウザでページを更新すると、変更内容は消えます。これらの変更はトラブルシューティングと潜在的な変更のプレビューのみを目的としています。

1. **右クリックして検証**: Chrome と同様に、ページの確認したい部分を右クリックし、コンテキストメニューから [検証] を選択します。
2. **HTML と CSS の検証と編集**: [インスペクタ](https://firefox-source-docs.mozilla.org/devtools-user/page_inspector/how_to/open_the_inspector/index.html) ツールが開き、ページの HTML と CSS が表示されます。インスペクタで HTML または CSS を直接編集して、すぐに変更内容を確認できます。インスペクタで行った変更は、CSS サイトのスタイルシートに直接保存されないことに注意してください。ブラウザでページを更新すると、変更内容は消えます。インスペクタは、トラブルシューティングと潜在的な変更内容のプレビューのみを目的としています。

1. **右クリックして検査**: 要素を右クリックし、[**検査**] をクリックして [開発者ツール](https://www.microsoft.com/en-us/edge/learning-center/how-to-inspect-elements?form=MA13I2) を開きます。これらのツールで行った変更は、CSS サイトのスタイルシートに直接保存されないことに注意してください。ブラウザーでページを更新すると、変更内容は消えてしまいます。これらのツールは、トラブルシューティングと潜在的な変更のプレビューのみを目的としています。
2. **DOM を操作する**: [要素] タブのドキュメント オブジェクト モデル (DOM) ツリーを使用して、Web ページの構造を表示および操作します。CSS の変更は [スタイル] ペインで行うことができます。このペインでは、入力するとページが動的に更新されます。

1. **開発者向けメニューを有効にする**：まず、*Safari > 設定*から[開発メニュー](https://developer.apple.com/documentation/safari-developer-tools/inspecting-safari-macos)を有効にします。**詳細**タブで、**Web開発者向けの機能を表示**ボックスにチェックを入れます。
2. **要素の検査**：開発メニューが有効になったら、Webページを右クリックします。**要素の検査**をクリックするか、画面上部の*開発*メニュー項目を使用して**Webインスペクタを表示**をクリックします。Webインスペクタで行った変更は、CSSサイトのスタイルシートに直接保存されないことに注意してください。ブラウザでページを更新すると、変更内容は消えます。これらはトラブルシューティングと潜在的な変更のプレビューのみを目的としています。

個々の要素を検査しているときに、[String Locator](https://wordpress.org/plugins/string-locator/) などの別のプラグインを使用して、調べている要素がどのテーマまたはプラグイン ファイルで提供されているかを判断することもできます。
