WooCommerce.com から製品をご購入いただきありがとうございます。このドキュメントでは、WooCommerce.com マーケットプレイス製品のサブスクリプションを管理する方法について説明します。

当サイトから購入した拡張機能やテーマはそれぞれ年間または 2 年間のサブスクリプションとなり、WooCommerce.com アカウント ダッシュボードの [**My Subscriptions**] セクションで表示および管理できます。

ここから、購入したライセンスをサイトにインストールし、有効化することでサポートやアップデートを受けることができます。また、WooCommerce.com 上の別のアカウントに購入を移行したり、別のアカウントとサブスクリプションを共有したりすることも可能です。他のサイトで必要になった場合は、ライセンスを追加購入することも可能です。

WooCommerce.com Update Managerは、Woo Marketplaceで購入したWooCommerce拡張機能とテーマの製品アップデートとサポートを管理するプラグインです。Woo Marketplaceから初めてサブスクリプションをインストールする際、このプラグインをインストールするように求められます。

Woo Marketplace の拡張機能を既にご利用の場合は、WP Admin の **プラグイン** ページ、または **WP Admin > WooCommerce > 拡張機能 > マイサブスクリプション** から WooCommerce.com Update Manager をインストールするよう求めるプロンプトが表示されます。どちらかをクリックすると、プラグインが自動的にインストールされます。

このプラグインをダウンロードして手動でインストールする必要がある場合は、次のリンクを使用してください。

WooCommerce.com アップデート マネージャーを使用すると、次のことが可能になります。

- WooCommerce.com から拡張機能とテーマをサイトに自動インストールします。
- 拡張機能のサブスクリプションを管理します。
- 期限切れのサブスクリプションを更新します。
- ドキュメントに移動します。
- 接続されたサブスクリプションのアップデートとサポートを受け取ります。

WooCommerce.com Update Manager に接続して使用すると、**My Subscriptions** タブに現在アクティブなサブスクリプションが表示されます。

隣接するタブ (**検索**、**参照**、**テーマ**) から、新しい拡張機能やテーマを見つけたり、**詳細を表示** して詳細を確認したり、サイトに追加するために購入したりできます。

WooCommerce.com で初めてご購入いただく際、ストアを接続するよう求められます。ストアを WooCommerce.com にリンクすると、[マイストアページ](https://woocommerce.com/my-account/my-stores/) でストアを確認したり、WooCommerce.com アカウントに接続したり、チケット送信時にストアのシステムステータスレポートを自動的にサポートチームに送信したりできます。Woo Marketplace での購入に使用しているアカウントに接続することをお勧めします。

ストアを WooCommerce.com に接続するには:

1. サイトのWP Adminダッシュボードから、**WooCommerce > 拡張機能 > サブスクリプション** に移動します。
2. **接続** ボタンをクリックします。
3. WooCommerce.comアカウントにログインしていない場合は、今すぐログインしてください。
4. **ストアを接続** ボタンをクリックして、サイトをWooCommerce.comアカウントに接続します。

*まだログインしていない場合は、WordPress.com のログイン資格情報を使用して WooCommerce.com にログインします。*

ストアを接続すると、WooCommerce.com のサブスクリプションとアカウント データが同期されます。

サイトが WooCommerce.com アカウントに接続されると、Woo Marketplace からのサブスクリプションを既に所有している場合は、WooCommerce.com Update Manager が自動的にインストールされます。

WooCommerce.com アカウントのサイトへの接続を停止したい場合は、アカウントを切断できます。これにより、接続済みのアカウントを切り替えたり、そのサイトの拡張機能の管理を停止したりできるようになります。

サイトの WP 管理ダッシュボードから、**WooCommerce > 拡張機能 > マイサブスクリプション** に移動し、右上のプロフィールアイコンにマウスを合わせて、**アカウントを切断** をクリックします。

**注**: WooCommerce.com からサイトを切断すると、アカウントで購入した拡張機能の自動更新が無効になります。更新を継続的に利用するために、サイトを WooCommerce.com に接続したままにしておくことをお勧めします。クライアントの代理でサイト管理を手伝っているビルダーの場合は、[サブスクリプションをクライアントの WooCommerce.com アカウントに移管](https://woocommerce.com/document/managing-woocommerce-com-subscriptions/#transfer-a-woocommerce-com-subscription)し、クライアントに [共同編集者](https://woocommerce.com/document/adding-collaborators/) として追加してもらうことができます。

ご注文完了後の最初のステップは、新しい製品をサイトにインストールし、サブスクリプションを有効化することです。ご購入いただいた製品は**自動**または**手動**でインストールできます。自動インストールを推奨しますが、ご希望の場合は製品の`.zip`ファイルをダウンロードして手動でインストールすることも可能です。

WooCommerce.com からの購入を自動的にインストールする方法は複数あります。

1つ目は、WooCommerce.comで注文を完了した直後に表示されます。注文確認ページから購入商品をインストールできます。「**ストアに追加**」ボタンが表示され、まだサイトに接続していない場合はストアのURLを入力するよう求められます。その後、このサイトをWooCommerce.comに接続するように求められます。

接続が完了すると、ご購入いただいたコンテンツがお客様のサイトへインストールされます。今後ご購入いただく際には、接続済みのサイトを選択して自動インストールをご利用いただけます。

2 番目のオプションは、WooCommerce.com の **My Subscriptions** ダッシュボードから実行できます。

1. WooCommerce.com の **My Subscriptions** セクションに移動します。
2. インストールする製品の **Add to store** ボタンをクリックします。
3. 接続されているストアを選択するか、新しいストアを接続します。
4. 選択した製品をインストールして設定します。

最後に、ストア管理の「マイサブスクリプション」ページから自動的にインストールすることもできます。

1. サイトのWP管理ダッシュボードから、**WooCommerce > 拡張機能 > マイサブスクリプション** に移動します。
2. **使用可能** の下で、インストールしたい拡張機能を見つけます。
3. **インストール** ボタンをクリックします。
4. 拡張機能またはテーマが自動的にインストールされます。

購入したものを手動でインストールする場合:

1. インストールしたい拡張機能を、以下のいずれかの場所から.zipファイルをダウンロードします。
- **My Subscriptions** ページで、拡張機能の横にある3つの点をクリックし、**Download extension** を選択します。
- 一括ダウンロードするには、WooCommerce.com アカウントの [ダウンロード](https://woocommerce.com/my-account/downloads/) セクションに移動します。各拡張機能の行にある **Download** テキストをクリックします。
  
2. Safariはデフォルトでダウンロードしたファイルを解凍します。これを防ぎ、WordPressに必要な`.zip`ファイルを保存するには、**Safari > 環境設定 > 一般**に移動し、「ダウンロード後に"安全な"ファイルを開く」のチェックを外してください。
3. `.zip`ファイルをダウンロードしたら、[プラグイン](https://wordpress.org/support/article/managing-plugins/#upload-via-wordpress-admin)または[テーマ](https://wordpress.org/support/article/using-themes/#adding-new-themes-using-the-administration-screens)として製品を適切にインストールしてください。

ストアがすでにWooCommerce.comに接続されている場合は、製品をインストールすると自動的に接続されます。それ以外の場合は、「**接続**」ボタンが表示され、ストアで拡張機能のアップデートとサポートを有効にできます。

WooCommerce.com のマルチストア設定により、ユーザーが複数のストアを接続している場合にサブスクリプションの管理が容易になります。

代理店または開発者が2人の顧客を抱えているとします。代理店は両方の顧客に代わってWooCommerce.comの拡張機能を購入し、代理店アカウントを使用してストアをWooCommerce.comに接続します。

代理店がAutomateWoo拡張機能を購入したと仮定しましょう。拡張機能は両方の顧客のサブスクリプションリストに表示されます。

しかし、場合によっては、選択したストアで特定の拡張機能のみを表示する必要があります。

ユーザーは、[マイアカウントセクション](https://woocommerce.com/my-account/)の**プロフィール**タブの**マルチストア設定**で**「ストアへの割り当て機能」**を有効にすることでこれを実現できます。

**注記：**

**ホワイトリスト モード** を選択すると、ストアに明示的に割り当てられていない限り、接続されたストアでのサブスクリプションが表示されなくなります。

これにより、ユーザーは [My Subscriptions](https://woocommerce.com/my-account/my-subscriptions/) セクションで、選択したサブスクリプションを接続されたストアに割り当てることができます。

サブスクリプションがストアに割り当てられると、接続されたストアの「サブスクリプション」リストに表示されるようになります。これは、**wallaby-of-whippets.jurassic.ninja** の「WooCommerce -> 拡張機能 - サブスクリプション」メニューに表示される内容です。

譲渡とは、拡張機能を新しい所有者に引き渡すことです。所有権と請求の両方が含まれます。具体的な使用例は以下のとおりです。

- 複数の WooCommerce.com アカウント間での購入を統合する。
- サイト/ストアの開発が完了したら、サイトをクライアントに引き渡す。
- 個人、企業、または代理店に代わって購入した拡張機能の所有権を譲渡する。

複数の WooCommerce.com アカウントを結合、統合、または管理する場合、サブスクリプションの転送は便利で必要な手順となることがよくあります。

1. WooCommerce.com アカウントにログインし、[My Subscriptions](https://woocommerce.com/my-account/my-subscriptions/) に移動します。
2. 移行したいサブスクリプションの右側にある3点メニューアイコンをクリックし、メニューから **Transfer subscription** を選択します。
3. 拡張機能を移行する WooCommerce.com アカウントの **メールアドレス** を入力します。
4. **Transfer** ボタンをクリックして、この拡張機能の移行を確定します。これにより、**所有権と請求の両方が新しい所有者に移行** されることにご注意ください。

- **マイアカウント** ページの下部に、移行が開始されたことをお知らせする通知が表示されます。
- サブスクリプションにも「移行中…」というメッセージが表示され、その後に転送先のメールアドレスが表示されます。
- 転送先の方には、**通知メール** が届き、移行の承認を求められます。
- 転送先の方の **マイアカウント** ページにも、移行の通知が表示されます。転送先の方は、ここから移行を承認または拒否できます。
- 転送先の人が移行を承認すると、拡張機能はあなたのアカウントから相手のアカウントに移動されます。

共有機能を使用すると、**他のユーザーに**、購入した拡張機能へのアクセスを許可できます。拡張機能の所有権は保持され、請求は引き続きお客様のものとなります。ストアビルダーの場合は、共有機能により、購入したサブスクリプションをクライアントのサイトで使用できるようになります。ただし、サブスクリプションを共有しても、[利用可能なサイト数](https://woocommerce.com/document/frequently-asked-questions/#how-many-sites-can-i-use-my-subscriptions-on)は変わりません。以下の点にご注意ください。

- 拡張機能の**使用権**のみを共有します。**所有権と請求は元の所有者に残ります**。拡張機能を譲渡したい場合は、[上記の手順](https://woocommerce.com/document/managing-woocommerce-com-subscriptions/#section-6)に従ってください。
- 拡張機能を他のユーザーと共有した場合、そのユーザーは**共有または譲渡できません**。**共有または譲渡できるのは**あなただけです**。

1. WooCommerce.com アカウントにログインし、[My Subscriptions](https://woocommerce.com/my-account/my-subscriptions/) セクションに移動します。
2. 移行したいサブスクリプションのメニューアイコン（縦に3つの点）をクリックし、メニューから **Share subscription** を選択します。
3. 拡張機能を共有する相手の **メールアドレス** を入力（メールアドレスは WooCommerce.com アカウントに登録されている必要があります）し、**Share** ボタンをクリックします。
4. サブスクリプションが共有されたことを確認する通知が表示され、サブスクリプションに共有アイコンが表示されます。
5. 共有された拡張機能を受け取る側の **My Account** サブスクリプションリストに、共有した相手のメールアドレスと共に拡張機能が表示されます。
6. 拡張機能を共有した相手がWooCommerceプラグイン経由でサイトをWooCommerce.comに接続すると、WP AdminダッシュボードのWooCommerce拡張機能エリアにある「**My subscriptions**」タブにサブスクリプションが表示されます。そこからダウンロードしてインストールでき、元のサブスクリプションが有効な限りアップデートをダウンロードできます。

1. 他のユーザーとサブスクリプションを共有している場合は、共有アイコンが表示されます。共有状況を確認したり削除したりするには、サブスクリプションのメニューボタンを選択し、「**共有を管理**」を選択します。
2. サブスクリプションを共有したユーザーのリストが表示されるダイアログが表示されます。共有相手のメールアドレスの横にある「**削除**」のチェックボックスをオンにし、「**保存**」ボタンをクリックします。
3. 共有が削除されたことを確認する通知が表示されます。

WooCommerce.com のサブスクリプションをキャンセルするには、アカウントダッシュボードの「**My Subscriptions**」タブに移動し、アクティブなサブスクリプションの横にある省略記号メニューを使用してください。メニューから「**Cancel subscription**」を選択してください。

キャンセルされた WooCommerce.com サブスクリプションは、有効期限まで引き続き更新とサポートを受けられます。

期限切れのサブスクリプションがあり、ダッシュボードに表示する必要がなくなった場合は、それらをアーカイブして、サブスクリプション リストを整理しておくことができます。

**注:** アーカイブできるのは期限切れのサブスクリプションのみです。アクティブなサブスクリプションはアーカイブできません。

- サブスクリプションはメインのサブスクリプション一覧から削除されます。
- 当該製品のサポートやアップデートはご利用いただけなくなります。
- サブスクリプションライセンスは非アクティブのままです。
- 将来的に製品が必要になった場合は、新規購入が必要となります。

- [WooCommerce.com](https://woocommerce.com/) アカウントにログインし、[マイサブスクリプション](https://woocommerce.com/my-account/my-subscriptions/) セクションに移動します。
- 下にスクロールして「期限切れのサブスクリプション」セクションに移動します。
- アーカイブしたいサブスクリプションを見つけ、右側にある3点メニューアイコン (**⋮**) をクリックします。
- ドロップダウンメニューから「この商品をアーカイブ」を選択します。
- サブスクリプションが期限切れのサブスクリプションリストから削除されます。

アーカイブすると、サブスクリプションはサブスクリプションビューから完全に削除されます。同じ商品に再度アクセスする必要がある場合は、[WooCommerce.com Marketplace](https://woocommerce.com/) から新しいサブスクリプションを購入する必要があります。

自動インストールを有効にするには、ストアでWordPress REST APIを有効にする必要があります。また、サイトが一般公開されており、近日公開ページや認証の影響を受けていない必要があります。

REST APIは**プレーン**パーマリンクをサポートしていません。WooCommerce.comの自動インストール機能を使用するには、**WP Admin > Settings >****Permalinks**で他のパーマリンクオプションを選択してください。

WordPressサイトによっては、REST APIを無効化したり、ログインユーザーのみにアクセスを許可したりすることでアクセスを制限している場合があります。これは通常、プラグインやカスタムコードを介して行われます。

#### サイトのREST APIが動作していることを確認する方法

REST APIをテストするには、`http://yoursite.test/wp-json/`を開きます。`http://yoursite.test/`はサイトのドメインに置き換えてください。開いたリンクにサイト名や説明などが表示されれば、REST APIは動作していることになります。

「*認証済みユーザーのみがREST APIにアクセスできます。*」のようなメッセージが表示される場合、サイトのREST APIがログインユーザーのみに制限されていることを意味します。この問題を解決するには、サイトにログインしてもう一度お試しください。

#### サイトのREST APIを有効にする方法

上記の手順を試しても問題が解決しない場合は、サイトのREST APIが無効になっている可能性があります。サイトのプラグインや拡張機能を確認し、REST APIをブロックするために特別に作成されたものを使用していないことを確認してください。セキュリティプラグインを使用している場合は、設定を確認し、REST APIが有効になっていることを確認してください。

サイト所有者がREST APIを無効にするカスタムコードをサイトに追加している場合があります。このコードは削除または修正し、すべての`/wp-json/wccom-site/`エンドポイントへのREST APIアクセスを許可する必要があります。このコードがどこにあるか分からない場合は、サイトのプラグイン／拡張機能やテーマファイルで、`/wp-json/`から返されるメッセージを探してみてください。

#### REST API アクセシビリティ

サイトを WooCommerce.com に接続し、拡張機能の自動インストールを有効にするには、サイトの REST API にアクセスできることを確認してください。

- WordPress.com でホストされているサイトの場合、サイトが現在「非公開」に設定されている場合、自動インストール機能はご利用いただけませんのでご注意ください。この機能を利用するには、[こちらのガイド](https://wordpress.com/support/settings/privacy-settings/?_gl=1*fe8f68*_ga*MTc2OTE0MDQ3My4xNjg1Njg3ODg2*_ga_98K30SHWB2*MTY5MDE3MTU5MS4yMTIuMS4xNjkwMTczMTg3LjAuMC4w#find-the-privacy-settings) に従って、サイトを「近日公開」に設定してください。
- または、[こちらのガイド](https://woocommerce.com/document/installing-and-activating-woocommerce-extensions/#manual-installation) に従って、WooCommerce.com 拡張機能を手動でインストールすることもできます。

自動インストールは、WordPress cron を使用してプラグイン/拡張機能をインストールします。サイトで cron が有効になっていることを確認してください。**WooCommerce > ステータス > システムステータス > WordPress cron** で cron が有効になっているかどうかを確認できます。cron が無効になっている場合は、WordPress のメインフォルダにある `wp-config.php` ファイルに次の行があるかどうかを確認してください。

`define('DISABLE_WP_CRON', true);`

この行が存在する場合は、削除またはコメントアウトして、サイトでWordPressのcronを有効にしてください。何らかの理由でWordPressのcronを有効化したくない場合は、代わりに実際のcronジョブを使用することもできます。[WordPressのcronを実際のcronジョブに置き換える方法のチュートリアル](https://www.siteground.com/tutorials/wordpress/real-cron-job/)をご確認ください。

サイトとWooCommerce.comの間で問題が発生した場合は、すぐに解決するために2つの方法を試すことができます。まず、接続を更新してみてください。**WP Admin > WooCommerce > Extensions > My Subscriptions** の右上にある**Refresh** ボタンを選択してください。

リフレッシュしても問題が解決しない場合は、サイトをWooCommerce.comアカウントから完全に切断してから再接続してみてください。これにより、サブスクリプションのキャンセルやサポートアクセスの取り消しなどは発生しません。この方法は、ホスト名やドメイン名の変更など、残っている接続の問題を洗い出すのに役立つことが多いです。

WooCommerce.com へのサイト接続を切断してから再接続するには、Gravatar とメール アドレスが表示されているアカウント ドロップダウンを選択し、「アカウントを切断」を選択してから、ポップオーバー確認の「切断」ボタンを選択して選択を確定します。

そこから、指示に従ってストアを WooCommerce.com に再度接続します。

正しい WooCommerce.com アカウントに接続されていることを確認してください。必要に応じて、[接続を解除](https://woocommerce.com/document/installing-and-activating-woocommerce-extensions/#disconnect-your-site-from-your-woocommerce-com-account)し、再接続してください。

**WP Admin > WooCommerce > Extensions > My Subscriptions** にアクセスし、更新しようとしている拡張機能が表示されていることを確認してください。表示されない場合は、正しいアカウントがサイトにリンクされていること、および拡張機能に [接続](https://woocommerce.com/document/installing-and-activating-woocommerce-extensions/#activate-a-subscription-on-your-site) のプロンプトが表示されていないことを確認してください。必要に応じて接続し、サブスクリプションを有効化して更新を行ってください。

以前は5サイトおよび25サイトのサブスクリプションをご提供していました。5サイトまたは25サイトのサブスクリプションを分割してキーを個々のクライアントに譲渡することはできません。キー番号はフルパッケージに帰属するためです。また、1つのサイトで拡張機能の所有権を共有する方法として、2つの異なるWooCommerce.comアカウントを同じサイトにリンクすることもできません。

拡張機能の所有権をクライアントに付与したい場合は、クライアントごとに個別のサブスクリプション／キーを購入するか、クライアント自身のアカウントでキーを購入してもらう必要があります。また、[こちらの手順](https://woocommerce.com/document/installing-and-activating-woocommerce-extensions/#share-a-subscription)の共有機能を使用して、マルチサイトサブスクリプションで単一のキーを共有することも可能です。この場合、所有権はクライアントが保持します。
