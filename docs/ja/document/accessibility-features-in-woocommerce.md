---
title: WooCommerceプラグインのアクセシビリティ機能

---

WooCommerce は、厳格なアクセシビリティ コーディング標準に準拠した WordPress 上に構築されています。

WooCommerceプラグインはこれらの慣行を継承し、スクリーンリーダーやキーボードのみのナビゲーションなどの支援技術との互換性を確保するために独自のアクセシビリティ機能を導入しています。このページでは、プラグインに組み込まれているアクセシビリティ機能について概説します。

**注:** 多くの地域では、eコマースサイトのアクセシビリティが法的に義務付けられています。ストアのアクセシビリティを確保することで、米国の[米国障害者法（ADA）](https://ada.gov/)や、すべてのEU加盟国に適用される指令である[欧州アクセシビリティ法（EAA）](https://employment-social-affairs.ec.europa.eu/policies-and-activities/social-protection-social-inclusion/persons-disabilities/union-equality-strategy-rights-persons-disabilities-2021-2030/european-accessibility-act_en)など、アクセシビリティに関する法律を制定している国における法令遵守につながります。他の国にもアクセシビリティに関する法律があるため、[世界のウェブアクセシビリティに関する法律](https://www.w3.org/WAI/policies/)を確認し、地域の要件に準拠していることを確認することをお勧めします。

これに関するガイダンスについては、[WooCommerce ストアのアクセシビリティのヒント](https://woocommerce.com/document/accessibility-tips-for-store-owners/) のガイドを参照してください。

[アクセシビリティ対応テーマ](https://wordpress.org/themes/tags/accessibility-ready/) を備えた WordPress を使用する場合、WooCommerce はフルキーボードナビゲーションをサポートし、ユーザーはキーボードのみを使用してインターフェース内を移動できます。

- チェックアウト、カート、フォーム全体で論理的なタブ順序を採用しています。
- 選択中の要素を明確に示すフォーカスインジケーターを備えています。
- すべてのインタラクティブコンポーネント（ボタン、ドロップダウン、トグル）はマウスなしでアクセスできます。
- ボタンはスペースキーとEnterキーの両方で操作でき、リンクはEnterキーで起動します。ダイアログとモーダルは、開いている間もキーボードフォーカスを適切に保持します。

テーマ スタイルは WooCommerce によって設定された内容を上書きする可能性があるため、サイトで使用しているテーマがアクセシビリティをサポートしていることも確認する必要があります。

WooCommerce は、スクリーン リーダー ユーザーがサイト コンテンツをナビゲートして理解できるように、セマンティック HTML と ARIA ロールを使用します。

- フォーム、ボタン、インタラクティブな要素には明確なラベルが付けられています。
- 動的な更新（カートの合計金額、検証メッセージなど）では、ARIA ライブリージョンを使用してアナウンスが行われます。
- ページ構造では、見出し、リスト、ランドマークを使用してコンテンツの階層構造を伝えます。
- 要素には適切な役割が割り当てられています。ボタンはボタンとして、リンクはリンクとしてアナウンスされ、フォームフィールドには「押された」や「展開された」などの適切なラベルと状態が含まれています。

特にチェックアウト時のフォームは、アクセシビリティを考慮して設計されています。

- すべての入力項目には説明的な<label>が関連付けられています。
- 必須項目は明確にマークされており、エラーは視覚的にもプログラム的にも通知されます。テーマがこの機能をサポートしているかどうかもご確認ください。
- 説明とフィールドグループはスクリーンリーダーに対応しています。

WooCommerceのチェックアウトエクスペリエンスを変更するテーマやサードパーティ製プラグインは、アクセシビリティを損なう可能性がありますのでご注意ください。チェックアウトページを変更する場合は、表示されているラベル、必須インジケーター、エラーメッセージが表示されていることを確認してください。

WooCommerce プラグインは、アクセシビリティを考慮してフォーム エラーを処理します。

- エラーは関連するフィールドの近くに表示されます。
- ARIA 属性は、エラーメッセージと入力内容をリンクします。
- リアルタイム検証により、ユーザーはフォームを再送信することなく問題を修正できます。
- 成功メッセージとエラーメッセージは、ARIA ライブリージョンを使用してスクリーンリーダーでも読み上げられます。

ARIA ロールはスクリーン リーダー ユーザーのエクスペリエンスを強化します。

- プラグインによって制御されるレイアウト領域には、メイン、ナビゲーション、リージョンなどの役割が付与され、固有の名前が付けられます。
- 折りたたみ可能なメニューやモーダルなどのインタラクティブ要素は、ARIA の状態（例：aria-expanded）を使用します。
- ライブリージョンは、コンテンツが動的に変化したときにユーザーに通知します。

商品リスト、カート、チェックアウトで使用される WooCommerce ブロックは、アクセシビリティを考慮して構築されています。

- HTML5 セマンティック要素の使用。
- キーボードとスクリーンリーダーに完全対応。
- WordPress アクセシビリティガイドラインに準拠して構築。

*注: 一部の古い非推奨ブロックはアクセシビリティのサポートが制限されており、テストや追加の設定が必要になる場合があります。アクセシビリティと機能の向上のため、新しいブロックが利用可能になったらすぐに更新し、非推奨ブロックの使用を避けることをお勧めします。*

- **ズームサポート**: WooCommerce はブラウザのズーム設定を最大 400% まで対応しています。コンテンツは読みやすく、インタラクティブなまま、重なり合ったり機能の損失が発生したりすることはありません。
- **モーション感度**: アニメーションは、該当する場合、ユーザーのオペレーティングシステムの prefers-reduced-motion 設定に従います。
- **アクセシブルなデフォルト**: すぐに使用できる設定は、WCAG レベル AA の色コントラストとユーザビリティ要件を満たすように設計されています。
- **スクリーンリーダーテスト**: WooCommerce は、VoiceOver (macOS)、JAWS (Windows)、NVDA (Windows) などのスクリーンリーダーを使用してテストされています。

WooCommerce のアクセシビリティに関するフィードバックをお待ちしております。問題が発生したり、ご提案がございましたら、[サポートにお問い合わせ](https://woocommerce.com/contact-us/)いただくか、[accessibility@woocommerce.com](mailto:accessibility@woocommerce.com) までメールでお問い合わせください。アクセシビリティは継続的な優先事項であり、皆様からのご意見は、すべてのお客様により良いエクスペリエンスを提供するために役立てさせていただきます。
