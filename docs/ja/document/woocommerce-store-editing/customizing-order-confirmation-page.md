購入者が注文後に表示される「ありがとうございます」ページ、別名「注文確認ページ」は、購入後に注文内容を確認することで顧客に安心感を与えます。ここでは、WooCommerceがブロックテーマでこのページを表示するために使用するテンプレートと、WooCommerce注文確認テンプレートを構成するブロックの構造について解説します。

このページは注文が完了した後にのみ表示されます。このページを離れた後でも、購入者はアカウントエリアの「注文」セクションからこれらの詳細を再度確認できます。

通常、注文確認ページは、サイトのテーマに合わせてスタイルを少し変更する程度で、編集する必要はありません。すべてのコンテンツは注文ごとに動的に更新されます。

注文確認ページは、**注文確認**テンプレートを編集することでカスタマイズできます。注文確認ページの各セクションにラベルを付ける見出しブロックを除き、注文確認テンプレートに含まれるすべてのブロックは動的に変化するため、各注文に合わせて内容が調整されます。

このテンプレートのセクションのいずれかが現在の注文に該当しない場合、そのセクションは表示されません。例えば、顧客がダウンロード可能な製品を購入していない場合、ダウンロード可能な製品の詳細が記載された「ダウンロード」セクションは表示されません。

注文確認テンプレートでは、顧客の注文確定後にその詳細を表示するために、様々な動的ブロックを使用しています。これらのブロックの内容は動的であり、現在の注文内容に基づいて変化するため、設定は最小限に抑えられており、ブロック自体の内容をエディターで変更することはできません。

**アカウント作成の遅延** が有効で、現在の顧客がゲストの場合、アカウント作成ブロックが自動的に表示されます。これにより、顧客はチェックアウト時に入力した情報を使用してアカウントを作成できます。アカウント情報は顧客にメールで送信され、新しいアカウントは注文にリンクされます。

「特典」の内容はカスタマイズ可能です。「設定」＞「アカウントとプライバシー」の設定によっては、パスワード入力欄が表示される場合があります。

### 注文状況

注文状況ブロックには、現在の注文状況に基づいてメッセージが動的に表示されます。ほとんどの場合、「ありがとうございます。ご注文を承りました」のようなメッセージが表示されます。このメッセージは現在編集できません。

注文ステータス ブロックには、ブロックの表示を調整するための色、書体、寸法の設定があります。

注文概要ブロックには、**注文番号、日付、合計、顧客のメールアドレス、**支払い方法** が表示される注文概要が表示されます。

注文概要ブロックには、ブロックの表示を調整するための色、タイポグラフィ、サイズの設定があります。さらに、選択した色と太さでセクションに境界線を追加できる境界線設定もあります。

注文詳細セクションには、すべての明細項目、製品数量、注文合計を含む注文の詳細が表示されます。セクションヘッダーはカスタマイズ可能です。

ダウンロードセクションは、ダウンロード可能な製品をご購入いただいた場合にのみ表示されます。ここでは、ご注文いただいた商品からダウンロード可能な商品、ダウンロード制限や有効期限、そしてダウンロードにアクセスするためのリンクを確認できます。

このセクションには、チェックアウト時に入力された配送先住所と請求先住所の情報が表示される2つの列があります。店舗受け取りを選択されたご注文には、配送先住所は表示されません。

各住所セクションには、セクションの周囲のパディングと余白を調整するための寸法設定があります。請求先住所と配送先住所のブロックには、各情報ブロックの周囲の境界線を調整するための設定があります。各セクションの色とタイポグラフィの設定は、親列のブロック設定で調整できます。

追加フィールド ブロックには、「連絡先」および「注文」の場所からの追加のチェックアウト フィールドの情報が表示されます。

追加フィールドブロックには、互換性のある拡張機能または決済ゲートウェイによって提供される、現在の注文に関する追加情報が表示されます。例えば、組み込みのBACS決済ゲートウェイを使用する場合、購入者が支払いを行うための銀行口座情報がここに表示されます。

新しいカスタマイズ可能なブロックテンプレートではなく、従来の注文確認テンプレートをご利用になりたい場合は、すべての注文確認ブロックを含む親グループを削除し、従来のテンプレートのプレースホルダーとして機能する「注文確認ブロック」を挿入してください。切り替え手順は以下のとおりです。

**注:***これらの手順はブロック テーマにのみ適用されます。*

1. サイトエディターで注文確認テンプレートを開きます。**「外観 > エディター > テンプレート > 注文確認」** に移動します。
2. リストビューを開き、すべての特別な注文確認ブロックを削除します（リストビューでブロックの範囲を選択するには、Shiftキーを押しながら選択範囲の最後のブロックをクリックします）。
3. ブロック挿入ツールを開き、「注文確認ブロック」を検索して追加します。上記のように、クラシックテンプレートのプレースホルダー通知が表示されます。
4. 変更を保存します。

これで、サイトではクラシックテーマと同じように、レガシーテンプレートを使用した注文確認ページが表示されます。そのテンプレート上のカスタムコードも使用可能になります。
