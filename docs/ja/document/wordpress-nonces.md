「nonce」とは「**一度だけ使用される番号**」の略です。これは、特定のリクエストや送信が悪意を持って再利用されることを防ぐためのセキュリティ対策です。

このワンタイム番号は、各トランザクションまたはセッションが一意であり、一時的に有効であることを確認することで、偽造などのさまざまな種類の攻撃から Web サイトを保護するのに役立ちます。

**注:**このドキュメントは、高度なトラブルシューティングに役立つガイドとして提供されています。ただし、記載されている手順は[サポートポリシー](https://woocommerce.com/support-policy/)の対象外であり、実装に関する直接的なサポートは提供できません。資格のあるWordPress/WooCommerce開発者のサポートをご希望の場合は、[Codeable](https://codeable.io/?ref=z4Hnp)または[Woo Agency Partner](https://woocommerce.com/development-services/)を強くお勧めします。

ナンスはWooCommerceサイトの安全を守るのに役立ちます。これらの特別なセキュリティコードは、サイト上での操作が意図した人物によって行われたことを証明します。これにより、誰かがその人物になりすますことを防ぎます。これは特に、攻撃者がシステムを騙して有害な操作を実行しようとする不正な操作（クロスサイトリクエストフォージェリ（CSRF）攻撃など）を防ぐ上で重要です。

ノンスを使用することで、サイト上のフォームやリンクを保護できます。つまり、サイトは正当な送信やクリックのみを処理することになります。これにより、サイトの機能を悪用したり、有害な方法でアクセスしたりするのを防ぐことができます。この簡単な手順で強力なセキュリティレイヤーを追加し、サイトとユーザーの両方を安全に保つことができます。

nonceはWordPressの組み込み機能です。WooCommerceはセキュリティ強化のためにこれを広く利用しています。WordPressコアの一部として、nonce機能はプラグインとテーマにこれらのセキュリティトークンを生成および検証する方法を提供します。

これにより、フォームの送信、データの変更、その他の機密性の高い操作などのアクションを安全に検証できます。WooCommerceはノンスを使用することで、これらのアクションが本物であることを確認します。これにより、不正アクセスを防ぎ、サイト上のeコマース取引の安全性を確保できます。

WooCommerceサイトにノンスの問題がないか確認するには、具体的な症状を探し、特定のアクションをテストします。確認する簡単な方法は次のとおりです。

- **フォーム送信エラー**: フォームを送信しようとした際にエラーメッセージ、特にセキュリティや無効なリクエストに関するメッセージが表示される場合は、ノンスの問題が疑われます。
- **ユーザー操作中の予期しない動作**: セキュリティチェックを必要とする操作（ログインやユーザー情報の変更など）が予期しない動作をする場合。
- **一貫性の確認**: ノンスの問題は、多くの場合、一貫したパターンを示します。たとえば、ページを長時間開いた後にのみエラーが発生する場合は、ノンスの有効期限が切れている可能性があります。
- **別のブラウザでのテスト**: 別のブラウザでサイトにアクセスしてみるか、ブラウザのキャッシュをクリアしてからアクセスしてみてください。キャッシュをクリアしたブラウザで問題が発生しない場合は、ノンスの処理に問題がある可能性があります。
- **コンソールエラー**: ブラウザの開発者ツールを使用して、ノンスチェックをトリガーする操作を実行中にコンソールにエラーがないか確認してください。リクエストの失敗やセキュリティ警告に関するメッセージは、ノンスの問題を示している可能性があります。
- **プラグインの相互作用**: 他のプラグインを一時的に無効にして、再度テストしてください。問題が解決した場合は、他のプラグインの nonce の処理方法との[競合](https://woocommerce.com/document/how-to-test-for-conflicts/)が原因である可能性があります。
