支払いエラーのトラブルシューティングは、使用している支払いゲートウェイとその設定方法によって異なり、サイトや顧客に固有のものである可能性もあります。

WooCommerce には、数多くの [組み込み支払いオプション](https://href.li/?https://docs.woocommerce.com/documentation/plugins/woocommerce/getting-started/sell-products/core-payment-options/) が付属しており、[WooCommerce.com マーケットプレイスで利用可能な支払いゲートウェイ](https://href.li/?https://woocommerce.com/product-category/woocommerce-extensions/payment-gateways/) も確認できます。

エラーが発生した場合は、次の一般的なガイドラインが役立ちます。

- **400 エラー**: これは通常、連絡先 URL が間違っている場合に発生します。
- **200 エラー**: これは通常、認証情報が正しくない場合に発生します。
- **空白ページ エラー**: これは多くの場合、PHP の問題を示しています。根本原因を特定するには、デバッグを有効にしてください。[WordPress のデバッグ方法](https://wordpress.org/support/article/debugging-in-wordpress/)。

支払いエラーのトラブルシューティングを行うために、WooCommerce サイトに別のテスト環境を設定する場合は、[ステージング サイトを設定することをお勧めします](https://woocommerce.com/posts/what-is-staging-site-wordpress-how-to-set-one-up/)。

多くの決済ゲートウェイは、機能をテストするためのテストモードまたはサンドボックスモードも提供しています。これを有効化する具体的な手順については、ご利用の決済ゲートウェイのドキュメントをご覧ください。

注文ステータスは、注文が支払いおよび履行プロセスのどの段階にあるかを示します。内訳は以下のとおりです。

- **保留中**: 支払いの確認待ちです（例：銀行振込や小切手などの手動支払い方法の場合）。ゲートウェイがキャプチャではなく承認に設定されている場合、このステータスには手動による対応が必要です。
- **保留中**: 注文は受領されましたが、支払いが未完了です。これは通常、通知URLまたはプラグインの競合に問題があることを示しています。
- **処理中**: 支払いは受領され、注文は発送を待っています。このステータスは通常、物理的な商品に適用されます。
- **完了**: 注文は正常に処理されました。
- **失敗**: 支払いが失敗または拒否されました。一般的な理由としては、支払い期限の超過や注文の放棄などが挙げられます。具体的なエラー情報については、注文メモを参照してください。

注文管理の詳細については、[注文管理](https://href.li/?https://docs.woocommerce.com/document/managing-orders/)をご覧ください。

注文メモには、取引中に何が起こったかの概要が記載されています。さらに詳しく確認するには、決済プラグインで取引ログを有効にしてください。

1. **WooCommerce > 設定 > 支払い > [プラグイン]** に移動します。
2. ログ記録を有効にします。
3. **WooCommerce > ステータス > ログ** からログにアクセスします。

ログは長くなり、時間の経過とともに増え続ける可能性があることにご注意ください。**トラブルシューティングの場合を除き、**ログは無効にしておくことをお勧めします。

プラグインやテーマとの競合により、次のような問題が発生する可能性があります。

- チェックアウト時に支払いオプションが欠落しているか、正しく表示されない
- 顧客を正しい確認ページにリダイレクトできない
- ゲートウェイからのAPIレスポンスがブロックされる

問題が発生した場合は、次の点を検討してください。

- 古くなったプラグインやテーマがないか確認する
- [競合のテスト](https://woocommerce.com/document/how-to-test-for-conflicts/)：WooCommerceと決済ゲートウェイ以外のすべてのプラグインを無効にし、その後、1つずつ再度有効化する。

競合は通常、API認証情報には影響しません。「認証情報が正しくありません」などのエラーが発生した場合は、決済代行業者に認証情報が正しいことを確認してください。

[WooCommerce システムステータスレポート](https://woocommerce.com/document/understanding-the-woocommerce-system-status-report/) は、潜在的な問題に関する洞察を提供します。確認すべき主な項目は次のとおりです。

- サイトでは最新バージョンの WordPress と WooCommerce が稼働していますか？
- PHP は最新ですか？古いバージョンは互換性の問題を引き起こす可能性があります。
- SOAP は有効ですか？一部のゲートウェイでは必須です。
- WooCommerce ページ（チェックアウトなど）は適切に設定されていますか？

お支払い通知により、ショップは取引状況の最新情報を受け取ることができます。通知内容はゲートウェイによって異なります。

- [Stripe](https://woocommerce.com/products/stripe/): 通知を自動処理します。設定は不要です。
- **PayPal**: 設定によっては手動設定が必要な場合があります。
- **Authorize.Net**: 多くの場合、手動設定が必要です。

通知が機能しない場合は、特定のゲートウェイの通知 URL または構成を再確認してください。

これらのトラブルシューティング手順を理解することで、問題を迅速に解決し、顧客にとってスムーズなチェックアウト体験を維持できるようになります。

まだ質問があり、サポートが必要ですか?

- ヘルプデスクからハピネスエンジニアに[お問い合わせ](https://woocommerce.com/my-account/create-a-ticket/)ください。WooCommerce.comで開発または販売されている拡張機能、およびJetpack/WordPress.comをご利用のお客様には、サポートを提供しています。
- お客様でない場合は、[WooCommerceサポートフォーラム](https://wordpress.org/support/plugin/woocommerce/)でサポートを探すか、Wooエージェンシーパートナーにご依頼いただくことをお勧めします。これらのエージェンシーは、高度にカスタマイズされ、拡張性の高いオンラインストアの構築で実績のある信頼できるエージェンシーです。[Wooエージェンシーパートナーの詳細](https://woocommerce.com/development-services/)
