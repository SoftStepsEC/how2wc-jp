*最終更新日：2025年7月18日*

このガイドは、WooCommerce の組み込み配送機能に関する問題の解決に役立つ情報を提供します。以下の手順に従って、料金の誤り、配送方法の不足、予期しないチェックアウト動作などの問題を解決してください。

このガイドでは、WooCommerce コアに含まれる配送オプションに関する問題に焦点を当てています。

- [定額配送](https://woocommerce.com/document/flat-rate-shipping/) – 商品、クラス、注文ごとの標準配送料
- [送料無料](https://woocommerce.com/document/free-shipping/) – オプション条件付きで送料無料
- [店舗受取](https://woocommerce.com/document/local-pickup/) – お客様が店舗で商品を受け取る

このガイドでは、他の配送プラグインのサポートや、コア機能を超えたより高度な構成を見つける場所についても説明します。

**注:** 組み込みの WooCommerce 配送機能には、ライブ料金やラベル印刷は含まれません。

- **WooCommerce Shipping** は、WooCommerce ダッシュボードから USPS、UPS、DHL のラベルを印刷し、送料を節約できる無料の拡張機能です。
- **Live rates** を利用するには、チェックアウト時にリアルタイムの料金を表示するには、USPS、UPS、FedEx などの高度な拡張機能が必要です。

複雑なシナリオや商品には、より高度な配送拡張機能をご利用いただけます。下記の[コア配送制限事項](https://woocommerce.com/document/troubleshooting-core-shipping/#core-shipping-limitations-and-alternatives)をご覧ください。

配送に関する問題は、通常、次のいずれかのカテゴリに分類されます。

- 配送料が不足しているか、間違っている
- チェックアウト時に配送方法が表示されない
- 予期しない配送料
- 配送地域と顧客の住所が一致していない
- 配送税が適用されていない、または間違っている

以下の手順に従って、既知のバグや修正済みのバグが既に修正されているか、また設定が正しいかを確認してください。これは、ハピネスエンジニアが問題を調査する際に使用する手順と同じです。

WordPress、WooCommerce、現在のテーマ、そしてすべての拡張機能が最新バージョンであることを確認してください。アップデートには、既知の問題の修正が含まれていることがよくあります。

**アップデートを確認する方法:**

1. WordPress管理画面で**ダッシュボード > アップデート**に移動します。
2. WordPressコア、テーマ、プラグインのアップデートが利用可能かどうかを確認します。
3. トラブルシューティングに進む前に、すべてのコンポーネントを更新します。

#### 配送ゾーン

1. **WooCommerce > 設定 > 配送 > 配送ゾーン** に移動します。
2. 顧客の住所が[設定された配送ゾーン](https://woocommerce.com/document/setting-up-shipping-zones/)のいずれかと一致していることを確認します。
3. 該当する各配送ゾーンに、少なくとも1つの有効な配送方法が設定されていることを確認します。

#### 配送方法

1. 各ゾーンで、[適切な配送方法](https://woocommerce.com/document/setting-up-shipping-zones/#adding-shipping-methods-to-zones)が有効になっていることを確認してください（例：定額配送、送料無料、店舗受取）。
2. 配送方法に適用されている条件を再確認してください。
3. 送料無料の場合、最低注文金額またはクーポンが正しく設定されていることを確認してください。

#### 商品の配送設定

1. 商品エディタの商品データパネルで、[配送設定の管理](https://woocommerce.com/document/product-shipping-classes/#section-3)オプションを選択します。
2. 商品に配送区分が設定されていることを確認します（設定で必要な場合）。
3. [商品タイプ](https://woocommerce.com/document/managing-products/#virtual-and-downloadable-products)が「Virtual」に設定されていないことを確認します。実体のない商品には配送は適用されません。

WooCommerceには、定額配送、送料無料、店舗受取といった主要な配送方法に対応したデバッグモードが組み込まれています。このモードを有効にすると、カートページに直接情報が記録され、顧客の住所とどの配送ゾーンが一致しているかを確認できます。

**配送デバッグモードを有効にするには:**

1. **WooCommerce > 設定 > 配送 > 配送設定** に移動します。
2. デバッグモードで、**デバッグモードを有効にする** ボックスにチェックを入れます。
3. 変更を保存します。
4. カートまたはチェックアウトページを再読み込みし、利用可能な配送オプションの下にデバッグメッセージが表示されます。

**重要:** デバッグ モードは顧客のフロントエンドに表示されるため、テスト後はデバッグ モードを必ず無効にしてください。

場合によっては、プラグイン、テーマビルダー、またはカスタムモジュールを使用して構築されたカスタムカートページとチェックアウトページが予期しない動作を引き起こしたり、特定の機能が動作しなくなったりすることがあります。

これが設定と競合していないことを確認するための簡単なテストは、デフォルトの WordPress エディターと、カートおよびチェックアウトのショートコードまたはブロックを使用して、デフォルトのカートとチェックアウトのページで試してみることです。

**利用可能なオプション:**

- [カートショートコード](https://woocommerce.com/document/woocommerce-shortcodes/page-shortcodes/#cart): `[woocommerce_cart]`
- [カートブロック](https://woocommerce.com/document/woocommerce-store-editing/customizing-cart-and-checkout/#using-the-cart-and-checkout-blocks) (ブロックエディターで利用可能)
- [チェックアウトショートコード](https://woocommerce.com/document/woocommerce-shortcodes/page-shortcodes/#checkout): `[woocommerce_checkout]`
- [チェックアウトブロック](https://woocommerce.com/document/woocommerce-store-editing/customizing-cart-and-checkout/#using-the-cart-and-checkout-blocks) (ブロックエディターで利用可能)

詳細については、[WooCommerce ページ設定ドキュメント](https://woocommerce.com/document/woocommerce-pages/)を参照してください。

## 高度なトラブルシューティング手順

以下は、より複雑な配送問題のトラブルシューティングに役立つツールとテクニックです。これらの方法を使用して、配送ロジックを検査し、テーマやプラグインの競合を特定し、隠れたエラーを見つけることができます。

ブラウザ開発者ツール (Chrome DevTools や Firefox 開発者ツールなど) を使用すると、カート ページやチェックアウト ページの動的な動作を検査できます。

**一般的なブラウザで開発者ツールにアクセスする方法:**

- **Chrome:** F12キーを押すか、右クリックして「検証」を選択します
- **Firefox:** F12キーを押すか、右クリックして「要素を検証」を選択します
- **Safari:** Safariの環境設定で「開発者」メニューを有効にし、Option+Command+Iキーを押します
- **Edge:** F12キーを押すか、右クリックして「検証」を選択します

**確認すべき重要なタブ:**

#### コンソールタブ

配送方法の表示や更新を妨げる可能性のある JavaScript エラーや警告を探します。

#### ネットワークタブ

`?wc-ajax=update_shipping_method` や `?wc-ajax=update_order_review` などの AJAX リクエストを監視します。これにより、カートの商品や住所が変更されたときに料金の再計算や配送方法の更新が行われているかどうか、また何らかの理由で AJAX リクエストがブロックされていないかどうかを確認できます。

#### 要素タブ

HTML 構造をチェックして、配送オプションがレンダリングされており、テーマの CSS または JavaScript によって非表示になっていないことを確認します。

**これらのツールを使用して、次のような問題を診断します。**

- 数量変更後に配送情報が更新されない
- カスタムチェックアウトの変更により配送情報の入力欄がブロックまたは非表示になる
- JavaScript エラーによりフォームの送信や料金の表示ができない

WooCommerce には、WP Admin の **WooCommerce > ステータス > ログ** からアクセスできるログ システムがあり、PHP の致命的なエラーなどの情報が記録されます。

このログは、サーバー上のPHPエラーログに通常含まれる詳細情報を反映した、貴重な初期リソースとして機能します。さらなるトラブルシューティングのためにサーバー上のPHPログを探す前に、まずこのログを参照することをお勧めします。

[PHP エラー ログの検索](https://woocommerce.com/document/finding-php-error-logs/) について詳しくは、こちらをご覧ください。

WooCommerce で配送オプションの表示に問題が生じる原因として、テンプレートのオーバーライドがよく挙げられます。テーマが WooCommerce のコアテンプレートをカスタマイズしている場合、古いオーバーライドや不完全なオーバーライドによって、メソッドの欠落、配送情報の誤り、チェックアウト時のエラーなどが発生する可能性があります。

#### よくある問題とトラブルシューティング

サイトに競合が発生していると思われる場合は、[競合テストガイド](https://woocommerce.com/document/how-to-test-for-conflicts/)に従ってください。このガイドでは、プラグイン、テーマ、またはカスタムコードが問題の原因となっているかどうかを最も安全に特定する方法と、変更を加える前に考慮すべきベストプラクティスについて説明しています。

競合テストでも問題が解決しない場合は、[サポート リクエストを開いて](https://woocommerce.com/my-account/create-a-ticket/)、修正すべき問題を特定できるようにしてください。

**WooCommerce バージョン 9.9 の新機能:** [Blueprints](https://woocommerce.com/document/woocommerce-blueprints/) は、ストア設定をインポートおよびエクスポートするためのツールです。開発者、代理店、販売者は、ストア設定を再利用・共有することで時間を節約し、セットアップを迅速化し、トラブルシューティングを容易にすることができます。

**ブループリントの場所:** ブループリントは**WooCommerce > 設定 > 詳細設定 > ブループリント**にあります。現在、WooCommerceのコア設定をサポートしており、管理インターフェースまたはWP-CLIから使用できます。

**トラブルシューティングにブループリントを使用する方法:**

ブループリントを使用すると、トラブルシューティングのためにストア設定（配送設定を含む）をエクスポートできます。

上記の手順を実行しても配送に関する問題が引き続き発生する場合:

1. Blueprints を使用して [設定をエクスポート](https://woocommerce.com/document/woocommerce-blueprints/#section-3) します。
2. WooCommerce を有効化（Blueprints も有効化）した状態で、[Storefront](https://wordpress.org/themes/storefront/) や [Twenty Twenty-F​​ive](https://wordpress.org/themes/twentytwentyfive/) などのデフォルトテーマを使用してステージングサイトを構築します。
3. Blueprints を使用してステージングサイトに [設定をインポート](https://woocommerce.com/document/woocommerce-blueprints/#section-7) します。

ステージングサイトで問題が発生しない場合は、WooCommerceの設定の問題ではなく、ライブサイトでの競合を示している可能性があります。

このプロセスにより、ストアの環境を再現して、より正確なテストとトラブルシューティングを行うことができます。

**重要:** 以下のいずれかの問題について、指定された手順で問題が解決しない場合は、まず [基本的なトラブルシューティング手順](https://woocommerce.com/document/troubleshooting-core-shipping/#basic-troubleshooting-steps) と [高度なトラブルシューティング手順](https://woocommerce.com/document/troubleshooting-core-shipping/#advanced-troubleshooting-steps) を必ず実行してください。

これは、配送ゾーンの設定ミスに関連する一般的な問題ですが、[テーマまたはプラグインの競合](https://woocommerce.com/document/how-to-test-for-conflicts/)によってもこのエラーが発生する場合があります (例: 一部の国または地域への配送を制限するサードパーティのプラグイン)。

**顧客に表示されるエラー メッセージ:** `エラー:「この住所では配送オプションを利用できません。住所が正しいことを確認するか、別の住所をお試しください。」`

**この問題のトラブルシューティングを行うには:**

1. **配送ゾーンを確認する:** 顧客の配送先住所が配送ゾーンと一致していることを確認します。設定されている配送ゾーンは、**WooCommerce > 設定 > 配送 > 配送ゾーン** で確認できます。
2. **配送方法を確認する:** 配送ゾーンを特定し、顧客の住所がいずれかのゾーンと一致していることを確認したら、一致した配送ゾーンで少なくとも1つの[配送方法](https://woocommerce.com/document/setting-up-shipping-zones/#adding-shipping-methods-to-zones)が有効になっていることを確認してください。
3. **郵便番号範囲を確認する:** 配送ゾーン設定ページで、重複または不完全な郵便番号範囲（特に[ワイルドカードまたはスペース](https://woocommerce.com/document/setting-up-shipping-zones/#adding-a-new-zone)）がないか確認してください。
4. **商品設定を確認する:** 商品編集ページで、商品が「仮想」としてマークされていないこと、重量、寸法、配送区分が追加されていることを確認してください。これらは配送方法の条件に一致している必要があります（配送方法によって異なります）。 （サイトで使用されている）
5. **デバッグモードを有効にする:** [配送デバッグモード](https://woocommerce.com/?post_type=documentation&p=18734005250648&preview=true#enable-shipping-debug-mode) を有効にすると、役立つエラーメッセージが表示されます。

配送料に課税されるべきではない税金が表示されている場合は、次の手順に従って配送料の課税を無効にしてください。

1. **WooCommerce > 設定 > 配送 > 配送地域 > 配送方法の編集** に移動します。
2. 配送方法の **税金ステータス** が **なし** に設定されていることを確認します（配送料に対して税金を徴収する必要がない場合）。

### 配送に税金はかかりません

チェックアウト時に送料に税金が適用されない場合は、税金設定の誤りが原因であることが多いです。こちらの[送料の税金設定に関するトラブルシューティングガイド](https://woocommerce.com/document/setting-up-taxes-in-woocommerce/)をご確認ください。

予期しない定額配送の動作は、多くの場合、1 つ以上の設定の問題によって発生します。

**レート計算設定を確認してください:**

- 送料が商品ごと、クラスごと、注文ごとに適用されるかを確認してください。
- 商品ごとの価格に掛けるには、`[数量]` を使用してください。
- パーセンテージベースの送料には、`[料金]` を使用してください。
- [高度な定額送料の計算式の使用方法の例](https://woocommerce.com/document/flat-rate-shipping/) をご覧ください。

**配送クラスの設定を確認してください:**

- 配送方法設定で配送クラス料金が定義されていることを確認してください。
- 商品が正しい配送クラスに割り当てられていることを確認してください。
- クラス料金と基本料金を加算して計算する場合は、[計算タイプが適切に設定されている](https://woocommerce.com/document/flat-rate-shipping/#shipping-classes) ことを確認してください。
- 動作に一貫性がない場合は、カートに商品を1つだけ入れてテストし、原因を絞り込んでみてください。

WooCommerce のデフォルトでは、顧客とカートの内容に一致するすべての配送方法が表示されます。つまり、定額配送やその他の配送方法に加えて、送料無料も表示されます。

**WooCommerce 9.9 以降:** 送料無料が利用可能な場合に他のすべての配送方法を非表示にする組み込み設定があります。

1. **WooCommerce のバージョンを確認してください:** バージョンは **WooCommerce > ステータス** で確認できます。
2. **WooCommerce 9.9 以降をご利用の場合:** **WooCommerce > 設定 > 配送 > 配送オプション** に移動し、**送料無料が利用可能な場合は送料を非表示にする** を有効にしてください。
3. WooCommerce の以前のバージョンをご利用の場合: 以下の条件を満たしていない限り、送料無料は他の配送方法を自動的に非表示にしません。
- [カスタムコードスニペット](https://woocommerce.com/document/hide-other-shipping-methods-when-free-shipping-is-available/#php-snippets) が正しく追加され、サイトで有効になっていること
- または、この機能を提供する [サードパーティ製プラグイン](https://woocommerce.com/document/hide-other-shipping-methods-when-free-shipping-is-available/#use-a-plugin) がインストールされ、有効になっていること
  

WooCommerce のデフォルトでは、顧客とカートの内容に一致するすべての配送方法が表示されます。つまり、[定額配送](https://woocommerce.com/document/flat-rate-shipping/) やその他の [配送方法](https://woocommerce.com/product-category/woocommerce-extensions/shipping-methods) に加えて、[送料無料] も表示されます。

[送料無料](https://woocommerce.com/document/free-shipping/) がチェックアウト時に見つからない場合、またはカートが設定された条件を満たしているときに表示されない場合は、以下を確認してください。

- **配送エリアの確認:** 顧客の配送先住所が設定された配送エリアと一致していること、および該当する配送エリアに正しい地域が含まれていることを確認してください。
- **送料無料方法の設定の確認:** 送料無料方法が追加されていること、および意図した「[送料無料には次の条件](https://woocommerce.com/document/free-shipping/#section-2)」ドロップダウンオプションが選択されていることを確認してください。
- **商品の配送クラスの確認:** 商品が送料無料を除外するクラス、または異なる送料を適用するクラスに割り当てられていないことを確認してください。

チェックアウト時に表示される配送料が顧客の所在地に対して設定した料金と一致しない場合は、次の設定を確認してください。

1. **顧客住所の確認:** チェックアウトページで、国/州/市/郵便番号を含む顧客の配送先住所がすべて入力されていることを確認してください。
2. **ストア住所の確認:** **WooCommerce > 設定 > 一般** でストア住所が入力されていることを確認してください。
3. **配送デバッグモードの使用:** このオプションは、**WooCommerce > 設定 > 配送 > 配送設定** で有効にできます。このオプションを有効にしたら、チェックアウトページ（チェックアウトショートコードを使用）で配送ゾーンが一致していることを確認してください。
4. **配送ゾーン設定の確認:** **WooCommerce > 設定 > 配送 > 配送ゾーン** に移動して、郵便番号範囲の重複や配送ゾーンの順序付けの問題がないか確認してください。
5. **地域コードの確認:** ゾーン内の国名と州の略語に2文字のISOコードが使用されていることを確認してください。
6. **プラグインの競合テスト:** プラグインが配送料を動的に調整する場合（例: [条件付き配送と支払い](https://woocommerce.com/products/conditional-shipping-and-payments/)）、プラグインを無効にして動作が継続するかどうかを確認し、ロジックルールをテストしてください。

### カートやチェックアウトで送料が更新されない

配送料に商品、住所、数量の変更が反映されない場合は、以下を確認してください。

#### 配送ゾーンの設定を確認する

**WooCommerce > 設定 > 配送 >**[配送ゾーン](https://woocommerce.com/document/setting-up-shipping-zones/) に移動し、顧客の住所が、少なくとも 1 つのアクティブな配送方法がある定義済みのゾーンと一致していることを確認します。

#### ブラウザとサイトのキャッシュをクリアする

キャッシュプラグインやサーバーサイドキャッシュにより、ライブレートの更新が正しく表示されない場合があります。この問題を解決するには、以下の手順に従ってください。

1. ブラウザのキャッシュをクリアする
2. サイトのキャッシュをクリアする
3. キャッシュプラグインを使用している場合、またはホスティングプロバイダーがキャッシュ機能を提供している場合は、以下のページがキャッシュから除外されていることを確認してください。
- カート
- チェックアウト
- マイアカウント
  

サイトとブラウザのキャッシュをクリアし、配送方法の除外を更新した後、サイトをリロードしてテストチェックアウトを完了し、正しい配送料が表示されていることを確認します。

#### AJAX でカートを強制的に更新する

テーマが AJAX アップデートをサポートしていることを確認するか、料金が自動的に再計算されない場合はユーザーに「カートを更新」を選択するように指示します。

このセクションでは、配送クラスに関連する一般的な構成ミスの概要と、その解決手順について説明します。

#### 配送クラス料金は配送料金に適用されません

これは、配送クラスと料金が空欄になっているか、正しく設定されていない場合に発生する可能性があります。以下の点をご確認ください。

1. **商品クラスの割り当て:** 商品データパネルで、配送設定を管理するオプションを選択します。各商品が正しいクラスに割り当てられていることを確認してください。
2. **配送方法クラス料金:** 定額配送方法の設定で、該当する「配送クラス料金」オプションにクラス固有の料金が定義されていることを確認してください。

#### 送料は常に $0 と表示されます

定額配送方法が有効になっているにもかかわらず、料金が 0 ドルと返される場合は、以下を確認してください。

1. **基本料金の設定:** 少なくとも基本料金または配送クラス料金を設定してください。
2. **計算タイプを設定:** 配送クラスの計算タイプを「クラスごと」または「注文ごと」に設定します。これは、配送料金の計算方法によって異なります。この設定は、チェックアウト時の料金の適用方法に直接影響します。

顧客が配送先住所を入力するまで配送料が計算されない場合は、以下を確認できます。

1. **計算設定を確認する:** **WooCommerce > 設定 > 配送 > 配送設定 > 計算** で、「住所が入力されるまで配送料を非表示にする」チェックボックスがオフになっていることを確認してください。
2. **位置情報の設定を確認する:** Geolocate を使用している場合は、**WooCommerce > 設定 > 統合 > MaxMind Geolocation** の [統合タブ](https://woocommerce.com/document/maxmind-geolocation-integration/#create-a-maxmind-license-key) に、MaxMind Geolocation の適切な [ライセンスキー](https://woocommerce.com/document/maxmind-geolocation-integration/#add-maxmind-geolocation-license-key-to-woocommerce-settings) が入力されていることを確認してください。

[配送クラス](https://woocommerce.com/document/product-shipping-classes/)は、[定額配送](https://woocommerce.com/document/flat-rate-shipping/#shipping-classes)などの配送方法における送料ルールを定義します。特定の配送クラスの商品でチェックアウト時に正しい料金が表示されない場合は、以下の設定に誤りがないかご確認ください。

**商品に正しい配送クラスが割り当てられていることを確認してください:** 商品データパネルで、配送設定を管理するオプションを選択します。正しい配送クラスが選択され、保存されていることを確認してください。変動商品の場合は、各バリエーションの設定を個別にご確認ください。

**適切な配送ゾーンの配送方法設定をご確認ください。** **WooCommerce > 設定 > 配送 > 配送ゾーン** に移動します。顧客が該当するゾーンを選択し、「定額料金」（またはその他の該当する方法）を開きます。「配送クラス料金」で、該当する配送クラスの料金を入力していることを確認してください。

**定額料金がコストを優先する仕組みを理解する:** デフォルトでは、定額料金配送はカート内の商品の中で最も高いクラスのコストを使用します。[定額料金の配送クラスを統合](https://woocommerce.com/document/flat-rate-shipping/#shipping-classes)してクラスごとに送料を請求する場合は、「計算タイプ」で「クラスごと：クラスごとに送料を個別に請求する」を有効にする必要があります。

配送方法は、ショートコードを使用する場合とチェックアウトブロックを使用する場合で表示が異なる場合があります。両方のチェックアウトインターフェースで十分にテストしてください。

- **ショートコードベースのチェックアウト:** `[woocommerce_checkout]` ショートコードを使用して、ショートコードベースのチェックアウトを作成します。
- **ブロックベースのチェックアウト:** ブロックベースのチェックアウトには、チェックアウトブロックを追加します。

チェックアウト時に重複した受け取りオプションが表示される場合は、**WooCommerce > 設定 > 配送 > ローカルピックアップ** でブロックベースのローカルピックアップオプションを有効にする前に、従来のローカルピックアップ方法が配送ゾーンに追加されたことが原因である可能性があります（チェックアウトブロックを使用していない場合は、このオプションは表示されません）。これは、両方のローカルピックアップオプションが同時に有効になっていることを意味します。

**重要:** お客様に混乱が生じる可能性があるため、両方の受け取りオプションを同時に使用しないでください。両方のオプションが有効になっている場合、店舗受け取り設定ページに通知が表示されます。

**解決策:** ブロックベースのチェックアウトをご利用の場合は、**WooCommerce > 設定 > 配送** で、配送エリアから従来のローカルピックアップ方法を削除してください。ブロックベースのチェックアウトの新しいローカルピックアップオプションは、配送エリアに追加する必要はありません。**WooCommerce > 設定 > 配送 > ローカルピックアップ** のローカルピックアップ設定ページで個別に管理できます。

WooCommerceは、顧客の配送先住所に一致する最初の配送地域を顧客にマッチングします。カートページやチェックアウトページで正しい配送方法が表示されない場合は、配送地域の順序が間違っている可能性があります。

**ベストプラクティス:** チェックアウト時に適切なオプションを表示するには、配送ゾーンを最も具体的なものから最も広いものの順に並べます（例：市、州、国）。[配送ゾーンの並べ替え](https://woocommerce.com/document/setting-up-shipping-zones/)について詳しくは、こちらをご覧ください。

配送ゾーンに追加された配送方法に設定されたタイトルは、チェックアウト手続き中に顧客に表示されます。配送ゾーンに追加された配送方法を編集することで、配送方法のタイトルを簡単に変更できます。

WooCommerce コア機能との互換性を確保するには、デフォルトのページ エディターと、コアの WooCommerce カートおよびチェックアウトの [ショートコード](https://woocommerce.com/document/woocommerce-shortcodes/) または [ブロック](https://woocommerce.com/document/woocommerce-store-editing/customizing-cart-and-checkout/) を使用します。

カートページやチェックアウトページにサードパーティ製のテーマ、ビルダー、プラグインを使用していて、配送に関する問題のトラブルシューティングを行っている場合は、WooCommerceのデフォルト設定に切り替えて原因を特定してください。デフォルトのページエディターと、WooCommerceのコアとなるカートページとチェックアウトページのショートコードまたはブロックを併用してください。

これにより、問題がカスタム設定に関連しているのか、WooCommerce 自体に関連しているのかを判断するのに役立ちます。

製品を「仮想」としてマークすると、その商品のすべての配送オプションが無効になるため、仮想製品では配送計算が実行されず、チェックアウト時に配送フィールドが表示されません。

**製品を仮想として設定するには:** 製品を編集し、[仮想] チェックボックスをオンにします。

配送税はカート商品または標準税率に基づいて算出されます。チェックアウト時に配送料に税金が適用されない場合は、多くの場合、税金設定の誤りが原因です。こちらの[配送税設定のトラブルシューティングガイド](https://woocommerce.com/document/setting-up-taxes-in-woocommerce/)をご確認ください。

定額料金、無料、およびローカルピックアップの主要配送方法は、配送区分、配送地域、配送方法と連動しており、商品の重量や寸法はこれに影響しません。ただし、これは顧客に商品メタデータを適切に表示するために役立ち、特にリアルタイム料金で配送する場合に役立ちます。

WooCommerceは、顧客の住所が他の定義済みの配送ゾーンと一致しない場合に「その他の地域」の配送ゾーンを適用します。この配送ゾーンは移動、名前変更、削除できません。

**特殊なケース:** 北アイルランドと同様に、[Shipping Locations Pro](https://woocommerce.com/products/shipping-locations-pro/) などの拡張機能を使用して、北アイルランドを英国から分離できます。

**郵便番号ゾーンの設定:** ワイルドカード (*) を使用した郵便番号に基づいて [新しい配送ゾーン](https://woocommerce.com/document/setting-up-shipping-zones/#adding-a-new-zone) を設定することもできます。

**郵便番号を使用して配送ゾーンを制限する方法:** ローカル配送ゾーンには、特定の郵便番号、数値の範囲 (例: 90210…99000)、ワイルドカード (*) を含む郵便番号のうち 1 つまたは複数 (1 行に 1 つ) を含めることができます。

**ワイルドカードの使用:** ワイルドカードを使用すると、同じ先頭文字を持つすべての郵便番号を取得できるため、数字以外の文字を含む郵便番号の場合はワイルドカードの使用をお勧めします。

**例:**

- `902*` は 90210 と 90288-1234 をキャプチャします
- `CB23*` は CB23 1EX をキャプチャします
- `CB2*` は CB2 3AA と CB23 1EX をキャプチャします

**注意:** 数値範囲でのワイルドカードの使用 (例: 902*…990*) はサポートされていません。

以下のオプションは WooCommerce に組み込まれており、多くのストアの最も一般的な配送ニーズに対応しています。

#### 定額送料

商品ごと、配送クラスごと、または注文ごとに[標準配送料](https://woocommerce.com/document/flat-rate-shipping/)を定義できます。

＃＃＃＃ 送料無料

顧客注文が特定の条件（最低注文金額など）を満たしている場合、または特定の地域内のすべての顧客に送料無料を提供する場合は、顧客の注文に「送料無料」オプションを追加できます。

#### ローカルピックアップ（レガシー）

お客様に[購入商品を店舗で直接受け取るオプション](https://woocommerce.com/document/local-pickup/)を提供できます。このオプションを利用するには、お客様は住所を入力し、この配送方法が追加された配送地域に一致している必要があります。税金は店舗の所在地に基づいて計算されます。

#### WooCommerce ブロック: ローカルピックアップ

[WooCommerce Checkoutブロック](https://woocommerce.com/document/woocommerce-store-editing/customizing-cart-and-checkout/checkout-block/woocommerce-blocks-local-pickup/)を使用すると、チェックアウト時に1つ以上の受取場所を指定できます。顧客は配送先住所を入力せずに受取場所を選択でき、税金は選択した受取場所に基づいて計算されます。

**重要な制限事項:** WooCommerce コアの配送機能では、チェックアウト時にリアルタイムの配送料を表示するオプションや、注文処理用のラベルを印刷する機能が提供されていません。これらの推奨事項については、以下をご覧ください。

以下のソリューションを使用すると、WooCommerce ダッシュボードから直接、または統合プラットフォームを通じて配送ラベルを生成して印刷できます。

#### WooCommerce 配送

WooCommerce ダッシュボードから直接 [割引された USPS、UPS、および DHL ラベルを印刷](https://woocommerce.com/products/shipping/) できます。

#### シップステーション

WooCommerce ストアを [ShipStation](https://woocommerce.com/products/shipstation-integration/) に接続して、ラベルの一括印刷、発送の自動化、割引料金などの注文処理を効率化できます。

#### スタンプス.com

[Stamps.com](https://woocommerce.com/products/woocommerce-shipping-stamps/) アカウントを使用して、WooCommerce ダッシュボードから直接 USPS 配送ラベルを生成および印刷できます。

**WooCommerce ShippingよりもShipStationを使うべき場合：** WooCommerce Shippingは、配送ニーズが中程度のほとんどの小規模から中規模のストアに適しています。大量の注文を処理する場合、ラベルを一括印刷する場合、複数のチャネルで販売する場合、または配送の自動化が必要な場合は、ShipStationをお選びください。

以下の拡張機能を使用すると、主要な運送業者の配送料金をチェックアウト時にリアルタイムで表示できるため、お客様に正確な配送料金をご提示できます。ただし、これはすべてを網羅したリストではありません。WooCommerce.com で利用可能なその他の配送拡張機能については、[マーケットプレイス](https://woocommerce.com/product-category/woocommerce-extensions/shipping-methods/) をご覧ください。

#### 利用可能なライブレート拡張機能:

- **UPS®配送方法** – チェックアウト時にUPSのリアルタイム料金が表示されます
- **USPS配送方法** – チェックアウト時にUSPSのリアルタイム料金が表示されます
- **FedEx配送方法** – WooCommerceストアにFedEx APIを統合することで、チェックアウト時にFedExのリアルタイム配送料金が表示されます（米国またはカナダのみ）
- **Royal Mail配送方法** – API接続を必要とせずに、チェックアウト時にRoyal Mailの配送料金が表示されます（英国のみ）

#### API 配送料金のトラブルシューティングに関する一般的なヒント

チェックアウト時に実際の配送料が表示されない一般的な問題を特定して修正するには、次のトラブルシューティング手順に従ってください。

1. **プラグイン固有のデバッグ/ログツールを有効にする** – デバッグモードを有効にする手順は[こちら](https://woocommerce.com/document/shipping-method-debug-mode/)をご覧ください。
2. **商品の重量と寸法データがすべて入力されていることを確認する**
3. **API認証情報を検証する** – APIを利用するリアルタイム配送料金拡張機能を使用する場合、認証情報が正しくないと問題が発生することがよくあります。これらの問題は、多くの場合、承認または認証エラーに関連しています。その一例は、[UPSトラブルシューティングガイド](https://woocommerce.com/document/ups-shipping-method/#troubleshooting)に記載されています。
4. **プラグイン設定で発送元の郵便番号を確認する**
