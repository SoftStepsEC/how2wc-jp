WooCommerce および WordPress でのセキュア ソケット レイヤー (SSL) 証明書の使用に関するよくある質問への回答を入手します。

Qualys SSL Labs [SSLサーバーテスト](https://www.ssllabs.com/ssltest/index.html) を利用して、SSL証明書のインストール後にウェブサイトが正しく設定されているかどうかを確認してください。このテストでは、ウェブサーバー設定のセキュリティも確認・評価されます。ウェブホスト側でこれらの変更に対応する必要があります。

いいえ。SSL 化は正しい方向への一歩ではありますが、SSL 化だけではサイトの PCI DSS 準拠は達成されません。詳しくは、[PCI-DSS 準拠と WooCommerce](https://woocommerce.com/document/pci-dss-compliance-and-woocommerce/) をご覧ください。

SSL証明書の機能と価格は、無料のものから年間1,000ドル以上するものまで様々です。高額なSSL証明書で提供される追加機能に見合うだけの収益がない限り、SSL証明書は必要ありません。

考慮すべき最も重要な 3 つの要素は次のとおりです。

- 暗号化レベル（256ビット推奨）
- ブラウザ認識
- 保証

例として、2つのSSL証明書を比較してみましょう。1つは年間10ドル、もう1つは年間1,000ドルです。通常、上記の3つの要素に関する2つの証明書の違いは、保証と（場合によっては）ブラウザ認識のみです。どちらのSSL証明書も、256ビット暗号化と99%以上のブラウザ認識率を提供するはずです。

最終的には、ブランド名と保険のために高い料金を支払うことになります。

無料および有料の SSL 証明書が用意されており、年間 10 ドル未満のお手頃価格のオプションもあります。

- [Let’s Encrypt](https://woocommerce.com/document/ssl-and-https/#free-option-lets-encrypt) (無料。一部のウェブホストではワンクリックインストールで利用可能)
- [Namecheap](http://www.namecheap.com/ssl-certificates.aspx) (有料)

通常、この問題は、Web サイトが `http://` を含む URL からロゴやその他の画像を読み込むときに発生します。

これを解決するには、ロゴ/画像の URL で `http://` を `https://` に置き換えます。

WordPress はほとんどのアセットの URL を `https://` に自動的に更新しますが、**リバース プロキシ** を使用したホスティング構成ではこの機能が機能しない可能性があります。

適切に設定されたウェブサーバーとリバースプロキシは接続タイプをそのまま渡すため、WordPressやその他のPHPファイルに変更を加える必要はありません。一部のウェブホストでは、`wp-config.php`ファイルへの追加パッチが必要になる場合があります。また、[Network Solutions](https://woocommerce.com/document/ssl-by-proxy-problems-network-solutions/)などのホストでは、設定に問題があるため、適切な修正プログラムが用意されていません。

これはお勧めしません。常時SSL接続は通常、設定されているキャッシュを破壊し、ウェブサイトの拡張時に問題を引き起こします。

小規模または中規模のサイトでは、常時SSL接続は問題にならない可能性があります。この点についてご質問がある場合は、ホスティングプロバイダーにお問い合わせください。
