WooCommerceには、問題のトラブルシューティング、パフォーマンスの最適化、主要な設定を一元管理するためのツールが組み込まれています。データベースのクリーンアップやサイトのキャッシュクリアなどのWooCommerceシステムツールは、WooCommerceをスムーズに動作させるのに役立ちます。

これらの設定には、**WooCommerce > ステータス > ツール** からアクセスできます。

トランジェントキャッシュとは、データが一定期間データベースに保存され、その後自動的に削除されるキャッシュの一種です。このデータにはすぐにアクセスできるため、時間を節約し、ウェブサイトのパフォーマンスを向上させることができます。

よくある例としては、ショッピングカートの内容が挙げられます。これらの内容はトランジェントを使用して保存されるため、顧客はカートページを離れても、後で戻ってきても商品がまだ残っていることを確認できます。

WooCommerce がインストールされアクティブになっている場合、以下のツールはデフォルトで利用できます。

**すべてのショップと製品の一時データを削除**: ショップと製品に関連する一時データをクリアします。

**すべての WordPress 一時データを削除**: 期限切れの一時データを削除します。

**親製品のないバリエーションを削除する**: これらは、関連付けられたメイン製品がなくなった製品バリエーションであり、製品数の誤りなどのエラーが発生する可能性があります。

**期限切れのダウンロード権限を削除**: 期限が切れた、または許可されたダウンロード数の制限に達したダウンロードの権限を削除します。

**商品ルックアップテーブルを再生成**：これらのテーブルは商品データを保存し、迅速なアクセスを可能にしてデータベースの高速化に貢献します。商品データとの同期が取れていない場合、誤った商品情報が表示される可能性があります。商品ルックアップテーブルを再生成することで、実際の商品データとの整合性が確保されます。

**商品数キャッシュをリセット**：これにより、各カテゴリーまたは属性の商品数のキャッシュが更新されます。これにより、設定変更後にサイトに表示される商品数の不正確さを修正できます。

**ユーザー権限をデフォルトにリセット**：これを使用すると、ユーザーの役割と権限をデフォルト設定にリセットできます。プラグインやカスタムコードによってこれらの設定が変更され、ユーザーが本来アクセスできるはずの管理ページにアクセスできなくなるなどの問題が発生する場合に役立ちます。

**アクティブな顧客セッション データを削除**: カートの内容を含む、進行中のすべての顧客セッションからデータを消去します。

**テンプレートキャッシュをリセット**: テーマテンプレートのキャッシュをクリアします。テーマのカスタマイズがフロントエンドに表示されない場合に役立ちます。

**不足しているWooCommerceページを作成**: 既存のページを上書きせずに、不足しているデフォルトの[WooCommerceページ](https://woocommerce.com/document/woocommerce-pages/)を生成します。これは、カートページやチェックアウトページが割り当てられていない、または不足している場合に役立ちます。

**すべての税率を削除**: 設定されているすべての税率を削除します。これは、税金の設定を最初からやり直す必要がある場合に便利です。

**ショップ画像の再作成**：カスタマイザーまたはテーマで画像サイズ設定を変更すると、ショップ画像が再生成され、以前のサイズが反映されます。このプロセスはバックグラウンドで実行されるため、時間がかかる場合があります。

**WooCommerce データベースを更新**: これにより、WooCommerce データベースが最新バージョンに更新されます。バージョンの不一致が問題の原因となっていると思われる場合に役立ちます。**この更新を実行する前に、必ずデータベースをバックアップしてください**。

このツールは注文住所の全文検索インデックスを再作成します。これは、顧客住所に基づいて注文情報を検索するときに役立ちます。

FTS 検索の詳細については、[こちら](https://woocommerce.com/document/high-performance-order-storage/#full-text-search-indexes) をご覧ください。

**不足している WooCommerce テーブルがないか確認します**: これにより、必要なすべての WooCommerce テーブルがデータベース内に存在することが確認されます。

**WooCommerce Analyticsのキャッシュをリセット**：これによりアナリティクスのキャッシュがクリアされます。アナリティクスデータが不正確に見える場合に役立ちます。その後、「アナリティクス」→「設定」ページからデータを再インポートできます。

**リモート受信トレイ通知を更新**ツールを使用すると、WooCommerce Marketplaceから新しい通知を更新したり取得したりできます。これらの通知は、タイムリーな情報、製品のアップデート、プロモーション、その他の重要なメッセージを提供します。

WooCommerce でリモート通知を有効にするには、次の設定を有効にする必要があります。

1. **WooCommerce > 設定 > 詳細設定 > WooCommerce.com** に移動します。
2. **マーケットプレイスの提案** を有効にします。

有効にすると、WooCommerce は WooCommerce マーケットプレイスを通じて入手可能な製品やサービスに関連する通知を表示し、役立つ提案や更新について知らせてくれます。*リモート受信トレイ通知の更新*ツールを使用すると、WooCommerce.com からの新しい通知を更新して同期できるため、関連する製品の更新やベストプラクティスなど、最新のマーケットプレイスの提案を常に把握できます。

WooCommerce の **受信トレイの通知を削除** ツールを使用すると、「スラッグ」を使用して受信トレイから特定の通知を削除できます。この機能は、受信トレイを管理し、関連性のある重要な通知のみを表示するのに便利です。

各受信トレイ通知には、「スラッグ」と呼ばれる固有の識別子があります。削除したい通知のスラッグを見つけて、「**受信トレイ通知を検索**」というテキストボックスに**通知スラッグ**を入力すると、特定の受信トレイ通知を削除できます。

このツールは、製品カタログの新規スキャンを開始します。既存の承認済みダウンロードディレクトリルールに含まれていないダウンロード可能な製品パスが検出されると、それらはリストに追加されますが、最初は無効になっています。

[詳細については、こちらをご覧ください](https://woocommerce.com/document/approved-download-directories/#additional-tools)。

このツールは、既存の承認済みダウンロード ディレクトリ リストを完全に削除するために使用されます。

[詳細については、こちらをご覧ください](https://woocommerce.com/document/approved-download-directories/#additional-tools)。

このツールは、注文に関連付けられた古いクーポンデータを、新しい簡素化された形式に移行するのに役立ちます。具体的には、注文アイテムメタデータ内の既存の「coupon_data」エントリを「coupon_info」エントリに変換し、データの保存と取得を最適化します。この変換プロセスは、アクションスケジューラを使用してバックグラウンドで実行されるため、店舗の業務を中断することなく、シームレスかつ効率的に処理されます。

このツールは、変換可能なクーポンエントリがある場合にのみ有効になります（例：「現在変換可能なエントリはありません」のような通知が説明に表示されます）。

`coupon_data` から `coupon_info` への移行に関する詳細は、[注文クーポン明細ストレージの変更](https://developer.woocommerce.com/2024/02/08/changes-in-order-coupons-line-item-storage/) を参照してください。

このツールは、期限切れの一時ファイルを削除するアクションをスケジュールし、即時実行するために使用します。期限切れの一時ファイルは24時間ごとに削除されます。

[高パフォーマンス注文ストレージ (HPOS)](https://woocommerce.com/document/high-performance-order-storage/) が有効になっている場合は、さらにいくつかのツールが利用できるようになります。

**レガシーテーブルからの注文データのクリーンアップ**ツールは、HPOSが完全に稼働し始めた後に、レガシーテーブルから古い注文データを消去するように設計されています。**HPOSが注文保存の権限テーブルとして設定され、**互換モードがオフになっている場合、このプロセスはレガシーテーブルから古い注文情報を削除することでデータベースのパフォーマンスを最適化します。**

バックグラウンドでは、WooCommerce はコマンド `wc hpos cleanup all` を実行します。

このコマンドは、レガシー テーブルから古い注文データを削除し、削除前に各注文を検証して、レガシー 投稿データに HPOS よりも新しい情報が含まれていないことを確認します。

**注意:** このツールは、投稿テーブルからプレースホルダー レコード (タイプ `shop_order_placehold` の投稿) を削除しません。

**カスタム注文テーブルの削除** ツールを使用すると、カスタム注文テーブルを完全に削除できるため、HPOS から従来の注文ストレージに戻す場合にデータベース領域を解放できます。

このツールは、HPOS が正式な注文ストレージとして設定されておらず、HPOS とレガシー テーブル間の同期が無効になっている場合に使用する必要があります。
