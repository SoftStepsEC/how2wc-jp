WordPress と WooCommerce の最も優れた点は、コードとコンテンツを必要に応じて変更できることです。つまり、Web サイト全体を変更してカスタマイズできます。

WooCommerce を使ってオンライン販売を行う最大のメリットの一つは、その柔軟性です。プラグイン、拡張機能、テーマを活用して要件を満たし、その後、調整やコーディングを行うことで、サイトの外観や機能をカスタマイズできます。

ただし、サイトを微調整する際は従うべき**ベスト プラクティス**がいくつかあり、これらの変更をどのように行う必要があるかは必ずしも明らかではありません。

多くのユーザーは、デフォルトの外観からテーマをカスタマイズし、リンクの色やレイアウト、雰囲気を変えています。多くの場合、コードを直接変更するのが最も簡単な方法です。しかし、この方法では、ファイルが新しいバージョンで上書きされると、すべての変更が失われるため、将来のアップデート時に問題が残ります。

このチュートリアルでご紹介したいのは、テーマをカスタマイズするための最適な方法です。これにより、更新が可能な限り簡単になり、カスタマイズを制御できるようになります。

カスタマイズには 2 つの方法があります。

- **プラグイン** — 軽微な変更
- **子テーマ** — 大幅な変更

スタイルシートまたは CSS に小さな変更を加えるだけの場合は、カスタム スタイルを追加できるプラグインを使用するのが最適です。

CSS または [Sass](http://sass-lang.com/) に関連する変更には、**Jetpack** を使用することをお勧めします。

Jetpack をインストールします。完了したら、**ウェブサイト > ダッシュボード > Jetpack > 設定** に移動し、**カスタム CSS** を有効にします。

次に、**「外観 > CSS 編集」** に進みます。ここで、すべてのカスタム CSS スタイルを追加できます。

**例**

**Chrome Dev Tools** を使って、Storefront テーマのナビゲーションリンクの色を確認する簡単な例を見てみましょう。要素の検査方法は以下のとおりです。**要素を右クリック > 検査** すると、nav 要素のスタイル名が表示されます。

上のスクリーンショットでは、ナビゲーションのリンクの色のスタイルが次のように表示されています。

```
.main-navigation ul li a {
  color: #ffffff;
}
.main-navigation ul li a:hover {
  color: #e6e6e6;
}
```

次に、**これをcustom.css**に貼り付けます。色を赤、ホバー時の色を白に変更します。変更しない他のスタイルも削除します。これで、最終的な見た目は次のようになります。

```
.main-navigation ul li a {
  color: red;
}
.main-navigation ul li a:hover {
  color: white;
}
```

**custom.css** を保存してページを更新すると、ナビゲーション リンクの色が赤になり、マウス カーソルを合わせると白に変わることがわかります。

テーマに既に存在するスタイルを補完するために、スタイルシートに新しいスタイルを追加することもできます。テーマがアップデートされても、Jetpack は既存のスタイルをそのまま維持します。新しいバージョンにアップデートする前に、必ずテーマをバックアップすることをお勧めします。また、アップグレード方法については、[アップグレードチュートリアル](https://woocommerce.com/document/how-to-update-your-site/)もご確認ください。

[コード スニペット プラグイン](https://wordpress.org/plugins/code-snippets/)を使用すると、`functions.php` ファイルに貼り付けるのではなく、サイトにスニペットを追加し、その機能について簡単な説明を記述して、必要に応じてこれらのスニペットを有効または無効にすることができます。

人気のあるテーマのスタイルシートまたは CSS を変更する場合、将来のアップデートに対してカスタマイズを保持するために子テーマを使用するのが最適です。

多くの人気テーマには子テーマが用意されており、インストールして有効化することで、コーディングやスタイルの調整をすることなく親テーマの外観を変更できます。購入済みの子テーマは**カスタムコードには使用しないでください**が、他の人のデザインを通してテーマの外観を変更する手段となります。例えば、当社のStorefrontテーマには、非常に豊富な子テーマのカタログが用意されています](https://woocommerce.com/product-category/themes/storefront-child-theme-themes/)。

このような子テーマは、親テーマにカスタムコードを追加しない方が良いのと同じ理由で、カスタムコードを追加するのには適していません。子テーマが更新されると、すべてのカスタムコードも上書きされてしまいます。

あるいは、子テーマを自分で作成することもできます。これは、親テーマの上にレイヤーを追加するようなもので、カスタムコードを優先し、親テーマの次回のアップデートでコードが削除されるのを防ぐことのみを目的としています。

弊社のチュートリアル「子テーマの設定と使用方法」を使って、独自の子テーマを作成できます。子テーマの仕組みについては、WordPress.orgの「子テーマ」でも詳しく説明しています。

- [子テーマ](http://codex.wordpress.org/Child_Themes)
- [WordPressにカスタムコードを安全に追加する方法](https://www.skyverge.com/blog/add-custom-code-to-wordpress/)
- [WooCommerceカスタマイズのヒント](https://woocommerce.com/2014/08/five-quick-woocommerce-customization-tips/)

まだ質問があり、サポートが必要ですか?

このドキュメントは、無料の[コア WooCommerce プラグイン](https://wordpress.org/plugins/woocommerce/)に関するものです。このプラグインのサポートは、[WordPress.org のコミュニティフォーラム](https://wordpress.org/support/plugin/woocommerce) で提供されています。このフォーラムを検索すると、同じ質問が過去に投稿され、回答されているケースがよくあります。フォーラムを利用するための WordPress.org アカウントをまだ作成していない場合は、[作成方法はこちら](https://make.wordpress.org/contribute/join/)をご覧ください。

- ここで紹介したコア機能を**拡張**したい場合は、[WooCommerce マーケットプレイス](https://woocommerce.com/products/)で利用可能な拡張機能をご確認ください。
- 継続的な高度なサポートや WooCommerce 向けの**カスタマイズ**が必要ですか？[Woo エージェンシー パートナー](https://woocommerce.com/customizations/)をご活用ください。
- 独自の WooCommerce 統合機能や拡張機能を開発している**開発者**ですか？[開発者向けリソース](https://developer.woocommerce.com/)をご確認ください。
