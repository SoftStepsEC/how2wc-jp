---
title: プラグインとテーマの競合をテストする方法
url: https://woocommerce.com/document/how-to-test-for-conflicts/
---

競合とは、コードの2つの部分が矛盾する信号を送信することで発生するエラーまたは誤動作のことです。例えば、プラグインAは注文を「完了」ステータスにする信号を送信しますが、プラグインBはそれを防ぐことができます。

テーマとプラグインは追加の機能を提供しますが、ウェブサイト上で実行されるコードが増えるため、互換性の問題が発生するリスクが高まります。WooCommerce では自社の拡張機能との互換性を確保するよう努めておりますが、WooCommerce 向けに開発されたサードパーティ製品は、必ずしも当社のソフトウェアと互換性があるとは限りません。このドキュメントページでは、プラグインとテーマの競合をテストする方法について説明します。

競合の原因を特定するには、テストが唯一の方法です。

サイトのバックアップを作成することを強くお勧めします。プラグインの無効化と有効化は通常は問題を引き起こしませんが、万が一問題が発生した場合でも、適切なバックアップを作成しておけば、時間と労力を大幅に節約できます。

ほとんどのホスティングプロバイダーはバックアッププランを提供しています。提供されていない場合は、次の2つの選択肢があります。

- [Jetpack バックアップ](https://jetpack.com/support/restoring-with-jetpack-backup/) をご検討ください。
- **ステージングサイト** を使用してください。これは本番サイトのクローン（注文処理中で訪問者がいるサイト）です。本番サイトに影響を与えず、収益の損失を招くことなく、安全に競合をテストできます。ほとんどのホスティングプロバイダーがサポートを提供しています。また、ほとんどのホスティングプラットフォームでは、[WP Staging](https://wordpress.org/plugins/wp-staging/) プラグインを使用してダッシュボードでクローンを作成できます。

テーマとプラグインの競合をトラブルシューティングするには:

1. [すべてのプラグインとテーマを更新](https://woocommerce.com/document/how-to-update-woocommerce/)。場合によっては、サイトソフトウェアを最新バージョンにすることで競合を解決できることがあります。新しいリリースには、新機能だけでなく、セキュリティとバグ修正も含まれています。
2. Twenty Twenty-F​​iveなどのWordPressのデフォルトテーマに切り替えるか、WooのStorefrontテーマをインストールして切り替えて、問題が解決するかどうかを確認してください。
- 問題が解決しない場合は、テーマが問題の原因です。以下の方法をお試しください。
- テーマを変更する。
- テーマの作者に連絡して修正を依頼する。
    
- 問題が解決しない場合は、次の手順に進みます。
  
3. 問題が発生しているWooCommerceとWooCommerce拡張機能を除くすべてのプラグインを**一時的に無効化**します。
4. 競合がまだ存在するかどうかをテストします。テスト方法は、発生している競合の種類によって異なります。
- サイト閲覧中またはダッシュボード閲覧中に競合が発生した場合は、同じ場所に移動します。例：
- 商品編集画面で機能が動作しない
- カートページでボタンが表示されない
    
- 特定のプロセス後にのみ競合が発生する場合は、同じ手順を再度実行してください。例としては、以下のものがあります。
- 商品Aを追加して支払いを済ませた後、注文が「完了」ではなく「保留中」と表示される。
- 2つの商品をカートに追加する際にエラーが表示される。
    
- 競合が解消された場合は、無効化したテーマまたはプラグイン／拡張機能が競合の原因であったことを意味します。競合がまだ解消されていない場合は、以下の「ドロップインと必須使用」および「競合テストの失敗」をご覧ください。
  
5. 以下の方法で、どのプラグインが競合の原因となっているかを特定します。
- プラグインを1つずつ再有効化する
- **再有効化するたびに再度テストする**。プロセス関連の競合の場合、これは同じプロセスを何度も再作成することを意味します。
  

- [Meks Quick Plugin Disabler](https://wordpress.org/plugins/meks-quick-plugin-disabler/) は、テスト完了時に有効になっていたプラグインを記憶します。
- [Theme Switcha](https://wordpress.org/plugins/theme-switcha/) を使用すると、フロントエンドに影響を与えることなく、サイト上のテーマの切り替えをテストできます。
- [WP Rollback](https://wordpress.org/plugins/wp-rollback/) を使用すると、テーマやプラグインを手動で再インストールすることなく、以前のバージョンに戻すことができます。特定のアップデートで問題が発生している場合のトラブルシューティングに役立ちます。
- [Query Monitor](https://wordpress.org/plugins/query-monitor/) は、サイトのパフォーマンス問題を特定するのに役立ちます。データベースクエリ、PHPエラー、HTTP API呼び出しに関する詳細な情報を提供します。この情報は、サイトの効率性と速度を確認するのに役立ちます。

一部のサイトでは、プラグインリストに「必須」や「ドロップイン」のセクションがあります。これらのセクションは直接無効化することはできませんが、競合の原因となる可能性があります。

- これらのプラグインの一部は、**別のプラグイン**によってインストールされ、「ヘルパープラグイン」として機能します。競合テストを行う際に、これらの「親」プラグインを無効化すると、これらのプラグインも無効化されます。
- また、多くのホスティング会社は、サーバーのセットアップを容易にするため、ドロップインプラグインや必須プラグインをサイトにプリインストールしています。

上記のテストを行っても後者のケースで競合が続く場合は、ホスティング会社がインストールしたドロップインプラグインが原因である可能性があります。例えば、ドロップインキャッシュプラグインによる競合が発生するケースがあります。この場合、ホスティング会社に連絡して無効化についてサポートを受ける必要があります。

競合テスト中にブラウザのキャッシュをバイパスすることで、ウェブサイトの最新バージョンで作業できるようになります。キャッシュによって古い情報が表示される場合があり、問題の根本原因の特定を妨げる可能性があります。

Chrome でこれを行うには、**[表示] > [開発者] > [開発者ツール] > [ネットワーク]** をクリックし、ボックスにチェックを入れて [キャッシュを無効にする] を選択します。

このボックスの選択を解除すると、キャッシュを再度有効にすることができます。

競合の有無をテストしても問題やエラーが改善しない場合は、別の原因がある可能性があります。[セルフサービスガイド](https://woocommerce.com/document/woocommerce-self-service-guide)に詳細な解決策が記載されています。または、[サポートヘルプデスク](https://woocommerce.com/my-account/contact-support/)からハピネスエンジニアにお問い合わせください。サポートにお問い合わせいただく際は、以下の内容を詳しくお知らせください。

- 問題を再現するために実行できる手順。
- 競合を排除するために実行した競合テスト。

このドキュメントは、無料の[コア WooCommerce プラグイン](https://wordpress.org/plugins/woocommerce/)に関するものです。このプラグインのサポートは、[WordPress.org のコミュニティフォーラム](https://wordpress.org/support/plugin/woocommerce) で提供されています。このフォーラムを検索すると、同じ質問が過去に投稿され、回答されているケースがよくあります。フォーラムを利用するための WordPress.org アカウントをまだ作成していない場合は、[作成方法はこちら](https://make.wordpress.org/contribute/join/)をご覧ください。

- ここで紹介したコア機能を**拡張**したい場合は、[WooCommerce マーケットプレイス](https://woocommerce.com/products/)で利用可能な拡張機能をご確認ください。
