このドキュメントでは、Secure Sockets Layer（SSL）とHypertext Transfer Protocol Secure（HTTPS）の概要を説明し、WooCommerceストアのセキュリティ確保における重要性を解説します。サイトに無料または有料のSSL証明書を設定する方法、そして顧客データの保護と信頼性の向上に不可欠な理由を解説します。

このガイドの目的は、ユーザーにとって安全で信頼できるショッピング体験の確保を支援することです。

さらに、WooCommerce の **Force SSL** 設定について解説し、よくある SSL の問題に対するトラブルシューティングのヒントもご紹介します。多くの場合、SSL を機能させるには、ホスティング会社や開発者と協力するのが最善です。

Secure Sockets Layer (SSL) は、Web 上で次の目的で使用されるプロトコルです。

- ブラウザからサーバー（およびその逆）に送信されるウェブサイトデータを暗号化して保護します。
- サイトの認証を行い、訪問者にサイトの身元が確認されていることを保証します。

SSL は、Web または内部ネットワークを介して 2 台のマシンまたはデバイス間に安全な接続を確立し、機密データを保護し、非公開で送信できるようにします。

Hypertext Transfer Protocol Secure (HTTPS) は、ブラウザーと Web サイト間でデータが送信されるプロトコルである HTTP の安全なバージョンです。

SSL 証明書は、ウェブサイトの信頼性と所有者の身元を確認した後に発行されます。これは、サイトの URL のプレフィックス「https://」と南京錠アイコンで示されます。

ほとんどの決済ゲートウェイはSSLを必須としており、**弊社では**SSLを**強く推奨**しています。ストアを訪れる顧客は、SSLを信頼性の証と見なします。閲覧、購入、アカウント情報や支払い情報の入力時に、アドレスバーに「https://」と南京錠アイコンが表示されることを期待しています。

顧客情報を安全に送信する必要がある場合は、SSL が必須です。以下の場合に SSL を使用する必要があります。

- 住所やクレジットカード番号などの機密情報を送受信するため、それらを安全に保管する必要があります。
- サイトへのログインとサインアップのセキュリティを確保する必要があります。
- プライバシーとセキュリティの要件を遵守する必要があります。
- ユーザーにウェブサイトを信頼してもらいたいと考えています。
- お客様にオンラインストアで商品を購入してもらいたいと考えています。

SSL を使用する Web サイトは、検索エンジンで上位にランクされ、読み込みが速くなるとも言われています。

ウェブサイトでSSLを使用するには、**SSL証明書**を取得する必要があります。無料と有料のオプションがあります。

この取り組みは素晴らしいですね！[Let’s Encrypt](https://letsencrypt.org/)は、より安全でプライバシー重視のインターネットを実現するために、無料のSSL証明書を提供する証明機関（CA）です。[Let’s Encryptの詳細](https://letsencrypt.org/about/)と[仕組み](https://letsencrypt.org/how-it-works/)をご覧ください。

Let’s Encrypt から無料の SSL 証明書を取得するには 2 つの方法があります。

#### 1.無料SSLを備えたホスティング会社を選択する

WooCommerceの[ホスティングパートナー](https://woocommerce.com/hosting-solutions/)は、数分でインストールできる無料のSSL証明書を提供しています。ホスティングパッケージにウェブドメインを同梱して購入する場合、多くの場合、インストールは不要です。

ホスティング会社にLet’s Encryptの無料SSL証明書を提供しているかどうかを確認し、指示に従ってください。提供されていない場合は、自分でインストールすることもできます。

#### 2. 自分でインストールする

ホスティング会社がクリックアンドインストールツールを提供していなくても、無料の SSL 証明書を自分でインストールするオプションを提供している場合は、Let’s Encrypt を使用できます。

**注:**WooCommerce.com では SSL 証明書のインストールと設定に関するサポートは提供しておりません。サポートが必要な場合は、ご利用のウェブホストにお問い合わせください。

一部のサードパーティプロバイダーは、サイトにインストールできる様々なSSL証明書を販売しています。ホスティング会社もSSL証明書を販売しており、新規サイトまたは既存のサイトへのインストールをサポートしてくれる場合があります。

低コストのオプションについては、「手頃な価格の SSL 証明書はどこで入手できますか?」を参照してください。(https://woocommerce.com/document/ssl-faq/#section-4)

サーバーに証明書が設定されると、「https://yoursite.com」（「yoursite.com」はお客様のドメインに置き換えてください）からストアにアクセスできます。ブラウザのアドレスバーには、セキュリティ保護されていることを示す**南京錠アイコン**が表示されます。また、Google Chromeでは、アドレスバーの歯車アイコンをクリックすると情報が表示される場合があります。

#### 新しいウェブサイト/ストアの場合

SSL 証明書をインストールした後、WordPress/WooCommerce の URL を更新する必要があります。

1. ストアのWordPress管理ダッシュボードで、*設定 > 一般*に移動します。
2. **WordPressアドレス（URL）**と**サイトアドレス（URL）**のフィールドを更新し、プレフィックス「https://」を含めます。
3. **変更を保存**ボタンをクリックします。

#### 既存のウェブサイト/ストアの場合

あなたのウェブサイト／ストアは既に検索エンジンによって「http://」アドレスとしてインデックスされているため、訪問者や顧客が引き続きあなたのサイトを見つけられるように、「https://」にリダイレクトする必要があります。[Really Simple SSL](https://wordpress.org/plugins/really-simple-ssl/) は、この作業に役立つサードパーティ製プラグインです。

その他のヒント:

- 検索と置換を使用して、コンテンツ内の内部リンクを修正します。
- Google Analytics を使用している場合は、設定を更新します。
- [SSL サーバーテスト](https://www.ssllabs.com/ssltest/) を使用して、SSL が実装されていることを確認し、問題があれば修正します。
- [SSL チェック](https://www.jitbit.com/sslcheck/) を実行して、安全でないリンクやコンテンツを検出します。

この設定を使用する前に、チェックアウトページだけでなく、ストア全体を `https://` で運用することを**強く推奨**します。このアドバイスに従わない場合は、チェックアウトページのみでSSLを有効にできます。

**注:** サイトの URL にすでに `https://` が含まれている場合、**SSL を強制** 設定は使用できません。

**強制SSL**を使用してチェックアウトページでSSLを有効にするには、*WooCommerce > 設定 > 詳細設定*に移動してください。この設定を有効にすると、特定のページが `https://` 経由でのみ表示されるようになります。

これらのページは次のとおりです。

- チェックアウト
- マイアカウント
- お支払い（チェックアウトエンドポイント）

これはSSL証明書の設定に問題があることを示している可能性があります。ホスティングプロバイダーにお問い合わせいただくことをお勧めします。

安全なページを表示しているときに安全でないコンテンツの警告が表示される場合は、**スクリプト**、**画像**、または**スタイルシート**が `https://` ではなく `http://` 経由で直接リンクされていることを示している可能性があります。

ほとんどの場合、リンクを `https://` に変更するか、相対 URL (`http://yoursite/wp-content/file` ではなく `/wp-content/file` など) を使用することでこれを修正できます。

安全でないリンクを特定するには、ブラウザに組み込まれている開発者ツールを使用してエラーコンソールを表示してください。表示しているページの安全でないリソースが一覧表示されます。また、サードパーティ製のプラグイン（[Better Find Replace](https://wordpress.org/plugins/real-time-auto-find-and-replace/)など）を使用して、サイト内で「http://」が使用されている箇所を検索し、「https://」に置き換えることもできます。

WooCommerceは、安全でないページをリダイレクトするために `is_ssl()` [WordPress関数](https://developer.wordpress.org/reference/functions/is_ssl/) を使用します。これにより、以下の場合にリダイレクトループが発生する可能性があります。

- セキュアURLへの接続を強制解除しようとする別のSSLプラグイン（[Easy HTTPS Redirection](https://wordpress.org/plugins/https-redirection/)など）がインストールされています。このプラグインを削除するか、**Force SSL**設定をオフにしてみてください。
- ホストがプロキシ経由のSSLを使用しているため、`https://`が検出できません。[プロキシ経由のSSLに関する問題](https://woocommerce.com/document/ssl-by-proxy-problems-network-solutions/)をご覧ください。

この高度なトピックについては、[WooCommerce 開発者向けドキュメント](https://developer.woocommerce.com/docs/ssl-and-https-and-woocommerce/) で詳しく説明しています。
