WooCommerce はすでにいくつかの言語に翻訳されており、すぐに翻訳に対応しています。必要なのは、ご利用の言語の翻訳ファイルだけです。サイトの言語を変更すると、[このリストにある言語には自動的にインストールされます](https://translate.wordpress.org/projects/wp-plugins/woocommerce/)。

このページでは以下の内容を学びます:

- WordPress を自分の言語で設定する方法。
- WooCommerce の言語に翻訳を投稿する方法。
- translate.wordpress.org から翻訳を手動でダウンロードする方法。
- WooCommerce のカスタム翻訳を作成する方法。
- ローカライズなしでテキストを翻訳する方法。

WooCommerce を翻訳する最初のステップは、サイトが翻訳したい言語に設定されていることを確認することです。WordPress にデフォルトの言語を設定する必要があります。これは以下の手順で設定できます。

1. WordPress管理ダッシュボードから**設定 > 一般**に移動します。
2. **サイトの言語**設定を希望の言語に変更します。
3. 下にスクロールして**変更を保存**をクリックします（これにより、ナビゲーションメニューの言語もすぐに変更されます）。
4. **ダッシュボード > 更新**に移動します。
5. 一番下までスクロールして**翻訳を更新**をクリックします。

言語ファイルが存在する場合、ショップはあなたのロケールで表示されます。存在しない場合は、以下に説明する手順に従って言語ファイルを作成する必要があります。

## ローカリゼーションを Core に提供

翻訳は[GlotPressプロジェクト](https://translate.wordpress.org/)で管理しています。ご自身の言語に翻訳された文字列を投稿したい場合、または新しい翻訳を開始したい場合は、WordPress.orgでアカウントを登録して翻訳を開始してください。

WooCommerce 翻訳は、[WooCommerce 翻訳プロジェクト](https://translate.wordpress.org/projects/wp-plugins/woocommerce) のチームによって管理されています。WooCommerce の**安定版**と開発版はそこで翻訳されており、誰でも貢献できます。

WordPress は WooCommerce をインストールまたはアップデートする際に、お使いの言語で少なくとも **90% 完成した安定版翻訳** を探し、自動的にダウンロードします。**安定版** 翻訳が存在するものの、完成度が 90% 未満の場合は、手動でダウンロードできます。すべてのユーザーが恩恵を受けられるよう、特定の言語で 90% 以上の完成度を目指して貢献していただくことを推奨します。

翻訳が初めての方は、[翻訳者ハンドブックをチェックして始めましょう。](https://make.wordpress.org/polyglots/handbook/tools/glotpress-translate-wordpress-org/)

から翻訳を手動でダウンロードするには、以下の手順に従ってください。

1. **WooCommerce Translations Project** にアクセスし、リストからご希望の言語を見つけます。
2. タイトルをクリックして、言語のサブセクションに進みます。

1. **Set/Sub Project** 列の見出しをクリックして、**Stable (最新リリース)** バージョンを表示およびダウンロードします。

1. ページの一番下までスクロールしてエクスポートオプションを選択します。サイトで使用できるように、**マシンオブジェクトメッセージカタログ (.mo)** ファイルとしてエクスポートします。

1. このファイルの名前を **woocommerce-YOURLANG.mo** に変更します（例：イギリス英語は *en_GB*、アフリカーンス語は *af*）。
2. このファイルをサイトの **wp-content/languages/plugins/** フォルダにアップロードします。
3. アップロード後、この翻訳を使用できます。

WooCommerce.com 拡張機能の多くでは翻訳版も提供されており、これに関する詳細とカスタム翻訳の追加方法については、[WooCommerce 拡張機能の翻訳](https://woocommerce.com/document/translate-woocommerce-extensions/) を参照してください。

WooCommerceには、すべての英語テキストを含む**言語ファイル**（.potファイル）が含まれています。この言語ファイルは、プラグインフォルダ内の**woocommerce/i18n/languages/**にあります。

### Loco Translate でカスタム翻訳を作成する

1. [Loco Translate](https://wordpress.org/plugins/loco-translate/) プラグインをインストールして有効化します。
2. **Loco Translate >** **プラグイン** に移動し、**WooCommerce** を選択します。

1. 新しい言語を追加するには、**「+ 新しい言語」** をクリックします。
2. ドロップダウンから追加する言語を**選択します**。
3. 翻訳ファイルを追加するフォルダを選択します。**「カスタム」** をお勧めします。
4. **「翻訳を開始」** をクリックします。

**注 #1:** Loco Translate では、翻訳の作成場所として 3 つの異なる場所を提供しています。独自の翻訳に最適なのは **Custom** です。これは、カスタム翻訳がアップデートによって元に戻ってしまうのを防ぐだけでなく、WooCommerce Subscriptions などの拡張機能内の文字列を正しく翻訳するためにも必要です。*Author* の場所はプラグイン内にあるため、拡張機能のアップデートによって上書きされてしまうため、また *System* の場所は *translate.wordpress.org* からの翻訳によって上書きされる可能性があるため、避けてください。**注 #2**: Loco Translate の使用中に技術的な問題が発生した場合は、[Loco Translate サポートチームにお問い合わせください](https://wordpress.org/support/plugin/loco-translate/)。

1. 完了したら**保存**します。

WooCommerce の新バージョンがリリースされ、サイトが更新された場合、新しい文字列を探すために .po ファイルを更新する必要があるかもしれません。「同期」ボタンを使って新しい文字列を探し、翻訳を開始してください。

### PoEdit でカスタム翻訳を作成する

PoEditは**Loco Translate**よりも高度な代替手段です。WooCommerceの翻訳にこの方法は、上級ユーザーまたは翻訳者にのみ推奨されます。

WooCommerceにはPOTファイルが付属しており、PoEditにインポートして翻訳できます。始めるには：

1. PoEdit を開き、**ファイル > POT ファイルから新しいカタログ** に移動します。
2. **woocommerce.pot** を選択すると、PoEdit にカタログプロパティウィンドウが表示されます。

1. 他の翻訳者があなたの情報を確認できるよう、**名前と詳細情報**を入力し、「**OK**」をクリックします。
2. **.poファイルを保存します**。翻訳対象に基づいて名前を付けます。例えば、GB翻訳の場合は*woocommerce-en_GB.po*という名前になります。これで文字列がリストされます。

1. 文字列を翻訳したら、**保存**してください。.moファイルが自動的に生成されます。
2. .poファイルを開き、**カタログ > POTファイルから更新**を選択して**更新**してください。
3. **ファイルを選択**すると、それに応じて更新されます。

[Webis Multilingual for WooCommerce](https://woocommerce.com/products/webis-multilingual-for-woocommerce/) は、月額サブスクリプションでマーケットプレイスからご利用いただけます。

1. [WooCommerce アカウント](https://woocommerce.com/my-account/downloads/) から .zip ファイルを**ダウンロード**します。
2. **WordPress 管理画面 > プラグイン > 新規追加** に移動し、**プラグインのアップロード** を選択します。ダウンロードしたファイルで、**ファイルを選択** を選択します。
3. 拡張機能を**今すぐインストール**し、**有効化** します。

詳細については、[プラグイン/拡張機能のインストールと有効化](https://woocommerce.com/document/installing-and-activating-woocommerce-extensions/)をご覧ください。

#### セットアップと構成

#### グローバル設定の構成

**WooCommerce > Webis Multilingual** に移動し、**設定** タブに移動します。

1. 言語
- 言語を追加するには、**「言語を編集」**をクリックし、追加したい言語を選択して保存します。
  
2. 言語コード
- このセクションでは、URLに表示される言語コードをカスタマイズできます。例えば、中国語（簡体字）のデフォルトの言語コードは「zh_CN」ですが、上記のスクリーンショットでは「cn」に変更しています。
  
3. 言語スイッチャー
- 「フラグを表示」チェックボックスをオンにすると、言語スイッチャーでフラグの表示/非表示を切り替えられます。
  
4. Cookie名
- お客様が選択した言語を記憶するためにCookieを使用します。サイトの言語がデフォルトの言語に頻繁に変更される場合は、ブラウザのCookieが無効になっているか、キャッシュされたページが表示されている可能性があります。後者の場合は、開発者にご相談の上、Cookie名をキャッシュバスティングCookieリストに追加してください。場合によってはCookie名の変更が必要になることがありますので、Cookie名を変更するオプションをご用意しています。
  

**WooCommerce > Webis Multilingual** に移動し、**文字列翻訳** タブに移動します。

このセクションを使用して、ストアフロントの文字列を翻訳します。

1. ページ下部の「文字列をスキャン」オプションを有効にし、「変更を保存」をクリックします。
2. 翻訳したい文字列がレンダリングまたは表示されるページにアクセスします。
3. 「文字列翻訳」メニューで文字列を検索し、「+」ボタンをクリックして翻訳を追加します。

**重要**: ページに文字列が多すぎる場合は、ページを数回更新する必要がある場合がありますのでご注意ください。また、スキャンが完了したら「文字列をスキャン」オプションを無効にすることが非常に重要です。このオプションはパフォーマンスに大きな影響を与え、サイトの速度を低下させる可能性があります。

プラグインを使えば、コンテンツの翻訳は簡単です。通常、元のコンテンツを入力する場所の下に、翻訳を追加するためのフィールドがあります。例えば、商品の翻訳を追加するには、商品編集ページを開き、Webis Multilingual画面要素を見つけて翻訳を追加します。同様に、投稿の翻訳を追加するには、投稿編集ページに移動し、Webis Multilingualメニューを見つけます。

以下にいくつかの例を挙げます。

製品の翻訳を追加するには、翻訳したい製品を編集し、「Webis Multilingual」セクションに翻訳を追加するだけです。

メニュー項目の翻訳を追加するには、**「外観」>「メニュー」**に移動してください。**例**：上記のスクリーンショットでは、「ショップ」メニュー項目の翻訳を追加しており、「すべての言語」、「中国語（簡体字）」、「日本語」の3つのドロップダウンリストが表示されています。「すべての言語」を選択すると、中国語と日本語の両方の翻訳を追加でき、「ショップ」メニューは両方の言語でストアフロントに表示されます。特定の言語のメニュー項目をストアフロントに表示したい場合は、「中国語（簡体字）」または「日本語」を選択してください。

プラグインを使えば、翻訳の追加は簡単です。普段コンテンツを作成する場所に移動すると、「Webis Multilingual」セクションがあり、そこに翻訳を追加できます。現在、商品、カテゴリー、属性、投稿、ページ、メニューの翻訳をサポートしています。

### 翻訳のアップグレードを安全に行う

WooCommerce は、他のプラグインと同様に、翻訳を **wp-content/languages/plugins** に保存します。カスタム翻訳を追加したい場合は、**wp-content/languages/woocommerce** ディレクトリを使用するか、スニペットを使用することができます。詳しくは、[WooCommerce 開発者ドキュメント](https://developer.woocommerce.com/docs/making-your-translation-upgrade-safe/) をご覧ください。

WordPressテーマの.POファイルを編集せずに、数語だけを翻訳または変更したい場合は、[Say What? プラグイン](https://wordpress.org/plugins/say-what/)が便利です。カスタムコードは必要ありません。

有効にすると、次のことを要求されます:

1. **元の文字列** – 翻訳するテキスト。正確な文字列を確認するには、プラグインのソースコードをご覧ください。
2. **テキストドメイン** – woocommerce
3. **表示したいテキスト**

チェックアウトページで、一部の文字列が翻訳されていないことに気付くかもしれません。例えば、下のスクリーンショットでは、配送方法の「ローカルピックアップ」、支払い方法の「代金引換」、そしてプライバシーポリシーに関するメッセージは翻訳されていませんが、フォームの残りの部分は翻訳されています。

これは通常、WooCommerce を初めてインストールし、デフォルトのサイト言語（英語）を選択し、その後サイト言語を別の言語に変更した場合に発生します。WooCommerce では、スクリーンショットで翻訳されていない文字列は、WooCommerce の初期インストール後にデータベースに保存されます。そのため、サイト言語を別の言語に変更した場合、これらの文字列はデータベースエントリであるため、WooCommerce は翻訳可能な文字列を検出できません。

これを修正するには、変更が必要な文字列に対応するWooCommerce設定に移動し、そこで直接翻訳を更新してください。例えば、上記のケースで文字列を修正するには、まず**Local Pickup**配送方法に対して以下の操作を行う必要があります。

1. **WooCommerce > 設定 > 配送 > 配送エリア** に移動します。
2. **ローカルピックアップ** 配送方法を追加した配送エリアを選択します。
3. **ローカルピックアップ** 配送方法の設定を開きます。
4. 配送方法の名前を適切な翻訳に変更します。
5. **保存** をクリックして設定を保存します。

次に、**代金引換**の支払い方法の場合：

1. **WooCommerce > 設定 > 支払い** に移動します。
2. **代金引換** の支払い方法を選択します。
3. 支払い方法のタイトル、説明、および指示を実際の翻訳に合わせて変更します。
4. **保存** をクリックして設定を保存します。

**プライバシーポリシー** に関するメッセージは次のとおりです。

1. **WooCommerce > 設定 > アカウントとプライバシー** に移動します。
2. **プライバシーポリシー** セクションまでスクロールします。
3. **登録プライバシーポリシー** と **チェックアウトプライバシーポリシー** のフィールドを正しい翻訳に編集します。
4. 下にスクロールして **設定を保存** をクリックします。

「チェックアウト」ページに戻り、新しい翻訳が反映されていることを確認します。

翻訳された文字列の一部が WooCommerce サイトに期待どおりに表示されない場合は、まず、これらの文字列の *ソース テキスト* セクションに **単数形** と **複数形** の両方の形式があるかどうかを確認します。

この場合、翻訳が期待通りに機能するには、両方の形式を翻訳する必要があります。そのためには、**Loco Translateの単数形と複数形のタブ**を使用してください。これらのタブは、下図のように画面下部の翻訳セクションに表示されます。
