title: WooCommerce システムステータスレポートを理解する

WooCommerce システムステータスレポートは、ストアの問題のトラブルシューティングに役立ちます。幅広い情報とツールが含まれており、ソフトウェアのバージョン、サーバー設定などを確認できます。

Woo サポートにサポートリクエストを送信する前に、以下の情報をご確認いただくことをお勧めします。必要に応じてサポートさせていただきます。システムステータスレポートを表示し、サポート用にコピーするには、以下の手順に従ってください。

1. **WooCommerce > ステータス** に移動します。
2. **システムレポートを取得** を選択し、**サポート用にコピー** して[サポートリクエスト](https://woocommerce.com/my-account/create-a-ticket/)にコピー＆ペーストするか、**サポート用にダウンロード** して添付ファイルとしてアップロードします。レポートに含まれる情報は、サポートチームが問題をより迅速に絞り込むのに役立ちます。

これには、サイトと WordPress のインストールおよび有効にした機能に関する一般情報が提供されます。

サーバーの一般情報が提供されます。

これにより、現在のデータベースのバージョンがわかり、すべてのデータベース テーブルとそれらが存在するかどうかが一覧表示されます。

ストアに存在する投稿タイプと、各タイプに含まれる投稿数を表示します。インストールしているプラ​​グインによって異なります。

接続が保護されているかどうかを表示します。信頼できない訪問者があなたの情報を閲覧しようとした場合、エラーは表示されません。

サイトにインストールされ、アクティブなすべてのプラグインが表示されます。ソフトウェアの現在のバージョン番号と、拡張機能のアップデートの有無を確認できます。

サイトにインストールされているものの非アクティブなプラグインがすべて表示されます。ソフトウェアの現在のバージョン番号と、拡張機能のアップデートの有無を確認できます。

これらは`wp-content`にありますが、特定のファイル名のみです。WordPress内の特定の機能を置き換えたり、拡張したりします。

サイトの一般設定が表示されます。

これには、[WooCommerce ページ](https://woocommerce.com/document/woocommerce-pages/) として指定されたページ ID とパーマリンクが表示されます。

たとえば、サイトが `https://example.com` の場合、ユーザーは `https://example.com/cart/` にアクセスしてカートページに移動します。

インストール時に現在実行されているテーマに関する情報が表示されます。

テーマが何をすべきか、何をすべきでないかについての標準がなく、Woo 以外のテーマが WooCommerce またはその拡張機能のいずれかと競合する可能性があるため、これはサイトのより問題のある領域になる可能性があります。

サイトに問題が発生している場合は、無料の[Storefrontテーマ](https://woocommerce.com/storefront)に一時的に切り替えることをお勧めします。Storefrontテーマを有効にすることで問題が解決した場合は、テーマに問題がある可能性があります。解決するには、アップデートを確認するか、テーマ開発者に連絡して修正プログラムがあるかどうかを確認してください。

このセクションには、デフォルトの WooCommerce テンプレート ページを上書きするファイルが表示され、テンプレートが古くなっているかどうかも示されます。

表示されるページのいずれかに問題がある場合は、[Storefront](https://woocommerce.com/storefront/) テーマに切り替えることで、元のテンプレートファイルが確実に使用されるようになります。詳しくは、[古くなった WooCommerce テンプレートファイルの修正](https://woocommerce.com/document/fix-outdated-templates-woocommerce/) をご覧ください。長期的な解決策については、テーマ開発者にお問い合わせください。

[WooCommerce Subscriptions 拡張機能がシステム ステータス レポートに追加する内容](https://woocommerce.com/document/subscriptions-system-status-report/) の詳細については、こちらのドキュメントを参照してください。

スケジュールされたアクションの数を、過去と最新の日付とともに表示します。アクションスケジューラに関する詳細な技術情報とFAQについては、[ActionScheduler.org](https://actionscheduler.org/)をご覧ください。

何らかの理由でサイトのシステム ステータス レポートにアクセスできない場合は、いくつかの代替手段があります。

`[example.com]/wp-admin/admin.php?page=wc-status` にアクセスし、`example.com` をサイトのドメインに置き換えて、関連ページに直接アクセスしてみてください。

または、管理画面から**「ツール」>「サイトヘルス」>「情報」** にアクセスして、そこから情報を共有することもできます。システムステータスレポートと同様に、サイトヘルスレポートにもクリップボードにコピーするボタンがあり、簡単に共有できます。

「ツール」タブには、WooCommerceシステムツールがあります。これらの設定は、サイトの問題のトラブルシューティング、サイトのパフォーマンスの最適化、設定の管理に役立ちます。

[Learn more about WooCommerce system tools.](https://woocommerce.com/document/understanding-the-woocommerce-system-status-report/tools/)

「ログ」タブは、WooCommerceがストア内で発生するすべてのイベントのログを保存する場所です。「致命的なエラー」などの一部のログは自動的に保存されますが、決済ゲートウェイなどのログは、該当する拡張機能の設定で事前に有効にする必要があります。

このタブでは、表示したいログをクリックしてログファイル全体を開くことができます。また、検索ボックスに検索したい語句を入力することで、すべてのログファイル内を検索することもできます。

サイトで**致命的なエラー**が発生した場合、そのエラーは「fatal-errors」という名前のログファイルに記録される可能性があります。これはデバッグ時に役立ちます。

[Learn more about navigating and understanding WooCommerce error logs.](https://woocommerce.com/document/finding-php-error-logs/)

WooCommerce Shipping & Tax のユーザーには、拡張機能の健全性、使用中の特定のサービス、デバッグ セクションなどを確認できる 5 番目のタブがあります。

ヘルスには、接続されたパーツのステータス（WooCommerce または Jetpack で注意が必要な点があるかどうか、サービスが稼働しているか停止しているか）が表示されます。

サービスには、特定のサービスの状態が表示されます。未使用、対処が必要なエラーが発生している、あるいは正常に動作しているなど、状況によって状況が異なります。デバッグログを有効にして問題を把握したり、サポートチームに連絡して配送ラベル印刷用のプリンターの設定をサポートしてもらったりすることも可能です。

「スケジュールされたアクション」タブでは、サイトのバックグラウンドで実行されるプロセスのログを確認できます。これには、サブスクリプションの更新、一部の顧客へのメール送信、テーマやプラグインの更新の確認などのタスクが含まれます。各タスクのステータス、完了したタスク、保留中のタスク、実行に失敗したタスクの詳細なエラー情報を確認できます。

[Learn more about WooCommerce scheduled actions.](https://woocommerce.com/document/understanding-the-woocommerce-system-status-report/scheduled-actions/)

まだ質問があり、サポートが必要ですか?

このドキュメントは、無料の[コア WooCommerce プラグイン](https://wordpress.org/plugins/woocommerce/)に関するものです。このプラグインのサポートは、[WordPress.org のコミュニティフォーラム](https://wordpress.org/support/plugin/woocommerce) で提供されています。このフォーラムを検索すると、同じ質問が過去に投稿され、回答されているケースがよくあります。フォーラムを利用するための WordPress.org アカウントをまだ作成していない場合は、[作成方法はこちら](https://make.wordpress.org/contribute/join/)をご覧ください。

- ここで紹介したコア機能を**拡張**したい場合は、[WooCommerce マーケットプレイス](https://woocommerce.com/products/)で利用可能な拡張機能をご確認ください。
- 継続的な高度なサポートや WooCommerce 向けの**カスタマイズ**が必要ですか？[Woo エージェンシー パートナー](https://woocommerce.com/customizations/)をご活用ください。
- 独自の WooCommerce 統合機能や拡張機能を開発している**開発者**ですか？[開発者向けリソース](https://developer.woocommerce.com/)をご確認ください。
